Introduction

Updates to Pension Master for the DC Generic Calculations.

System Level

A new field has been added to the Annual Amounts screen in System Data, BSP Disability, to record the state disability benefit and is used in calculating the members disability benefit.

The Contribution Earnings cap is extracted from the Earnings Cap field on the Annual Statutory Amounts.

Annual Statutory Amounts Screen

Reports Level

A template DC Generic Mail Merge Report has been added to all schemes for the administrator to copy and use to create a benefit statement mail merge report. This is not being done for non Irish schemes.

Mail Merge Report Screen

Category Level

We have set up a new DC Configuration Screen at Category level with the following:

  1. Fields to identify the salaries to use for Salary, Pensionable Salary, Risk Salary and Spouses DIS Salary.
  2. Several calculation have to be done per contribution type. As these are user defined, it is not possible to identify the contribution codes or how many there are. We have therefore set up a DC configuration screen at Category level to hold this information. We have allowed for up to 10 non TV DC contribution codes.
  3. Fields for Death In Service Multiples.
  4. Field for the Spouses Pension %.
  5. Fields to Calculate the Members Disability Benefit.

DC Generic Configuration Screen

A second screen has been added to the dc configuration screen at category level to be displayed for UK only schemes The screen holds projection criteria details held in factor tables to enable SMPI calculatiuons to be calculated.

DC Generic Configuration Screen - Projection Criteria

The factor tables have conditions as follows:

  • Interest Rate Factor Table - Factor table must be unisex actuarial type factor table, with factors for each age.
  • Expense Rate Factor Table - Factor table must be a term related unisex factor table.
  • Salary Increase Rate Factor Table - Factor table must be a term related unisex factor table.
  • Inflation Rate Factor Table - Factor table must be a term related unisex factor table.
  • Annuity Rate at NPD Factor Table - Factor table must be a term related, non unisex factor table. In addition the indexes must represent years (eg 2010).

A DC Generic Benefit Statement Quotation Type has been set up for all existing categories of non DB Irish schemes unless one already exists. This has now been expanded to cover all non db schemes. This is to hold the results of the calculations for a dc generic statement. It will automatically be set up when a new category is added in the future.

The DC Generic Benefit Statement Type has an additional field called Statement Period where you can identify the months to run the benefit statement for. The options are 1,3,6,9 and 12. It defaults to 12.

All other quote types have a new field DCGENST to tick to indicate if the dcgeneric quote is to be run before the quote selected (trigger quote). If the DCGENST field is ticked, it will automatically tick the SORPS field for irish schemes only as it is irrelevant to other scheme types.

Quotation Type Setup Screen

The SORPS quote will also be set up for all existing categories of non DB Irish schemes unless one already exists. It will automatically be set up when a new category is added in the future.
