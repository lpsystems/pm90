Introduction

Only users who belong to a User Group that has the correct permission can access this option.

If you do not have permission, you will not be permitted access to this option.

This program allows maintenance of Codes used within Members . These are codes that are used when entering additional details for a member.

The following options and screen appear when this program is selected:

  • Add
  • Query
  • Delete
  • Report
  • Exit

Codes Screen

Add Codes

Click ‘Add’.

This allows the user to enter the following details:

TYPE - Enter the type of the Code. This must be one of the following:

Code Description
G Accounting Code
A Additional Benefit Code
X Agent Network
L Asset Classes
U Award Description
F Award Type
Y Broker Company
E Career Average Revalued Earnings Code
C Contribution Transactions
D Deferred Pensioner Code
B Disability Benefit Code
M Investment Manager
H Location
P Pensioner Code
R Reason for Increase Code
K Risk Profile Code
S Special Code
I Transfer In Code
O Transfer Out Code

CODE - Enter the Code which must be relevant to the Type. The code can only contain alphanumeric characters. Some codes allow free format text, whereas others have validation applied to them using the following rules:

  • G - for Accounting Codes
    • No validation required
  • A - for Additional Benefit Codes
    • The first character must be either ‘A’,’S’ or ‘I’ (Add, Subtract or Ignore)
    • The second character must be either ‘F’ or ‘S’ (Fixed Benefit or Service)
    • Third character must be present
  • L - for Asset Classes
    • No validation required
  • F - for Award Types
    • Third character must be present
  • E - for Career Average Revalued Earnings Codes
    • The first character must be ‘C’ (Career Average Revalued Earnings)
    • The second character must be either ‘A’,’S’ or ‘I’ (Add, Subtract or Ignore)
    • Third character must be present
  • C - for Contribution Transactions
    • At least 3 character must be present
  • Q - for Country
    • Must be 2 characters long
  • D - for Deferred Pensioner Codes
    • The first character must be ‘D’ (Deferred Pension)
    • The second character must be either ‘A’,’S’ or ‘I’ (Add, Subtract or Ignore)
    • Third character must be present
  • B - for Disability Benefit Codes
    • The first character must be ‘B’ (Disability Benefit)
    • The second character must be either ‘A’,’S’ or ‘I’ (Add, Subtract or Ignore)
    • Third character must be present
  • T - for FATCA Status
    • No validation required
  • M - for Investment Managers
    • No validation required
  • H - for Locations
    • At least 3 character must be present
  • P - for Pensioner Codes
    • The first character must be ‘P’ (Pensioner)
    • The second character must be either ‘A’,’S’ or ‘I’ (Add, Subtract or Ignore)
    • Third character must be present
  • R - for Reason for Increase Codes
    • The first character must be ‘R’ (Reason for Increase)
    • Third character must be present
  • K - for Risk Profile Codes
    • No longer than 2 characters
  • S - for Special Codes
    • At least 3 character must be present
  • J - for Teams
    • No validation required
  • N - for Team Locations
    • No validation required
  • I - for Transfer In Codes
    • At least 3 character must be present
  • O - for Transfer Out Codes
    • The first character must be ‘T’ (Transfer Out)
    • The second character must be either ‘P’,’S’ or ‘C’ (Personal Pension, Section 32 Policy or New Company)
    • Third character must be present
  • X - for Agent Network
    • No validation required
  • Y - for Broker Company
    • No validation required

Apart from Country and Risk Profile codes, which have length restrictions on them, characters 3 to 8 are user defined. This will allow you to differentiate between Codes of the same type.

Depending on what you enter as the second character of the code, you will be presented with a different set of options when entering the actual details in Members e.g. you will not be permitted to enter an additional pension on the Transfer In screen if you have used a Transfer Code that has a second character of ‘S’.

DESCRIPTION - Enter a description for your Code. We would recommend that you use something meaningful here as this is what is displayed to users on screens and lookup lists.

When the Code details have been entered click on ‘OK’ to save the details to file. You can click on ‘Cancel’ at any stage to exit the add option without saving the entered Code details.

Query/Search For Codes

Click ‘Query’.

You may select records using the Code or Description fields.

Once you have entered your selection criteria, click on ‘OK’ to proceed and retrieve the records from the database or click on ‘Cancel’ to abort the selection and return to the main menu.

There is a lookup available which will display all Types and any existing Codes, just click on the lookup icon button.

If any match to the selection criteria is found then the first record will be displayed to the screen and a message displayed to the bottom of the screen.

Record 1 of 2

Viewing Codes

To view the other records selected press N to view the next record, and P to view the previous record. If N is chosen and there are no records after the current record an error message is displayed. If P is chosen and there are no previous records an error message is displayed.

Delete Codes

Click ‘Delete’.

In order to use the delete option , the code you want to delete must be displayed on the screen by previously using the Query menu option. You are then prompted to confirm that you want to delete this code and then prompted to re-confirm the deletion.

An error message is displayed if the code you are trying to delete is currently allocated to a scheme. This error message indicates the first scheme number that is currently allocated, so that the code can be deallocated by selecting the Delete a Code from the Scheme on the Code Allocation screen from the Schemes menu.

You cannot delete the following Reason for Increase Code:

  • RNEWPEN

This is because any Pension/Deferred Pension History lines that are added will use the ‘RNEWPEN’ as the default Reason for Increase Code.

You cannot delete the following Contribution Transaction codes:

  • BONUS
  • BUY
  • REBATE
  • SWIN
  • SWOUT
  • TVIN

This is because these codes are needed internally in the defined contribution functionality within Pension Master.

Printing Code Details

This option allows you to print a list of one or all of the types of Codes available on your system.

Click the ‘Report’ icon.

The following screen will appear when this program is selected:

Report Screen

Please select one of the following options:

  • All Codes - This will result in all codes being printed.
  • All Selected Codes - This will print any codes that have already been queried. If none have been queried, it will act in the same way as All Codes.

Exit Program

Click ‘Exit’. This leaves the program and closes the current browser tab.
