User Account and Valid Password

To use Pension Master, each person must have a valid user account with a username and password. These details help control which parts of the system the user can access. This allows the System Administrator to tailor access to the software based on the user’s role or level of access.

Security Level

Each user in Pension Master is given a security level between 1 and 99. These levels are set up in advance using the Security Levelsscreen, where you give each level a number and a description (for example, “Can access scheme 0000 but not scheme 0009”).

Each scheme and member record also has a security level. When a user tries to access a record, the system compares the user’s level with the level on the record. If the user’s level allows it, access is granted. This helps control access to sensitive information, such as director records in in-house schemes.

You can also use security levels to control access to specific screens within a member record on a scheme-by-scheme basis. For example, you might restrict access to a screen showing director salary details. This is managed using the Member Screen Selection screen in Schemes in the Schemes section.

User Groups

Every user in Pension Master must be assigned to a User Group. User Groups control which areas of the system the user can access.

What Are User Groups For?

User Groups let you limit access to certain areas of Pension Master. For example, you might want to stop some users from changing key codes or viewing audit trails.

Areas that can be restricted include:

  • Schemes
  • Companies
  • Categories
  • Status Codes
  • Brokers
  • Factors
  • Formulae
  • Codes
  • User Defined Messages
  • Security Levels
  • Annual Statutory Amounts
  • Section 148 Orders
  • CEP Factors
  • MLI/RPI Factors
  • Locks
  • Audit Trail Listing

The User Group menu option is in the System Administration menu of Pension Master. This option allows you to maintain the default User Groups as well as adding new groups of your own. You can keep the 6 defaulted groups or you can add your own and move the user across by updating their users details with the new user group. Alternatively you can amend the existing details of the defaulted user groups and allow or deny permission as you wish.

If a user belongs to a group that doesn’t have permission to perform a certain action they will see the following error message:

The User Group that you belong to doesn’t have permission to use this option. If you need permission ask your System Administrator giving them the following details: User ID, User Group, Product, Module and Description.

Update Member Records Status

If a user has a ‘No’ against ‘Update Member Records’ in Users they will not be allowed to add/update/delete member records in Members.

If it is not set to ‘No’, the user will have full access to member records—subject to any other security restrictions in place (such as security levels or user group permissions).

Restrict Access to Individual Schemes

It is possible to restrict access to particular schemes via use of the User Access Details screen in Schemes. There are three types of access level:

  • All users have access to the scheme.
  • All members of a particular workgroup have access to the scheme. See Workgroups.
  • User level access. Here you define particular users via their user ID (as set up in Users) so as to restrict access to a scheme on an individual user level.

If any user who does not have access to a scheme tried to select a record from that scheme, the only record header will be displayed e.g. in the member file, only the members name, scheme, company, category and status details will be displayed to the screen. Only schemes to which the user has access will be displayed in all lookup Scheme lists, available by clicking on the lookup icon.
