addben(code)

OVERVIEW

Use this function to extract a members additional benefit depending on the benefit code.

SYNTAX

addben(code)

EXPLANATION

  • code - is a reference to the Benefit Code of the Additional Benefit record that you wish to select the benefit from.

NOTES

  1. This function will return the Additional Benefit depending on the code entered. If the second character of the code is a “F” then the value of the Fixed Benefit for that code will be returned. If the second character of the code is a “S” then the value of the Added Years + Added Months / 12 + Added Days / 365 will be returned.
  2. If the initial letter of the code is A the result returned will be positive. If it is S the result will be negative and finally, if it is I the result will be set to 0.

EXAMPLES

addben(SSSTRIKE)
addben(AFEXEMP)

#contribution_history_field(fund_code,type,date,flag,cont_type)

OVERVIEW

Use this to refer to the value of a field on the Contribution History screen for the specified transaction type, date and contribution type code.

SYNTAX

#contribution_history_field(fund_code,type,date,flag,cont_type)

EXPLANATION

  • fund_code - this should always be ‘TOT’.
  • type - denotes the transaction type you wish to include e.g. BUY, TRANSFER, SWITCH. This defaults to BUY.
  • date - is the date at which the Contribution is effective. i.e. the value previous to this date will be returned.
  • flag - denotes which date on the Contribution History table to do the comparisons on as follows:
    • 1 - date due
    • 2 - date received
  • cont_type - denotes the contribution type code to be referred to.
  • contribution_history_field is any valid field on the Contribution History table. For a list of valid fields click here

NOTES

  1. contribution_history_field must be a valid field on the Contribution History File (pe_mcon).
  2. a fund code of ‘TOT’ is used to denote that you wish to get the total contribution for a particular date.
  3. type can take the value of either a valid transaction type date, or a previous line reference.
  4. date can take the value of either a hard-coded date in the format YYYYMMDD, a date field reference off the Basic Member Details table, a previous line reference, or TODAY which will return the value previous to today.
  5. flag must either be 1 or 2, or a previous line reference.
  6. cont_type must either be in UPPERCASE, no longer than 8 characters in length and exist in the Contribution Type Setup screen for the member’s scheme, or a previous line reference.

EXAMPLE

#pmc_amount(TOT,BUY,TODAY,1,EEORD)

This extracts the value held in the Contribution Amount field of the Contribution History record with the transaction type of BUY, date due of TODAY, and Contribution Type Code of EEORD for the member’s scheme.
