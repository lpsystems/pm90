Formulae

“YYYYMMDD”

OVERVIEW

Use “YYYYMMDD” to add a constant date to your formula.

EXPLANATION

  • YYYY is the year, MM is the month and DD is the day.

EXAMPLE

"19880512"

This example is May 12 1988.

‘Character String’

OVERVIEW

Use ‘Character String’ to add a string constant to your formula.

EXPLANATION

  • Use ‘Character String’ to specify that a formula line contains a string constant.

EXAMPLE

'Special'

+ - * / ^ > < ! =

OVERVIEW

Use these symbols to perform arithmetic operations in your formula.

EXPLANATION

  • + represents addition.
  • - represents subtraction.
  • * represents multiplication.
  • / represents division.
  • ^ represents exponentiation.
  • > represents greater than.
  • < represents less than.
  • ! represents doesn’t equal.
  • = represents equals.

NOTES

  1. Combining arithmetic operators is only allowed for => and <=.

EXAMPLES

@1 + @2 
@2 * #pm_rt1 
3 ^ 8

@line_number

OVERVIEW

Use the @ symbol to refer to the result of a previous line of your formula.

NOTES

  1. line_number must be >= 1 and cannot be greater than the current formula line number e.g. if you put “@55” in line 30 of the formula you will see an error message and be invited to re-enter the line number after the @ symbol.

EXAMPLE

@54

Example Formula

Line No. Contents
1. #pm_npa
2. @1 > 60 goto 5
3. avg(ps1_sal1,3) * $AAAAA
4. result @3
5. avg(ps1_sal1,4) * $AAAAA
6. result @5

The result of the formula is dependant on the member’s Normal Pension Age. If the NPA is <= 60, then the result is his/her average user defined salary 1 for the last 3 years, multiplied by the appropriate factor in the actuarial table AAAA. If the NPA is > 60, then the result is his/her average user defined salary 1 for the last 4 years, multiplied by the appropriate factor in table AAAA.

annuity(int_rate,date1,date2)

OVERVIEW

Use the annuity function to calculate the value of an annuity certain for a given interest rate and an outstanding term defined by two dates.

This may be used, for example when valuing future contributions paid into a money purchase type arrangement.

SYNTAX

annuity(int_rate,date1,date2)

EXPLANATION

  • int_rate - is the interest rate to calculate the annuity at.
  • date1 - is the start date for the annuity period.
  • date2 - is the end date for the annuity period

NOTES

  1. If date1 > date2 the value the annuity will be calculated with a term of 0
  2. The annuity is a continuous annuity which means that if it is being used to value future contributions in a defined contribution scheme, it assumes that contributions are payable at least monthly.
  3. The two dates must be field references to date fields on the member file (pe_mem) or constant dates entered in the format YYYYMMDD where YYYY is the year, MM is the month and DD is the day or references to the result of another line of formula code within the current formula e.g. @1.

EXAMPLES

annuity(8.5,19950112,pm_npd) 
annuity(7,pm_dps,pm_dtlft) 
annuity(10.6,19950112,20031115) 
annuity(9.5,pm_dps,@1)

fund_val(fund)

OVERVIEW

Use the fund_val function to calculate the total current fund value of a members pot based on unit bid prices and the total units allocated for all investment funds that the member invests in.

SYNTAX

fund_val(fund)

EXPLANATION

  • fund - is the fund code

NOTES

  1. To obtain the total fund value across all investment funds for the member, use a fund code of TOT.
  2. The unit price to use for the valuation is the latest (i.e. most recent in relation to the quotation date) one on the pe_prices table for the fund.
  3. The basis on which the units should be valued (i.e. Bid, Mid or Offer) is determined by reference to the pif_sell_basis field on the pe_invfun table for this fund.
  4. The total units to value can be found by summing the units across all contribution sources from the pe_minv table for the fund and member.

EXAMPLES

fund_val(EQUITY) 
fund_val(TOT) 
fund_val(MANAGED)

loop(line_ref,max_loop)

OVERVIEW

Use the loop function to allow the user to loop back to a previous line in the formulae.

SYNTAX

loop(line_ref,max_loop)

EXPLANATION

  • line_ref - to be entered in the format of @n, where n is the line number e.g. @1.
  • max_loop - this represents the maximum number of times the formula should go through the loop. This should be either a number or a reference to a previous line e.g. 20, @1.

NOTES

  1. If max_loop is entered in the format @n, then the value held on the referenced line must be numeric.

EXAMPLES

loop(@2,15) 
loop(@4,@2)

popup(msg_text,field_type)

OVERVIEW

Use the popup function to display a run-time pop-up window asking the user to input a value that is needed in the formula. In the Batch Quotation Run or the Pension Increase Run, the user has the option of using the same popup result for all members being processed.

SYNTAX

popup(msg_text,field_type)

EXPLANATION

  • msg_text - appropriate text to prompt user for input
  • field_type - this is a two character field that denotes the type of field you are asking for. Valid values are DT (date), TX (free format text), IN (Indicator - single character returned as uppercase), BO( Boolean, Y or N ), MO (monetary amount) or Nn (Number with n decimal places (n = 0-6)).

NOTES

  1. msg_text cannot be longer than 50 characters.
  2. field_type must be one of DT, TX, IN, BO, MO, N0, N1, N2, N3, N4, N5 or N6 and must be uppercase.
  3. If a field type is set as IN, then the value returned by the formula will be a text string, all in uppercase.

EXAMPLES

popup(Enter the amount of pre 1991 AVC's,MO) 
popup(Enter dependants age,N0) 
popup(Enter date started paying extra contributions,DT)

result

OVERVIEW

Use the function result to tell the formula to finish and display the result when the quotes have been run.

SYNTAX

result @line_number

NOTES

  1. The word result must appear in every formula. It means that the result of the formula is whatever follows the word result.
  2. You may use more than one occurrence of the word result in a formula, for example where you have some form of ‘if’ type branching dependant on sex.

EXAMPLES

result @45 
result @2 + 10

special(group,code)

OVERVIEW

If the member already has the Special Code allocated to them, then they will be placed in the Special Code Group.

NOTES

  1. The Special Code Group can be any 8 character code and is not validated.
  2. The Special Code must be a valid code and must be allocated to the member scheme.
  3. If the Special Code Group and Special Code have already been given to the member then nothing is done and no warning is produced.
  4. The function returns no value.

EXAMPLES

special(LUMPSUM,2TIMES)

smpi(date,age)

OVERVIEW

Extracts the SMPI factor for the appropriate tax year for the date specified and at the age specified.

SYNTAX

smpi(date,age)

EXPLANATION

  • date - refers to the SMPI tax date. If the date entered is not 06/04/yyyy, then the tax year, the date entered falls in, is worked out and used. For example, 12/10/2001 falls in tax year beginning 06/04/2001.
  • age - refers to the age last birthday at end of preceding tax year. Valid values are between 15 and 63.

NOTES

  1. SMPI factors are held on the SMPI screen for dates between 06/04/2000 and 05/04/2056.
  2. For each tax year the SMPI factors are held for ages between 15 and 63.

EXAMPLES

smpi(20030701,20) 
smpi(@2,@1)

curformat(line_ref,Cn)

OVERVIEW

Use the curformat function to format numerical values to be displayed with the thousand and decimal separators (where applicable). The result will be returned as a character string.

SYNTAX

curformat(line_ref,Cn)

EXPLANATION

  • line_ref - line_ref must be a reference to a previoius line in the formula e.g. @1.
  • Cn - where n denotes which numeric format is to be used.

NOTES

  1. n must be either 1 or 2.
  2. if n is 1, then the numeric value will be displayed as 99,999.99 for decimals, or 99,999 for integers.
  3. if n is 2, then the numeric value will be displayed as 99.999,99 for decimals, or 99.999 for integers.
  4. if the line reference is a number held in character format then it will be converted to a number.

EXAMPLES

curformat(@2,C1) 
curformat(@1,C2)

dateformat(line_ref,Dn)

OVERVIEW

Use the dateformat function to format date values to be displayed in either DD/MM/YYYY or MM/DD/YYYY format. The result will be returned as a character string.

SYNTAX

dateformat(line_ref,Dn)

EXPLANATION

  • line_ref - line_ref must be a reference to a previoius line in the formula e.g. @1.
  • Dn - where n denotes which date format is to be used.

NOTES

  1. n must be either 1 or 2.
  2. if n is 1, then the date value will be displayed as DD/MM/YYYY.
  3. if n is 2, then the date value will be displayed as MM/DD/YYYY.

EXAMPLES

dateformat(@2,D1) 
dateformat(@1,D2)
