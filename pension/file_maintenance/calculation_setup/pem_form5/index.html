Formulae

post_88_gmp_death(p,a,r,e,m,o)

OVERVIEW

Use the post_88_gmp_death function to calculate the Guaranteed Minimum Pension (GMP) accrued after 6 April 1988, revalued to date of death.

SYNTAX

post_88_gmp_death(p,a,r,e,m,o)

EXPLANATION

  • p - is a single character field that denotes the type of GMP revaluation as follows:
    • L - Limited rate revaluation
    • F - Fixed rate revaluation
    • S - Section 148 orders. These are assumed to be the same as for Fixed Rate revaluation.
  • a - is a single character field that takes the value ‘Y’ or ‘N’. It denotes whether to include any transferred-in GMP in the GMP calculations
  • r - is a single character field that takes the value ‘Y’ or ‘N’. It denotes whether to include an additional revaluation year in the GMP calculations.
  • e - is a single character field that takes the value ‘Y’ or ‘N’. It denotes whether to use equalisation when calculating female SPA.
  • m - is a single character field that takes the value ‘Y’ or ‘N’. It denotes whether to use equalisation when calculating male SPA.
  • o - is a single character field that takes the value ‘Y’ or ‘N’. When set to ‘Y’ it switches the calculation to use the opposite sex.

NOTES

  1. Date of death is taken as the current calculation date.
  2. If Date Left on the Member Basic Details screen is blank, it is assumed that you wish to calculate a death in service GMP. Otherwise it is assumed it will be a death in deferment GMP.
  3. If date of death is later than State Pension Date then the GMP is revalued up to State Pension Date using Fixed, Limited or Section 148 orders, and between State Pension Date and Normal Retirement Date at 1/7% per week, providing the period between State Pension Date and Normal Pension Date is at least seven weeks.

EXAMPLES

post_88_gmp_death(L,Y,N,Y,Y,N)
post_88_gmp_death(F,N,Y,Y,Y,N)
post_88_gmp_death(S,Y,Y,N,N,N)

post_88_gmp_exit(a,r,e,m)

OVERVIEW

Use the post_88_gmp_exit function to calculate the amount of the Guaranteed Minimum Pension (GMP) accrued after 6 April 1988 at termination of service.

SYNTAX

post_88_gmp_exit(a,r,e,m,o)

EXPLANATION

  • a - is a single character field that takes the value ‘Y’ or ‘N’. It denotes whether to include any transferred-in GMP in the GMP calculations
  • r - is a single character field that takes the value ‘Y’ or ‘N’. It denotes whether to include an additional revaluation year in the GMP calculations.
  • e - is a single character field that takes the value ‘Y’ or ‘N’. It denotes whether to use equalisation when calculating female SPA.
  • m - is a single character field that takes the value ‘Y’ or ‘N’. It denotes whether to use equalisation when calculating male SPA.
  • o - is a single character field that takes the value ‘Y’ or ‘N’. When set to ‘Y’ it switches the calculation to use the opposite sex.

NOTES

  1. If the members Date of Birth is blank, this function will return erroneous results and an error message will be displayed to the screen informing you of this if it happens for any member you are running quotes for.
  2. If the members State Pension Date is blank, this function will return erroneous results and an error message will be displayed to the screen informing you of this if it happens for any member you are running quotes for.
  3. The function calculates GMP to date using Date of Leaving, if present, otherwise the Calculation Date.
  4. If any required Section 148 orders are not on Pension Master, you will be warned of this and you should then add them using Section 148 Orders.

EXAMPLE

post_88_gmp_exit(Y,N,Y,Y,N)

post_88_gmp_npd(p,a,r,e,m,o)

OVERVIEW

Use the post_88_gmp_npd function to calculate the Guaranteed Minimum Pension (GMP) accrued after 6 April 1988, revalued to State Pension Age or to Normal Retirement Date, if later.

SYNTAX

post_88_gmp_npd(p,a,r,e,m,o)

EXPLANATION

  • p - is a single character field that denotes the type of GMP revaluation as follows:
    • L - Limited rate revaluation
    • F - Fixed rate revaluation
    • S - Section 148 orders. These are assumed to be the same as for Fixed Rate revaluation.
  • a - is a single character field that takes the value ‘Y’ or ‘N’. It denotes whether to include any transferred-in GMP in the GMP calculations
  • r - is a single character field that takes the value ‘Y’ or ‘N’. It denotes whether to include an additional revaluation year in the GMP calculations.
  • e - is a single character field that takes the value ‘Y’ or ‘N’. It denotes whether to use equalisation when calculating female SPA.
  • m - is a single character field that takes the value ‘Y’ or ‘N’. It denotes whether to use equalisation when calculating male SPA.
  • o - is a single character field that takes the value ‘Y’ or ‘N’. When set to ‘Y’ it switches the calculation to use the opposite sex.

NOTES

  1. If Normal Retirement Date is later than State Pension Date then the GMP is revalued up to State Pension Date using Fixed, Limited or Section 148 orders, and between State Pension Date and Normal Retirement Date at 1/7% per week, providing the period between State Pension Date and Normal Retirement Date is at least seven weeks.
  2. Full allowance is made for increases in the Post 88 GMP once past State Pension Date. This is done via references to the System Factor Table INFLA in Factors. If the appropriate factor cannot be found in the INFLA table, it will be assumed to be at the rate of 3% per annum and a message informing you of which tax years this has been assumed will be written to the Quotation Warnings file.

EXAMPLES

post_88_gmp_npd(L,Y,Y,N,Y,N)
post_88_gmp_npd(F,N,N,Y,Y,N)
post_88_gmp_npd(S,Y,N,Y,Y,N)

post_88_gmp_spd(p,a,r,e,m,o)

OVERVIEW

Use the post_88_gmp_spd function to calculate the Guaranteed Minimum Pension (GMP) accrued after 6 April 1988, revalued to State Pension Age.

SYNTAX

post_88_gmp_spd(p,a,r,e,m,o)

EXPLANATION

  • p - is a single character field that denotes the type of GMP revaluation as follows:
    • L - Limited rate revaluation
    • F - Fixed rate revaluation
    • S - Section 148 orders. These are assumed to be the same as for Fixed Rate revaluation.
  • a - is a single character field that takes the value ‘Y’ or ‘N’. It denotes whether to include any transferred-in GMP in the GMP calculations
  • r - is a single character field that takes the value ‘Y’ or ‘N’. It denotes whether to include an additional revaluation year in the GMP calculations.
  • e - is a single character field that takes the value ‘Y’ or ‘N’. It denotes whether to use equalisation when calculating female SPA.
  • m - is a single character field that takes the value ‘Y’ or ‘N’. It denotes whether to use equalisation when calculating male SPA.
  • o - is a single character field that takes the value ‘Y’ or ‘N’. When set to ‘Y’ it switches the calculation to use the opposite sex.

NOTES

  1. For females with a State Pension Date of 65, the legislation says that to calculate GMP’s you should assume a notional State Pension Date of 60 and then anything after age 60 is classified as a late retirement for GMP purposes i.e. the post 88 portion attracts 3% (or RPI if lower) increases each 5th April and the 1/7% per week late retirement increases also apply.
