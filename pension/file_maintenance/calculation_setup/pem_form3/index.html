Formulae

$table_code(date)

OVERVIEW

Use the $ symbol to refer to a particular actuarial factor table in your formula. This looks up the factor table at age nearest birthday. The argument is used to find the factor table that was effective at that date.

SYNTAX

$table_code(date)

EXPLANATION

  • date - is the effective date for the Factor Table. If no table exists at this date then the previous table will be used.

NOTES

  1. If you are unsure of the factor table required and you have already entered the $ part of the formula line, click the lookup icon for a list of the factor tables available.
  2. The member’s age and sex determine which factor is to be applied. A member’s age is calculated by subtracting his/her date of birth from the quotation date (which is entered by the user, and which is defaulted to today’s date).
  3. This table lookup interpolates between two factors if a member’s birthday does not fall exactly on the quotation date. e.g. $ER000 will extract a certain factor from table ER000, depending on a member’s age and sex.
  4. The date argument can be set to TODAY. This will then use the most recent actuarial factor.

EXAMPLES

$00001(TODAY)
$00001(20020101)

The first example refers to factor table 00001. The factor table used will be the one dated on or before today’s date.

The second example refers to factor table 00001. The factor table used will be the one dated on or before 01/01/2002.

$$table_code(date)

OVERVIEW

Use the $$ symbol to refer to a particular actuarial factor table in your formula. This looks up the factor table at age next birthday. The argument is used to find the factor table that was effective at that date.

EXPLANATION

  • date - is the effective date for the Factor Table. If no table exists at this date then the previous table will be used.

$$table_code(date)

:table_code(@n,date)

OVERVIEW

Use the : symbol to refer to a particular term related factor table in your formula. This looks up the factor table at nearest term. The second argument is used to find the factor table that was effective at that date.

SYNTAX

:table_code(@n,date)

EXPLANATION

  • @ - is the symbol “@”. This is used to denote the start of the line number you wish to refer to in the function argument.
  • n - is a valid formula line number which equates to the term you wish to look your term related factor at.
  • date - is the effective date. If no table exists at this date then the previous table will be used.

NOTES

  1. The member’s sex and relevant term determines which factor is to be applied.
  2. If you are unsure of the factor table required and you have already entered the : part of the formula line, click the lookup icon for a list of factor tables.
  3. This table lookup interpolates between two factors if a member’s term does not exactly match a full year.
  4. The “@” symbol is optional but we would recommend that you use it as it makes the formula more readable and immediately indicates to anyone looking at your formula that the number in the function argument refers to a formula line number and not a constant number.
  5. The date argument can be set to TODAY. This will then use the most recent term related factor.

EXAMPLES

:00001(@1,TODAY)
:00001(@1,20001231)

The first example refers to term related factor table 00001 which is to be looked up at a term equal to the result of line 1 in the formula. The factor table used will be the one dated on or before today’s date.

The second example refers to term related factor table 00001 which is to be looked up at a term equal to the result of line 1 in the formula. The factor table used will be the one dated on or before 31/12/2000.

factor(@x,term_act_flag,@y,date)

OVERVIEW

Use the factor function to refer to a particular term related or actuarial factor table in your formula. This looks up the factor table at nearest term or age nearest birthday. The fourth argument is used to find the factor table that was effective at that date.

SYNTAX

factor(@x,term_act_flag,@y,date)

EXPLANATION

  • @ - is the symbol “@”. This is used to denote the start of the line number you wish to refer to in the function argument.
  • x - is a valid formula line number which equates to the factor table you wish to use to look up the factor for.
  • term_act_flag - takes the value of either T or A, which denotes whether the factor you wish to look up is term related or actuarial.
  • @ - is the symbol “@”. This is used to denote the start of the line number you wish to refer to in the function argument.
  • y - is a valid formula line number which equates to the term or age you wish to look your term related or actuarial factor at.
  • date - is the effective date. If no table exists at this date then the previous table will be used.

NOTES

  1. The member’s sex and relevant term determines which factor is to be applied.
  2. This table lookup interpolates between two factors, if a member’s term/age does not exactly match a full year.
  3. The “@” symbol is optional but we would recommend that you use it as it makes the formula more readable and immediately indicates to anyone looking at your formula that the number in the function argument refers to a formula line number and not a constant number.
  4. The date argument can be set to TODAY. This will then use the most recent term related/actuarial factor.

EXAMPLES

factor(@1,T,@2,TODAY)
factor(@1,A,@2,20001231)

The first example refers to the term related factor table, that it referenced on line 1 in the formula, which is to be looked up at a term equal to the result of line 2 in the formula. The factor table used will be the one dated on or before today’s date.

The second example refers to the actuarial factor table, that is referenced on line 1 in the formula, which is to be looked up at an age equal to the result of line 1 in the formula. The factor table used will be the one dated on or before 31/12/2000.
