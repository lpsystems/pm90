Introduction

Only users who belong to a User Group that has the correct permission can access this option.

If you do not have permission, you will not be permitted access to this option.

This module allows you to define factor tables for use in your benefit calculation formulae. These are made up as follows:

  • Header information which holds details of the factor table code and a brief description of the factor table.
  • Sets of rates for various indices, one rate for each index.

The details can then be accessed during formulae calculation and can be used to store details of both actuarial (age and sex related) and term (period and sex) related factor tables. These could represent, for example, transfer value factors for a defined benefit scheme, or fund projection factors for a defined contribution scheme.

As you might expect, actuarial factors are indexed by age and sex and term related factor tables are indexed by term (in years) and sex.

The following options and screen appear when this program is selected:

  • Add
  • Query
  • Update
  • Delete
  • Report
  • Exit

Factors Screen

Add a Factor Table

Click ‘Add’.

FACTOR TABLE CODE - this is the code by which you can refer to this factor table in your formulae and must be entered. If the code already exists on the Factor Table File an error message will be displayed and you will be required to enter a unique factor code. Enter the code as appropriate. You can use up to 8 characters for your code. Spaces are not allowed in Factor Table Codes as these would be interpreted incorrectly in any formulae using these factors, and only uppercase is allowed.

You should note that where the factor table is not defined as unisex, you will be allowed to enter two separate factor tables under the same code, one for males and one for females.

DESCRIPTION - A description for your factor table must be entered. If a description is not entered an error message will be displayed and you will be required to enter a description for your factor table. The description will be used in the lookups, which are brought up when you click the lookup icon, in other parts of the system to identify each factor table, so we suggest you choose a meaningful description e.g. Immediate Annuities 9% Int, 0% Esc.

TYPE - Enter a factor table type for your factor table. You must enter a valid option here. The valid options are:

  • A - Actuarial
  • S - System
  • T - Term Related

If any other value is entered, an error message will be displayed and the table type should be re-entered.

SEX - Valid entries for this field are:

  • M - Male
  • F - Female
  • U - Unisex

If any other value is entered, an error message will be displayed and sex should be re-entered.

DATE - Enter a factor table date for your factor table. This should be entered either in the format “dd/mm/yy” or “dd/mm/yyyy”. This date will allow you to store historical factor tables. In formulae a factor code as of a certain date may be used.

RANGE FROM - Enter the lower range limit you wish your table to apply from e.g. if you wish to enter a table for ages 18-65, enter 18 here. This value must be entered and must be greater than 0. This value, combined with the RANGE TO field will be used to set up the indices in the lower portion of the screen so that you can input valid factors against each index.

RANGE TO - Enter the upper range limit you wish your table to apply to e.g. if you wish to enter a table for ages 18-65, enter 65 here. This value must be entered and must be no less that the value you entered in the RANGE FROM field. This value, combined with the RANGE FROM field will be used to set up the indices in the lower portion of the screen so that you can input valid factors against each index.

FACTOR - Enter the value of the factor for each index. The factors will initially all default to 0 and can be typed over with the appropriate values.

When the Factor Table details have been entered click ‘OK’ to save Factor Table details. You can click ‘Cancel’ at any stage to exit the add option without saving the entered Factor table details.

If, at this stage, any of your factors are still equal to 0, an error message is displayed warning you of this and then the following message appears at the top of the screen:

Are You Sure You Want To Save The Factor Table (Y/N)?

Enter Y if you wish to continue and save the factor table details or N if you wish to continue editing the factors prior to saving them.

Factor Tables can also be added or update via the General Data Load program and details of the templates available can be found there.

Query/Search

For A Factor Table

Click ‘Query’. The following pop-up window is displayed:

Factor Table Selection Screen

You may select records using the Table Code, Type or Sex fields. You may also use any combination (or even partial combination) of these fields in order to select the required factor table record(s).

If you are unsure of the factor table code to query, click the lookup icon to see a list of the factor tables. The Table Type and Sex fields will automatically be filled in based on the Factor Table Code chosen.

Once you have entered your selection criteria, click ‘OK’ to proceed and retrieve the records from the database or click ‘Cancel’ to abort the selection and return to the Factors main menu.

If any match to the selection criteria is found then the first record will be displayed to the screen and a message displayed to the bottom of the screen.

Record 1 of 2

Viewing Factor Tables

To view the other records selected press N to view the next record, and P to view the previous record. If N is chosen and there are no records after the current record an error message is displayed. If P is chosen and there are no previous records an error message is displayed.

Updating Factor Tables

Only users set up as system administrator status in Users may update system factor tables.

If you do not have system administrator status, you will not be permitted access to this option for system factor tables.

Before Updating, a record must be selected through the Query option, and an error message is displayed if a record for update has not been selected.

Click ‘Update’.

The Factor table code cannot be changed as this is one of the fields which is a key to the factor table file.

Changing the date at this point, will result in a new factor table being created for this date. Please use the delete option if you do not require the amended factor table.

If you wish to add or delete factors for a certain range, then amend the RANGE FROM and/or RANGE TO fields to reflect the new factor table range. The factors themselves will be added to or deleted as appropriate. If new factors are added in this way, they will initially default to 0 and should be typed over with their new values.

When changes have been made to the record click ‘OK’ to save the changes to file, or click ‘Cancel’ to exit the update option.

Deleting Factor Tables

A record must be selected through the Query option before a Factor can be deleted.

Click ‘Delete’.

When the delete option is selected the following window is displayed:

Delete Factors Screen

Select one of the following options:

  • Delete all Factor Tables for this code
  • Delete all Factor Tables for this code and type
  • Delete all Factor Tables for this code, type and sex
  • Delete current Factor Table

You will then be prompted to confirm the deletion of the Factor Table.

Printing Factor Tables

Click the ‘Report’ icon.

The following window now appears:

Select Report Options Window

  • Current screen - Prints the current screen record.
  • List of factor tables - Prints a list of all factor tables that meet the selection criteria.

The options for printing are described in the document Printing From Pension Master.

Exit Program

Click ‘Exit’. This leaves the program and closes the current browser tab.
