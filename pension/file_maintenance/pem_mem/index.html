Introduction

This program allows maintenance of the Member file. This is the core of the Pension Master system and is where you will find all the data relating to a member of the scheme. The data is accessed via a series of screens, each grouping together similar parts of the member’s data. To navigate between the screens in the Member File, use the ‘Menu’ as described in Navigating Between Screens.

Some of the main screens of information that you may find in the Member File are as follows:

Basic Member Details Member Indicators
Service Details Part Time Service Details
Dependants Member Notes
Status History Salary History
Contribution Rate Percentages Contribution History
Investment History Investment Switches
Investment Value Summary NI History
Additional Benefits Transfer In Details
Transfer Out Details Refund Of Contributions
Deferred Pensioner Pensioner
Death Details Quotation Screens
Member Address Internet Security Details

The available screens for a member are dependent on the type of scheme the member belongs to.

If your System Administrator has set the View Only rating to a value higher than your own security rating for the above screens, via the Member Screen Selection screen in Schemes, then ‘VIEW ONLY’ will be displayed on the bottom line of the respective screen(s), and you will not be allowed to use the Add, Update and/or Delete menu option(s).

The following options and screen appear when this program is selected.

  • Add
  • Query
  • Report
  • Letter/Cert
  • Exit

Basic Member Details Screen

Adding A Member

Basic Member Details

Click ‘Add’.

The Basic Member Details screen is used to hold the basic details about the member of the scheme.

Enter the fields as required as follows:

Member Number - The member number is automatically generated and displayed to the screen. The member number will be displayed as 0 until you have finished the input of the Basic Member Details screen and will then be displayed to the screen. You will not be permitted to amend this field.

Staff Number - This field may be useful in linking the staff number with member records for data transfer or reports. Typically you would enter the employers payroll number here. This field is optional and can be left blank if you so wish.

NI/PPS/TRN Number - This is where you store the members personal security number. As you enter this field, Pension Master will check all other member records on the database for duplicate numbers and if any are found, you will be asked to confirm that you wish to continue or stop adding a new member. There is also validation against the format of the number.

Title - The members title (e.g. Mrs, Mr, Dr ). This field is optional, but can be used in mail merges etc. and is used by Pension Master to default a new members sex as appropriate.

Forename & Surname - A forename and surname for the member must be entered. The forename field is used to automatically generate the member initials field on the Member Address screen.

Scheme - This represents the code corresponding to the scheme the member belongs to. The scheme code must be entered and the code entered must exist on the scheme file. To see a list of scheme codes click the lookup icon. When a valid scheme code has been entered the scheme name will be displayed.

Company - The company code must be entered and the combination of scheme and company code must exist on the company file. To see a list of the existing combinations of scheme and company codes on the company file click the lookup icon. When a valid company code has been entered the company description will be displayed.

Category - The category code must be entered and the combination of scheme and category code must exist on the category file. To see a list of the existing combinations of scheme and category codes on the category file click the lookup icon. When a valid combination of scheme and category code has been entered the category description will be displayed.

Status - This represents the current status of the member in the scheme, e.g. active member, deferred pension The status code entered must exist on the status file. To see a list of the status codes click the lookup icon. When a valid status code has been entered the status description will be entered.

Sex - Select one of the following options from the drop-down list:

  • Male
  • Female

Marital Status - Select one of the following options from the drop-down list:

  • Civil Partner
  • Cohabitant
  • Divorced
  • Married
  • Partner
  • Separated
  • Single
  • Single With Dependants
  • Unknown
  • Widowed
  • Judicially Separated
  • Former Civil Partner
  • Surviving Civil Partner
  • Nominated Partner
  • Divorced with Dependants

Date of Birth - This represents the members date of birth. If you enter a date for which the members current age is less than 16 or over 75, a message will be displayed informing you of this and asking if this is really correct.

Admitted - Select ‘Yes’ to confirm that the member’s date of birth has been confirmed with a birth certificate or other evidence of age.

Date Joined Company - This denotes Date Joined Company. If you are going to use the Inland Revenue Maxima (UK only) within the formulae you set up for this scheme, a valid date must be entered in this field. You will not be permitted to enter any date for date entered company which means that the member would have commenced employment at the current employer before his/her 14th birthday.

Date Joined For LA - Date Joined For Life Assurance Benefits.

Date Joined For Pension - Date Joined For Pension Benefits.

Pension Service Date - Date Pensionable Service Commenced.

Date Contracted Out - Date Contracted Out. If the scheme is not a contracted out scheme or the scheme has been set up as an Irish Scheme in Schemes, this field description will change to Qualifying Serv Date.

SORP Issued Date - Date the SORP was Issued.

Auto Enrolled UK Only Schemes

Adding a New Member

  1. If the AE (auto enrolled) field at scheme level is ticked then the value of the member AE field will default to Y unless the members date of leaving is before the staging date where it will be set to N.
  2. On saving the new member, if they are under age 22 today, a popup message will appear advising that the member is under 22 and asking if the value of the AE field should be Y. If the user clicks ‘Yes’ it will save it as Yes, if the user clicks ‘No’ it will save it as No.

Updating a Member

  1. The field can be updated to Y or N with no verification.

Age Last Birthday - This shows the members age as at today’s date and is automatically calculated as long as you have already supplied the member’s date of birth. This field is a view only field and is re-calculated each time you view a member’s record. The age will be calculated as age last birthday or age next birthday according to how your System Administrator has set the appropriate flag in the System Parameters program.

If the Member is of a Death Status Type then the label will change to Age at Death. If there is no Date of Death on the Death Details screen then the Status Effective Date, when the Member changed to a Death Status Type, will be used instead.

Normal Pension Age - Normal Pension Age. Enter a two digit number which is the age that the member will retire at. Usual values are any age between 50 and 75. If you do not enter anything in this field, it will default to a pension age of 65 for both males and females.

Normal Pension Date - Normal Retirement Date. This is a date field which will automatically be calculated if the members date of birth and Normal Retirement Age are both entered, otherwise a date can be entered in the format ddmmyy or dd/mm/yyyy.

Target Pension Date - If you intend using a term related lifestyle matrix for this member, enter a target retirement date here. This will default to the members Normal Pension Date.

State Pension Date - State Pension Date. This is a date field which will automatically be calculated if the members date of birth is entered. As State Pension Dates are laid down by Statute, you are not permitted to amend this field.

Date Left - This date is entered when the member leaves the scheme. This should be entered as and when you receive notification that the member has left the scheme.

This date is also used to set the ‘Date Of Exit From Active Status’ held on the Refund of Contributions Details screen.

Security Rating - Select the security rating you wish to allocate for this member from the drop-down list. This is used in conjunction with the Security Rating in the Users to determine whether the current user has access to view or update the current member record. The default security ratings are:

  • Default High Level
  • Default Medium Level
  • Default Low Level

AVC Indicator - The field can be updated to Y or N if the member has AVC type contributions. This field gets update automatically if a AVC type of contributions is added to the contribution screen.

PAO Indicator - The field can be updated to Y or N if the member has a Pension Adjustment Order held against them.

Outgoing Worker - The field can be updated to Y or N the default is ‘N’. This is a member who joins the company on or after 13 Sep 2019 (or prior to that date if in a waiting period to join the company’s scheme) whose most recent employment was in another EU member state and who was an active member of a pension scheme relating to that employment.

Last Amended - This field is used to denote the date of the last amendment of this member record. This field is a view only field and is updated automatically every time this member is updated by a user. Detailed details of the changes made each time are automatically logged to the system Audit Trail which can periodically be printed off by a suitably authorised user.

By - The User ID of the user who last amended the current member.

You will now be returned to the main menu. If there are any other details you wish to add to the member, you should go directly to the appropriate screen and add them now.

Query/Search For A Member

Click ‘Query’. The following pop-up window is displayed:

Query Options Screen

You may select records by using the Member Number, Member Forename, Member Surname, Scheme Code, Company Code, Category Code, Status Code, Status Type, Employer Number, Marital Status, Sex, Date of Birth, NI/PPS/SSNI Number, Special Code or Alternative Scheme Code fields. You may also use any combination, or even partial combination, of these fields in order to select the required member(s). See Selection Criteria.

Once you have entered your selection criteria, click ‘OK’ to proceed and retrieve the records from the database or click on ‘Cancel’ to abort the selection and return to the Members main menu.

If more than one record is found to match your search criteria, the following screen is displayed:

Multiple Member Selection Screen

Use the arrow keys to locate the member record you require and press Return to select it.

If only one record is found to match your search criteria, the member record is displayed directly to the screen.

If you wish to view any other page of member details other than the current screen, click ‘Menu’, then select the appropriate screen by moving the highlight bar. See later in this chapter for more on the ‘Menu’ option in Members.

Quick Search

Click ‘Quick Search’. This automatically re-opens the ‘Multiple Member Selection Screen’ as detailed in the Query/Search For A Member section above, enabling you to jump directly to another member record in the currently selected set of member records.

This saves you having to use the Query option to re-select the current set of member records.

Browsing Member Details

Click ‘Browse’ to view details in any screen where the data does not fit on the screen e.g. salary history may have more entries than can fit on the screen, so you could use the ‘Browse’ option to view those salary details not immediately visible.

Once in browse mode, use the arrow keys or the scrollbar on the right of the data to navigate around the data.

Click ‘OK’ or ‘Cancel’ to leave browse mode.

Viewing Member Details

To view the other records selected press N (Next) to view the next record, and P (Previous) to view the previous record. If N is chosen and there are no records after the current record an error message is displayed. If P is chosen and there are no previous records an error message is displayed.

You can also use the options on the toolbar to view the first, previous, next and last records in your query.

Before Updating, a record must be selected through the Query option, and an error message is displayed if a record for update has not been selected.

If you wish to send an ad-hoc letter to the currently selected member, you can easily do it using the ‘Quick Letter’ facility. All that is required is that you type the body of the letter, the rest is done automatically by Pension Master.

If the member’s address is flagged as suspect, then you will be asked if you wish to use the suspect address.
