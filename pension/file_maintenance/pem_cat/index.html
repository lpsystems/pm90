Introduction

Only users set up as Supervisor status in Users may access this program.

The type of access granted can be view only or full access. For the view only option, you will not be permitted access to any menu option that allows you to add/update/delete data.

If you do not have Supervisor status, you will not be permitted access to this program.

This program allows maintenance of the Category file. This allows you to:

  • set up the different benefit categories for your scheme.
  • set entry and exit criteria for members of this category. These are used to validate entry of new members to this category.
  • allow for any interest on contributions returnable on early leaving or death.
  • set up the Quotation Types you wish to use for this category.
  • attach formulae to these Quotation Types in order to value the various benefits the scheme provides.
  • define how results of finalised quotations are to be written back to the member record

The following options and screen appear when this program is selected.

  • Add
  • Query
  • Update
  • Delete
  • Report
  • Menu
  • Exit

Basic Category Details Screen

Add A New Category

Basic Category Details

This screen allows you to set the criteria by which you wish to validate entries and exits from this category.

Click ‘Add’.

You will be asked if you wish to copy an existing Category. If you select ‘Yes’ you will be prompted to select the category code of the category you wish to copy or select one from the lookup list.

Once the Scheme and Category codes have been entered you will be able to select which screens you wish to copy from the list. When this has been done, the window closes and you will be returned to the category code field for your new category.

SCHEME CODE - The scheme must be entered and the code entered must exist on the scheme file. When a valid scheme code has been entered the scheme description will be displayed. You can click the lookup icon to see a list of scheme codes.

CATEGORY CODE - The category Code must be entered and the combination of scheme and category code cannot already exist on the category file.

There are two columns of details headed ‘FULL BENEFIT’ and ‘DIS BENEFIT’ respectively. The former is used to validate the ‘Date Joined For Pension’ field and the latter is used to validate the ‘Date Joined For Life Assurance’ field on the Basic Member Details screen in Members when adding or updating a member record.

MID YEAR ENTRANTS ALLOWED - This is used to determine whether members of this category will only be allowed to join the scheme on the anniversary of the scheme renewal date (by reference to the renewal day and month on the Basic Scheme Details screen in Schemes) or at any date during the scheme year. It will default to ‘No’ for all categories.

MINIMUM AGE FOR JOINING - This is the minimum age an employee needs to be in order to be eligible to join the current category. This will be used to validate entry of new members to this category in Members. The default value is 18 (i.e. 18 years of age).

MINIMUM SERVICE FOR JOINING - This is the minimum period of company service in years (by reference to the Date Entered Company, pm_dec, field on the member record) an employee needs to be in order to be eligible to join the current category. This will be used to validate entry of new members to this category in Members. The default value is 0 (i.e. the member can join the category any time after they have joined the Company as an employee).

MAXIMUM AGE FOR JOINING - This is the maximum age an employee needs to be in order to be eligible to join the current category. This will be used to validate entry of new members to this category in Members. This field can not be greater than the higher of the normal pension date for males or that for females.

NORMAL PENSION AGE (MALES) - This is the normal pension age applicable to male members of this category. This will be used as a default in the entry of a new member record and will be used to automatically update the members Normal Pension Date (pm_npd). The default value is 65.

NORMAL PENSION AGE (FEMALES) - This is the normal pension age applicable to female members of this category. This will be used as a default in the entry of a new member record and will be used to automatically update the members Normal Pension Date (pm_npd). The default value is 65.

NORMAL PENSION DATE ADJUSTMENT - This is used to denote whether members of this category usually retire on the actual day of them reaching Normal Pension Age or whether a adjustment is to be made to this date. The options available are:

  • No Adjustment - Members Normal Pension Date is the actual day the member reaches Normal Pension Age i.e. no adjustment.
  • Start of Month - Members Normal Pension Date is the first day of the month in which the member reaches Normal Pension Age.
  • End of Month - Members Normal Pension Date is the last day of the month in which the member reaches Normal Pension Age.
  • First Day of Following Month - Members Normal Pension Date is the first day of the month following the member reaching Normal Pension Age.
  • Last Day of Previous Month - Members Normal Pension Date is the last day of the month prior to the member reaching Normal Pension Age.
  • Fixed Day of Month - Members Normal Pension Date is a fixed day of the month in which the member reaches Normal Pension Age. In this case the user will be required to enter a number between 1 and 31 as the day in the month the NPD is to be set. If the actual month the member is due to retire in does not have that number of days i.e. Days = 30, Month = February, the members Normal Pension Date will be calculated as the maximum number of days allowed in that month i.e. 28 or 29.
  • Last Working Day of Month - Members Normal Pension Date is the last working day of the month in which the member reaches Normal Pension Age.
  • Anniversary of joining the Scheme after the members 65th Birthday - Members Normal Pension Date is day and month of the members date of joining scheme following the member reaching Normal Pension Age.

Click ‘OK’ to save the details, or click ‘Cancel’ to exit without saving the changes.

You will now be returned to the main menu. If there are any other details you wish to add to the category, you should go directly to the appropriate screen and add them now.

Contribution Interest Details

Select the Contribution Interest Details screen using the menu.

The following screen is displayed:

Contribution Interest Details Screen

This screen allows you to record details of the rate of interest to be applied to a contribution type in the case of a defined benefit type scheme where interest is added to the total contributions on an regular basis.

The fields are described as follows:

FORMULA CODE - Enter a valid Formula Code or select a code from the lookup list. If a Formula Code is entered with an Interest Rate on the same line then an error message is displayed indicating that one of them must be removed.

The Formula must have previously been entered into the formula file using Formulae. It should be checked to ensure that the formula correctly calculates an INTEREST RATE ONLY. If the formula generates an inaccurate value, then the Exception Report will ONLY detect any interest rates that are greater than 100% or less than -100%.

The calculation used is as follows:

New Interest Value = ( Contribution Amount + Contribution Interest ) * ( Formula Result / 100 )

INTEREST - Enter a Percentage Rate. If a Percentage Rate is entered with a Formula Code on the same line then an error message is displayed indicating that one of them must be removed.

If interest is added to one or more contribution types, insert the interest rate against the appropriate contribution type. These are numeric fields and the default values are 0. These fields are used by the Contribution Interest Run.

The calculation used is as follows:

New Interest Value = ( Contribution Amount + Contribution Interest ) * ( Interest Rate / 100 )

PRO RATA - Tick this box if a Pro Rata increase should be calculated. This will determine the fraction of the Interest Rate increase based on the members last Interest Run date and the current run date used in the Contribution Interest Run.

If the partial year includes the 29th February, obviously of a leap year, then the number of days in the calculation below will be 366. Otherwise the number of days will be 365.

The calculation used for a Formula is as follows:

New Interest Value = ( Contribution Amount + Contribution Interest ) * ( ( Formula Result * ( Period in Days / 365 ) ) / 100 ) + Contribution Interest

The calculation used for a Fixed % Rate is as follows:

New Interest Value = ( Contribution Amount + Contribution Interest ) * ( ( Interest Rate * ( Period in Days / 365 ) ) / 100 ) + Contribution Interest

Click ‘OK’ to save the details, or click ‘Cancel’ to exit without saving the changes.

Contribution Rate Defaults

Select the Contribution Rate Defaults screen using the menu.

The following screen is displayed:

Contribution Rate Defaults Screen

This screen allows you to enter the specific contribution rate percentages for each applicable contribution type for members of this category. If you are doing a Contribution Request Run and the member in question does not have any entries on their Contribution Rate Percentages screen, the equivalent values at Category level are used in the calculations.

These are then used by the Contribution Request Run to generate batches of contribution requests for the scheme.

Click ‘Update’.

Enter the fields as follows:

CONTRIBUTION FREQUENCY - Enter the frequency of your contributions for members of this category. The valid options are:

  • 1 - Annual
  • 2 - Half yearly
  • 12 - Monthly
  • 4 - Quarterly
  • 52 - Weekly

CONT TYPE - Enter the contribution type code for this line.

TYPE - Enter the code for the type of calculation of contributions you require for this contribution type. The valid options are:

  • PERCENT - percentage of salary
  • FLATRATE - flat rate
  • EEPR - employee’s protected rights
  • ERPR - employer’s protected rights

Note that types EEPR and ERPR are only calculated for members who are on NI Table F i.e. an entry of ‘F’ is in the NI Table field on the Member Indicators screen in the members record.

ANNL RATE - If PERCENT type code is used, enter the percentage rate to use.

SALARY TO USE - If PERCENT type code is used, enter the salary to use in the calculation.

FORMULA - Enter a formula code to use to calculate the contributions for this contribution type. The formula must be a valid formula for this scheme. If a valid formula code is entered, all the other parameters for the calculation of contributions for this contribution type are ignored.

Using a formula to calculate contributions is especially useful if contributions are calculated in a more complicated way than the basic options here permit e.g. you might have an age related percentage of salary contribution rate. In this case, you could use Formulae in conjunction with an age related Factors to calculate the contributions.

Note that the numeric result of your formula is the value that is saved as the members contribution amount for that contribution type.

Click ‘OK’ to save the details, or click ‘Cancel’ to exit without saving the changes.

Update A Quotation Type

Select the relevant Quotation Type screen using the menu, then click ‘Update’.

This screen allows you to set up groups of calculations you wish to have for members of this category. This allows you to completely customise the way the Quotation Screens look in Members.

As part of the Pension Master quotations engine, the user can either write the results of finalised quotations back to the member record, or they can just simply finalise the results without writing back to a screen. Writing the results back to a screen can be done by two methods:

  1. By setting up a User Defined Screen to hold the details you wish to write back from your quotations. e.g. you might set up a User Defined Screen called ‘REFUND’ to hold details of the results of refund of contributions calculations that have been finalised.
  2. By directly sending the finalised quotations through to a screen held in Member File. The screens set up for this facility are:
Screen Name
Death Details
Deferred Pensioner Details (basic and history details)
Lifetime/Annual Allowance
Pensioner Details (basic and history details)
Protected A-Day Benefits
Refund Of Contributions Details
Transfer In Details
Transfer Out Details

In this Quotation Type screen, you set up each field that you wish to hold data for. The fields must be set up as either decimal, integer, character or date, so they can correctly display your calculation results. You then use the WRITE, NUM and SUB CODE fields on the second screen of Categories to indicate how these finalised results are to be written back to the member record.

Finalising the results without writing back to a screen can be done by choosing the ‘Finalise’ option in the ‘Write Results To’ field.

The following screen is displayed:

Quotation Types Screen

QUOTATION TYPE CODE - Enter a code for your Quotation Type. This is a 6 character field and must be unique for the current category and scheme combination e.g. you cannot have two Quotation Types of NORM for scheme 0000 category 000. If you do not enter a unique code in this way, an error message is displayed and you will be requested to re-enter your Quotation Type Code.

QUOTATION TYPE DESCRIPTION - Enter a name for your Quotation Type. This will be used in subsequent lookups and menus in other parts of Pension Master so it is important that you use a meaningful name.

WRITE RESULTS TO - Enter the name of the screen you wish to write results of this quotation type back to. Valid options are:

  • Death
  • Deferred Pensioner
  • Lifetime/Annual Allowance
  • Pensioner
  • Protected A-Day Benefits
  • Refund
  • Transfer In
  • Transfer Out
  • User Defined Screens
  • Finalise

FORMULAE DESCRIPTIONS/CODES - Enter the formulae codes and descriptions corresponding to the various fields. To get a list of existing formula codes for the current scheme click on the lookup icon whilst in any of the codes fields. The formula descriptions will default to the description you gave for the formula corresponding to the formula code you enter on the screen and can be typed over with any formula description you require in order to allow you to completely customise the quotation screens on a category by category basis.

When all the details have been entered click ‘OK’ to save the details to file. You can click ‘Cancel’ to exit the option without saving the details.

If you have entered a valid screen in the Write Result To field, click ‘Screen 2’ to go to the second screen in Quotation Type setup. This is where you match up which quotation result is written back to which field on the specified screen.

Writeback Setup Screen

The screen may vary slightly dependant on which Write Results To screen you have chosen. If you have chosen to ‘Finalise’ the results, then you won’t be allowed to update this screen as the results are not being sent anywhere.

User Defined Screens

You can writeback results to more than one user defined screen. Simply specify the different user defined screen descriptions in the User Defined Screen column of this screen.

USER DEFINED SCREEN - Enter the name of user defined screen you wish to write results back to. Click on the lookup icon to see a list of user defined screens.

NUM - This field is automatically filled in with the corresponding line number (from screen 1) for the calculation description that the user has entered. The user does not have access to this field.

CALCULATION - Enter the description of the calculation (from screen 1 of Quotation Type setup) that you wish to write back to the specified user defined screen. There is a lookup list of the calculations available by clicking on the lookup icon. After entering a valid description, the ‘Num’ field is automatically filled in with corresponding line number (from screen 1).

FIELD - Enter the position on the specified user defined screen to write this calculation back to. This is a valid number from 1 to 20.

Death, Lifetime/Annual Allowance, Protected A-Day Benefits, Refund, Transfer In & Transfer Out Screens

FIELD - This is automatically populated with a list of all available fields on the specified writeback screen.

Fields with a * against them are displayed towards the top of the list and are mandatory fields i.e. fields that you must write back some data to. All other fields are optional and you are not required to assign a calculation to them.

NUM - This field is automatically filled in with the corresponding line number (from screen 1) for the calculation description that the user has entered. The user does not have access to this field.

CALCULATION - Enter the description of the calculation (from screen 1 of Quotation Type setup) that you wish to write back to the specified screen. There is a lookup list of the calculations available by clicking on the lookup icon. After entering a valid description, the ‘Num’ field is automatically filled in with corresponding line number (from screen 1).

For transfer in / transfer out screen writebacks, you will be asked for the transfer in / transfer out code when you actually perform the writeback.

Deferred Pensioner/Pensioner Screens

You can writeback results to the deferred pensioner / pensioner basic details screen and the deferred pensioner / pensioner history screen. When writing back deferred pensioner / pensioner history, you can writeback to more than one deferred pensioner / pensioner code. Simply specify the different deferred pensioner / pensioner codes in the Sub Code column of this screen.

FIELD - Enter the description of the field you wish to writeback to. There is a lookup list of the fields available by clicking on the lookup icon.

NUM - This field is automatically filled in with the corresponding line number (from screen 1) for the calculation description that the user has entered. The user does not have access to this field.

CALCULATION - Enter the description of the calculation (from screen 1 of Quotation Type setup) that you wish to write back to the specified screen. There is a lookup list of the calculations available by clicking on the lookup icon. After entering a valid description, the ‘Num’ field is automatically filled in with corresponding line number (from screen 1).

SUB CODE - You only need to enter this if the field you are writing back to is on either the deferred pensioner history or pensioner history tables.

Enter a valid deferred pensioner / pensioner code for this calculation to be written back to. As a mimimum, each deferred pensioner / pensioner code that you specify must have at least a Date To Commence written back to the screen. All other fields are optional.

Note that if you are writing back any results to the deferred pensioner / pensioner basic details, you must writeback the Pay Frequency to that table as this is mandatory when performing a deferred pension / pension increase run.

When all the details have been entered click ‘OK’ to save the details to file. You can click ‘Cancel’ to exit the option without saving the details.

Deleting Categories

Before Deleting, a record must be selected through the Query option, and an error message is displayed if a record for deletion has not been selected.

You will not be allowed to delete categories that contain some member records and if you try to do so, a warning message will be displayed to the screen and you will be returned to the main menu.

Click ‘Delete’.

When the delete option is selected a window will appear with the options ‘Yes’ and ‘No’.

If you do not wish to delete the record, click ‘No’. You will return to the program menu.

If you wish to delete the record, click ‘Yes’.

Another window will appear asking if you are sure.

If you do not wish to delete the record, click ‘No’. You will return to the program menu.

If you wish to delete the record click ‘Yes’. The record is deleted and the program returns to the main menu.
