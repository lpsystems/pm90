Introduction

Only users set up as Supervisor status in Users may access this program.

The type of access granted can be view only or full access. For the view only option, you will not be permitted access to any menu option that allows you to add/update/delete data.

If you do not have Supervisor status, you will not be permitted access to this program.

This allows you to define status codes which can then be used to assign to individual member records on Pension Master.

A status code is a way of telling Pension Master what kind of member of a pension scheme a member record is e.g. Active, Active Paying AVCs, Deferred Pensioner or Pensioner.

This allows you to:

  • Define as many status codes as you require to suit your individual pension scheme needs.
  • Assign a Status Type to each Status Code so as to indicate whether the current Status Code is Active, Deferred, Leaver, Death, Pensioner or Widower. This can then be used in reports and other parts of Pension Master, for example, to run a report only for active members of the scheme.

Pension Master can be delivered with a standard set of status codes pre-installed. You can customise these as you wish. We would, however recommend the following conventions:

  • 0-9 - Active Members
  • 10-19 - Deferred Pensioners
  • 20-29 - Pensioners
  • 30-39 - Deaths
  • 40-49 - Leavers
  • >49 - Any other ad-hoc status codes

Status File Screen

Adding Status Details

Click ‘Add’.

STATUS CODE - The status code entered cannot already exist on the status file. If the code entered already exists on the status file an error message is displayed and a status code that does not exist on the file should be re-entered.

STATUS DESCRIPTION - The status description must be entered. If the status description is not entered an error message will be displayed, and a status description should be entered.

STATUS TYPE - The status type must be entered. Select the Status Type from the drop-down list:

  • Active
  • Death
  • Deferred
  • Leaver
  • Pensioner
  • Pensioner - No Liability
  • Risk Only
  • Widower

LIFE ASSURANCE ONLY - This field will default to ‘No’ for all status codes, but can be set to ‘Yes’ for those status codes that are for life assurance only type benefits. This will then be used within the Pensions Board Fee Invoice Report to determine which active members are pension members (and not just life assurance only members) and hence should be included in the Pensions Board Fee Invoice calculations. It is also used on the Underwriting/Life Assurance Report to determine which members to total up salary roll for when working out the total funding amount and the total employee contributions due.

If the status type entered is not one of these, an error message is displayed to the screen and you will be requested to re-enter a valid status type. When a valid status type is entered, the description of the type is automatically displayed to the screen, e.g. if you entered ‘A’ as the status type, the description ‘Active’ would automatically appear on the screen next to the status type field.

You can click ‘OK’ to save the status details, or click ‘Cancel’ to exit the add option without saving the entered details.

Query/Search For Status Details

Click ‘Query’. The following pop-up window is displayed:

Query Status Details Screen

You may select records using the Status Code or Status Type fields. You may also use any combination (or even partial combination) of these fields in order to select the required status record(s).

If you are unsure of the status code to query, click the lookup icon to see a list of the status codes.

If any match to the selection criteria is found then the first record will be displayed to the screen and a message displayed to the bottom of the screen.

Record 1 of 2

Viewing Status Details

To view the other records selected press N to view the next record, and P to view the previous record. If N is chosen and there are no records after the current record an error message is displayed. If P is chosen and there are no previous records an error message is displayed.

Updating Status Details

Before Updating, a record must be selected through the Query option, and an error message is displayed if a record for update has not been selected.

Click ‘Update’.

The status code cannot be changed as this is a key to the status file. When the changes have been made to the status description click ‘OK’ to save the changes to file, or click ‘Cancel’ to exit the update option without saving changes.

Deleting Status Details

Before Deleting, a record must be selected through the query option, and an error message is displayed if a record for deletion has not been selected.

You cannot delete a status code that is currently referenced to by some or all of your member records.

If you try to do this, an error message will be displayed and you will be returned to the main menu without deleting the status code.

Click ‘Delete’.

When the delete option is selected a window will appear with the options ‘Yes’ and ‘No’.

If you do not wish to delete the record click ‘No’. You will return to the program menu.

If you wish to delete the record click ‘Yes’. The following menu is displayed

Another window will appear asking if you are sure.

If you do not wish to delete the record click ‘No’. The program returns to the main menu.

If you wish to delete the record click ‘Yes’. The record is deleted and the program returns to the main menu.

Printing Status Details

Click the ‘Report’ icon.

If records have been selected in the program before the report option was selected the following window is displayed.

Print Options Screen 1

If no records have been selected in the program before the report option was selected the following window is displayed.

Print Options Screen 2

Record Prints the current screen Record.
Query Prints the all the selected records.
Select Allows selection of records for the report using the Status Code field.
Exit Closes the window and leaves the report option.

If the Select option was chosen and no records are found to match the selection criteria entered, an error message is displayed and new selection criteria should be entered.

If records are found to match the selection criteria the report continues and options are then displayed for printing.

The options for printing are described in the document Printing From Pension Master.

Exit Program

Click ‘Exit’. This leaves the program and closes the current browser tab.
