Introduction

Only users set up as Supervisor status in Users may access this program.

The type of access granted can be view only or full access. For the view only option, you will not be permitted access to any menu option that allows you to add/update/delete data.

If you do not have Supervisor status, you will not be permitted access to this program.

This program allows maintenance of the Scheme file and allows you to set up new schemes and maintain details of existing schemes. The different screens of information that you may find in the Scheme File are as follows:

Basic Scheme Details Screen

Adding A Scheme

Basic Scheme Details

Click ‘Add’.

You will be asked if you wish to copy an existing Scheme. If you select ‘Yes’ you will be prompted to select the Scheme Code of the Scheme you wish to copy or select one from the lookup list.

Input the details as follows:

SCHEME CODE - A new scheme code will be automatically generated for you. You will not be permitted to amend this.

SCHEME NAME - The scheme name must be entered.

REGION - This will determine how a members State Pension Age is calculated and how certain screens are displayed. Select the Region from the drop-down list:

  • UK
  • Ireland
  • Belgium
  • Denmark
  • Jamaica
  • Gibraltar
  • Trinidad and Tobago
  • Norway
  • Sweden

ALTERNATIVE SCHEME CODE - Enter an alternative to the ‘SCHEME CODE’ for your scheme, if necessary. This field will default to the value set for ‘SCHEME CODE’. You can use this field to search for member records using the Query facility in Members.

BASE CURRENCY - Select a currency from the drop-down list as the base currency for this scheme e.g. if your scheme is UK based, enter ‘Pound Sterling’.

When your scheme opts to have it’s base currency as the Euro, you should update this field to be ‘Euro’.

You will then be prompted that all monetary values for this scheme are to be converted from the NCU to Euro and that this process in not reversible. You will be asked whether you have an up to date backup of the current scheme data and if not, you will be given an opportunity to exit the change and take a backup. Once this has been done, all monetary amounts for the scheme, including all membership data and historical unit prices for that scheme will be converted to Euro’s.

A Euro conversion will only be performed on a Members Benefit Quotations/Calculations if the value is numeric and it hasn’t been passed through the format function in the formula.

RENEWAL DAY/MONTH - These are numeric fields and must contain valid date values i.e. A day value of 31 and month value of 09 would be invalid. These will be used to default the Renewal Date when adding new categories for this scheme (see the Categories option).

MARKET ADJUSTMENT FACTOR - This field can be used to record a market adjustment factor that you may use to adjust transfer values in a defined benefits scheme to allow for current market conditions.

SCHEME TYPE - Select one of the following options from the drop-down list:

  • Defined Benefits
  • Defined Contributions
  • Hybrid

CONTRACTED OUT - Indicates that the scheme is contracted out of the State Earnings Related Pension Scheme (SERPS). This field is only relevant to UK based schemes.

SCHEME ECON (UK)/REVENUE REFERENCE NUMBER (IRL) - Enter the Scheme ECON (UK schemes) or the Revenue Reference (Irish schemes) as appropriate.

SCHEME SCON (UK)/PENSIONS BOARD REFERENCE NUMBER (IRL) - Enter the Scheme SCON (UK schemes) or the Pensions Board Reference (Irish schemes) as appropriate. For Irish schemes, the Pensions Board Reference Number is printed on the new Pensions Board Fee Report.

PSO REFERENCE NUMBER (UK) - This field appears on the screen for UK based schemes only and holds the PSO Reference number for printing on reports as required.

PROVIDER REFERENCE FOR CPF (UK) - This field appears on the screen for UK based schemes only and holds the Provider Reference number for Combined Pension Forecast.

AUTO ENROLLED

  1. Following confirmation that all schemes will have a staging date. If the auto enrolled field is ticked the staging date field is mandatory.
  2. If the AE (auto enrolled) field at scheme level is ticked then the value of the member AE field will be automatically set to Y unless the members date of leaving is before the staging date where it should be set to N.
  3. If the AE field is unticked, the value of the member AE field will be automatically set to No.
  4. If the staging date field only is updated, no change will be made to the existing values on Pension Master. A message will be produced advising that non active members AE field may need to be updated following the staging date change.

STAGING DATE

DIRECTORS SCHEME - This field denotes that the scheme is a director scheme.

TEST SCHEME - This field denotes that the scheme is a test scheme. Test schemes can only contain up to a maximum of 10 members. These members are not included in the total of chargeable members calculated on the Quarterly Fees Print.

PRSA SCHEME - This field denotes that the scheme is a PRSA scheme.

TOLERANCE % - This field is used to set a tolerance percentage against which incoming salaries, contributions and NI history details are checked. E.g. if you set a tolerance percentage of 20%, any salaries, contributions or NI details that are more or less than 20% deviant from the value they were last time they were loaded onto Pension Master will produce a warning message. If you do not wish to use this feature, set the tolerance percentage to 0.

SECURITY RATING - Select a security rating from the drop-down list to use for this scheme. This is used in conjunction with the Security Rating in the Users option to determine whether the current user has access to any member records in the current scheme.

ALLOW MANUAL UPDATE OF UNITS? - Click this field if you want to allow manual update of the investment history records for members of this scheme.

STATUS - This field denotes that the status of the scheme Select one of the options from the drop-down list:

  • Active
  • Frozen
  • Frozen DB with Active DC
  • In Wind Up
  • Inactive
  • Paid Up
  • Test
  • Transferred
  • Wound Up
  • Not Staged Yet
  • Not Proceeding
  • Not Decided
  • Transferring Out

CARE SCHEME - This field denotes that the scheme is a CARE scheme. By ticking this then have access to the care screens at scheme and member level.

SSAS SCHEME - This field denotes that the scheme is a SSAS scheme. By ticking this then have access to the SSAS screens at scheme and member level.

TOTAL CHARGE - This field denotes the total charge % for a scheme.

TAX REFERENCE NUMBER - Enter the tax reference number for the pension scheme. The tax reference number must be in the format 12345678AB, where the first 8 characters are numeric and the last 2 characters are letters A-Z. If the user enters less than 10 characters then preceding 0s (zeros) will automatically fill up the field e.g. if the user enters 123456AB, then this will automatically be changed to 00123456AB.

PIP RENEWAL DATE - Enter the PIP renewal date in the format DD/MM/YYYY.

SALARY FREQUENCY - This field denotes the salary frequency used in the Contribution Salary Cap Report. Select a Salary Frequency from the drop-down list:

  • Annually
  • Monthly

CONTRIBUTION SALARY CAP - This field denotes the cap used in the Contribution Salary Cap Report.

Click ‘OK’ to save the details to the scheme file or click ‘Cancel’ to exit the add option without saving the details.

You will now be returned to the main menu. If there are any other details you wish to add to the scheme, you should go directly to the appropriate screen and add them now.

Member Screen Selection

Click Menu and then select the Member Screen Selection option and press Return.

This screen allows an appropriate user to customise the member record screens that are available for use for the selected scheme. It can also restrict access to those screens by reference to a security rating, as well as restrict access to viewing the data by reference to a view only rating. This enables the protection of sensitive data from unwanted viewing e.g. block access to directors salary screen from a junior administrator, or enables you to switch off screens that you do not intend to use for the selected scheme.

The following screen is displayed:

Member Screen Selection Screen

To update the display flags, view only ratings and security ratings, click ‘Update’.

DISPLAY - Each screen, including any User Defined Screens and History can be ‘switched off’ by unticking the corresponding Display flag for a screen. This will result in the screen not being displayed when the Menu or Report options are selected in Members. This could be used if it is envisaged that a particular screen is not required, so it could be ‘switched off’ to avoid confusion to users. If, in the future, the screen was needed then it could be simply switched back on by unticking the checkbox.

This function could also be used to archive member data following a data conversion for instance. Data items that aren’t really needed but could be of some use in the future could be loaded into a User Defined Screen or History and then the screen could be ‘switched off’.

VIEW ONLY - Each screen, including any User Defined and History screens, can be allocated a security rating from the drop-down list.

The objective of this rating is to prevent unauthorised users updating/deleting important data i.e. a junior with a low security rating, will not be allowed to update/delete the data of a screen which is set to a higher rating. Once the ratings have been set for the Members screens, then if a junior tries to get into a corresponding screen set to a high view access then ‘VIEW ONLY’ will be printed on the bottom line of the screen. The Add/Update/Delete buttons in the screen will therefore deny access to the user, and an appropriate error message will be displayed to the screen.

When all the details have been entered click on ‘OK’ to save the member screen selection details, or click on ‘Cancel’ to exit entry without saving the details.

SECURITY - The Security Rating will have been defaulted to that on the Basic Scheme Details when it was first entered. This can be amended to any available Security Rating to prevent junior users being able to access that particular screen of members in the selected scheme. As mentioned previously, this could be implemented to prevent a junior administrator being able to view a directors salary screen.

When all the details have been entered click ‘OK’ to save the member screen selection details, or click ‘Cancel’ to exit entry without saving the details.

User Access Details

Click Menu. Then select the User Access Details option from the selection screen.

The following screen is displayed:

User Access Details Screen

This screen shows who has access to this scheme and the members contained therein. There are three types of access level:

  • All users have access to the scheme. This is equivalent to an entry of ‘all’ in the USER column of this screen.
  • All members of a particular workgroup have access to the scheme. See Workgroups
  • User level access. Here you define particular users via their user ID (as set up in Users) so as to restrict access to a scheme on an individual user level.

USER - Enter ‘all’, a workgroup code or a username to indicate who you wish to have access to records in this scheme. In the latter two cases, you can click on the lookup icon for a list of the available workgroups and usernames. Workgroups are always displayed in uppercase, whilst usernames are always displayed in lower case.

For example, the user ‘john’ may have acces to the scheme via a user entry in the User Access Details screen, via a workgroup entry for the workgroup that ‘john’ is a member of and via the ‘all’ user setting. In this case, the user level access settings take priority. If the entry for ‘john’ is deleted either via this screen or via the User Access Details screen in the Users, then the other two options (one for the workgroup and one for the ‘all’ user remain) thus meaning that ‘john’ will still have access to this scheme. To remove ‘john’ from accessing this scheme, it will be necessary to remove ‘john’ from the appropriate workgroup (to do this, see the Workgroup Code section of the Users page) and also remove the ‘all’ user setting from this scheme.

When all the user access details have been entered click on ‘OK’ to save the user access details, or click on ‘Cancel’ to exit user access entry without saving the details.

Code Allocation

Click Menu and then select the Code Allocation option from the selection screen.

This screen is used to change the codes that have been assigned to the Scheme. The following types of code are used to identify separate details for members of the Scheme:

  • Additional Benefits
  • Transfer In
  • Transfer Out
  • Deferred Pensioner
  • Pensioner
  • Reason for Increase (for both Deferred Pensioners and Pensioners)
  • Special Codes
  • Accounting
  • Care Averaging Earnings
  • Disability

When a members details are updated the lookup list will only show the codes allocated to the members Scheme. New Schemes will already have a pre-set group of codes allocated when the scheme was created. Any additional codes that may be required for each scheme must first be entered using the Codes option on the System Data menu.

The following screen is displayed:

Code Allocation Screen

Click ‘Update’ to Update Codes for the Scheme. The Codes must already have been entered using Codes.

If you try to delete a Code that is in use by a Member then a message is displayed indicating the first Member number found.

Special Codes also use Groups. This is a way of grouping similar codes together under a common name. For example you could report on all members who have a Special Code of ‘2TIMES’ or any member who has codes in the ‘LUMPSUM’ group.

An error message is displayed if you try to delete the Reason for Increase Code called ‘RNEWPEN’. This is because any Pension/Deferred Pension History lines that are added will use the ‘RNEWPEN’ as the default Reason for Increase Code.

Lifestyle Matrix Setup

Click Menu and then select the Lifestyle Matrix Setip option and press Return.

The following screen is displayed:

Lifestyle Matrix Setup Screen

This screen is used to record details of any lifestyle matrices that may be applicable to this scheme.

The details are held by lifestyle matrix code and either age or term. To navigate between entries for different lifestyle matrix codes and age/terms, use the < and > buttons on the screen.

When a lifestyle matrix has been specified for a particular contribution type, Pension Master calculates the members age/term to retirement and then looks up either the appropriate age for the member or the appropriate term to retirement in the lifestyle matrix

Example 1

If a lifestyle matrix has entries for two terms to retirement, 5 years and 10 years and the member has a calculated term to retirement of 7 years, the lifestyle matrix entry for term 10 years will be the one used. When the member has 5 or less years to retirement, the lifestyle matrix entry for term 5 will be the one used.

Example 2

If a lifestyle matrix has entries for two start ages, 50 and 60 years and the member has a calculated age of 53, the lifestyle matrix entry for start age 50 will be the one used. When the member attains age 60, the lifestyle matrix entry for start age 60 will be the one used.

The Fund Balancing program allows you to report on members who should have had a lifestyle event since the last date they invested contributions. It shows you the current allocation of the members investments, and the proposed new split of investments after re-investing according to the appropriate lifestyle matrix.

A lifestyle event is where a member crosses a boundary within a lifestyle matrix e.g. if the lifestyle matrix sets out that at age 50 they must switch investments into cash based funds, when the member reaches age 50, they have a lifestyle event.

To add details for a new lifestyle matrix, or to add a new age/term to an existing lifestyle matrix, click ‘Add’. To update the displayed details, click ‘Update’.

Input the details as follows:

MATRIX CODE - Enter a code for your lifestyle matrix.

DESCRIPTION - Enter a description for your lifestyle matrix.

TYPE - Enter the type fo lifestyle matrix. This is only allowed when adding a completely new lifestyle matrix to Pension Master. The valid options are as follows:

  • A - Age related - based on the member’s age at date due (as entered in Contribution Allocations). Lifestyle matrix looked up at members age, rounded down to the nearest age relative to their position in the matrix.

  • T - Term related - term from date due (as entered in Contribution Allocations) to Target Pension Date. Lifestyle matrix looked up at term, rounded up to the nearest term relative to their position in the matrix.

START AGE/TERM TO TRD- Enter a start age or a term to retirement for your lifestyle matrix.

FUND - Enter a fund for this lifestyle matrix and age/term. You can enter as many valid funds as you like for this lifestyle matrix and age/term.

PERCENTAGE - Enter a percentage to allocate for this fund, lifestyle matrix and age/term. This must add up to 100% across all funds for this lifestyle matrix and age/term.

Contribution Expenses

Click Menu and then select the Contribution Expenses option and press Return.

The following screen is displayed:

Contribution Expenses Screen

It is important that you correctly set up the contribution expense assumptions at this stage as these will be used to deduct from incoming contributions and, if not correctly set will lead to incorrect unit allocations.

There are three types of expenses that are allowed in Pension Master:

  • Percentage of contributions - This can be used to take a charge based on a percentage of the members current contribution.
  • Percentage of salary - This can be used to take a charge based on the members current salary. You can choose which of the salary types you wish the charge to apply to.
  • Member Charge - This can be used to take a fixed monetary charge off each members incoming contributions. This fixed charge will be applied to each member concerned.

Alternatively, you can enter formula to calculate the expense values. This is useful when your expense setup does not conform to the simple model allowed for in this screen.

Fill out the details as follows:

MEMBER CHARGE - Enter the amount of flat monetary charge you wish to apply to each members incoming contribution e.g. if you wish to apply £5.00 to each contribution, then enter 5.00. Enter 0.00 if you do not wish to take a Member Charge off the incoming contributions.

SALARY TYPE - Click the lookup icon to select a salary type for calculating expenses as a percentage of salary.

CONTRIBUTION TYPE - Enter the contribution type code for this line of expenses.

CONT % - This represents the percentage of each contribution type amount you wish to take as contribution percentage expenses, e.g. if you wish to take 5% of contributions, set this field to 5.00.

SALARY % - This represents the percentage of each contribution type amount you wish to take as salary percentage expenses. These will take expenses as a percentage of the members current salary, e.g. if you wish to take 5% of salary, set this field to 1.00 (i.e. 1%).

MEM CH % - This represents the percentage of each contribution type amount you wish to take as a member charge. These must either total up to 0% or 100%. 0% is used to indicate that a member charge is not to be taken.

FORMULA - Enter the formula code you wish to use to calculate expenses for this contribution type. This is an optional feature and is only to be used when you cannot calculate your expenses using the simple method allowed for by the other fields on this screen.

Entering a valid formula code in this field will automatically set the other numeric expense assumption fields for this contribution type to zero as they are no longer relevant in the calculation of expenses for this contribution type.

In the formula you use to do this, you have access to the special formula syntax cont_value which inserts the gross amount of the current contribution type into the formula so you can use this to calculate your expenses.

When all the details have been entered click ‘OK’ to save the details, or click ‘Cancel’ to exit entry without saving the details.

Producing Ad-Hoc Letters

If you wish to send an ad-hoc letter to one of the addresses for the current scheme, you can easily do it using the ‘Quick Letter’ facility. All that is required is that you type the body of the letter, the rest is done automatically by Pension Master.

Select the ‘Quick Letter’ button from the screen. This automatically starts your wordprocessor and transfers enough data to start and end the letter i.e. it transfers the schemes address, inserts today’s date, sets out the Dear xxx part of the letter and then does the Yours sincerely… at the end of the letter. If the scheme’s address it not available, you will be informed that one must be entered before using the ‘Quick Letter’ option.

If there are no signatory details held against the address, then the letter will be printed without a contact name. If the address has one or both signatory details held, then you will be prompted with a window as follows:

Select Contact Name Window

Select one of the names to be displayed on the letter. Alternatively you can select to keep the name blank. Your name and job title will be automatically filled in at the end of the letter.

When you have finished your letter, print it off in the normal way. You can then exit from your wordprocessor as required. To return to Pension Master, simply click on the titlebar of an active Pension Master window.
