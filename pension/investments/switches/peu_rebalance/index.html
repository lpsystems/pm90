Introduction

The Rebalance switches program allows users perform a mini lifestyle on the members investment portfolio for members whose investment set up is currently not lifestyling. The members entire accumulated fund at the process date will be rebalanced. The objective of this program is to enable the user to process the outstanding ‘not yet processed’ switches on a scheme or for specific selection criteria in a scheme in one go. These rebalance switches would have been added via the member screen or through the Load Rebalance Switch Program

Adding Rebalance Switches

Click ‘Add’.

Selection Screen

If there are any rebalance switches where processing has already commenced but not finished for any member in the selection criteria input, the bulk switch will be aborted. You will be advised of the reason for the bulk switch cannot proceed and you will be able to view the members involved.

If there are any ‘not processed switches’ for the selection criteria they appear. You can click ‘OK’ to save the Rebalance Switch details, or click ‘Cancel’ to exit the add option without saving the entered details.

When the details have been saved you now have the option to sell. The process date will always be todays date. When you press sell the user is brought into the unit prices screen to input the sale date and the prices will be retrieved from the unit prices screen in investments. If there is no price for the date input, a warning will be produced.

You then have the option to complete the switch. If you choose NO, the sale process will be aborted. If you choose YES the sale will be updated in the member screen.

To post the proceeds of the rebalance sales to units, is completed via the Unit Posting Program . The transaction type to post is the REBALIN transaction type.

Delete Rebalance Switches

Before deleting, a record must be selected through the query option, and an error message is displayed if a record for deletion has not been selected.

The rebalance switch can only be deleted if the Purchase Stage has not been completed. If the Purchase Stage has been completed, you need to unpost the purchase stage units before you can proceed with the delete

Click ‘Delete’.

  • If the rebalance switch is processed via the Process Rebalance Switches Option, then it can only be deleted via this option by querying the bulk process id.
  • If the rebalance switch is deleted via the member record, all data relative to the switch is deleted.
  • If the rebalance switch is deleted via the bulk rebalance switch screen, you have the option of leaving the unprocessed Rebalance Switch on the member record so that you can start the rebalance process again.

Delete Message Rebalance Switch Screen

Undo

Click ‘Undo’.

An undo run effectively reverses a ‘Finished’ rebalance switch back to the ‘Pending’ stage. The ‘Undo’ button only appears for ‘Finished’ switches.

You are only allowed to undo for a switch if there are no posted units for the member after the current date due. If posted units exist for the member after the current date due, you should unpost all unposted units for the member in date descending order, starting with the latest date due.
