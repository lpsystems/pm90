Introduction

This program allows you to post allocated contributions to units.

Unit Posting Screen

Run Purchase

Click ‘Run Purchase’.

You will be asked if you wish to do a Provisional or an Actual run. A Provisional run does all that an actual run does, but does not write any results to the database.

You are only allowed to post units for a member if there are no unposted units for the member prior to the current date due. If unposted units exist for the member prior to the current date due, you should post all unposted units for the member in date order, starting with the earliest date due. You can get a list of the unposted units for the member by using the Unposted Units Report.

SELECTION CRITERIA - Select the group of members you require to post units for using the Scheme, Company, Category, Status, Special Code and Member Number fields (see Selection Criteria). When the selection criteria have been entered click ‘OK’ to proceed.

You can click ‘Cancel’ to return to the main menu and abort the unit post run.

The following screen is displayed:

Date Selection Screen

DATE DUE - Enter the date due for the contributions you are posting to units.

DATE RECEIVED - Enter the date you received the contributions you are posting to units.

DATE INVESTED - Enter the unit price date for the units you are posting. This should be found on the statement from the investment fund that shows how many units were bought by the contribution.

TRANSACTION TYPE - Enter the transaction type for the contributions you are posting to units. If you are unsure of what code to use, click the lookup icon to see a list of transaction codes. If the transaction type of SWIN, LIFEIN or REBALIN is entered. The screen will include a column for Origin Fund.

LOAD DETAILS FROM - Select a CSV file from a folder on your hard drive. This enables the user to populate the screen by loading the Fund and Date Invested from a CSV file. The CSV file should simply have 2 columns, column A for the Fund Code and column B for the Date Invested. No headers or footers are permitted. The file will be checked for invalid data in the process. The user will be able to amend the unit price date once the data from the load file is displayed to the screen.

Type Selection Screen

This allows you to only post the funds of the SWIN based on the origin of the Fund. So you may only have received the funds from a certain part of the switch and you can post those without having to wait for all of the funds.

REPORT TYPE - Select the report type as follows:

  • Summary - For each member selected, produce a report which shows the total amount of contribution received, total contribution allocated and total number of new units bought by the unit posting. Each fund starts on a new page.
  • Detailed - As for the Summary report, but the detail is broken down into individual contribution types for each member.

FUND - The investment funds that have unposted units at the selected dates will be listed, if you wish to posted units for a fund insert a date in the date invested field. You can have different dates for each of the rows, or if you don’t wish to post the units at this time leave the field blank. If you want to post units for all investment funds with unposted units at this date for the selected members, enter a date in the first row under ‘ALL’.

Pension Master then proceeds to calculate and post the units for the contributions to each member selected. The unit price used for this will be based on the unit buy basis field in Investment Funds and is looked up on the Unit Prices file at the date you have entered in Date Invested. Full account of any unit allocation percentage for the fund concerned is allowed for.

If no unit buy basis price exists for the funds an exception report is produced.

The number of new units is calculated as follows:

UNITS = ALLOCRATE * CONT / UNITPRICE

where:

UNITS = new units bought

ALLOCRATE = allocation percentage for fund concerned, expressed as a percentage

CONT = amount of gross contribution

UNITPRICE = unit buy price

If successful, a report detailing the proposed unit posting is displayed and can be printed at this time. The type of report produced depends on the Report Type field.

If the unit posting was un-successful, an exception report will be produced which details the records that were rejected and the reason why. You must address each exception before being able to post the contributions to units successfully.

The options for printing are described in the document Printing From Pension Master.

The system will then return to the main menu.

Undo

Click ‘Undo’.

An undo unit posting run effectively reverses an actual unit posting run as set out above and brings the selected member records back to the situation immediately before the actual unit posting run was performed.

You are only allowed to unpost units for a member if there are no posted units for the member after the current date due. If posted units exist for the member after the current date due, you should unpost all unposted units for the member in date descending order, starting with the latest date due.

If you undo a unit dividend run, then the corresponding entries are deleted from the members Investment History screen.

Details of how this works can be found in the ‘Run’ section detailed above.

Dividend

Click ‘Dividend’.

You will be asked if you wish to do a Provisional or an Actual run. A Provisional run does all that an actual run does, but does not write any results to the database.

SELECTION CRITERIA - Select the group of members you require to calculate unit dividends for using the Scheme, Company, Category, Status, Special Code and Member Number fields (see Selection Criteria). When the selection criteria have been entered click on ‘OK’ to proceed.

You can click ‘Cancel’ to return to the main menu and abort the unit dividend run.

The following screen is displayed:

Dividend Details Screen

DATE DUE - Enter the date for your unit dividend.

DATE RECEIVED - Enter the date for your unit dividend. This is generally going to be the same as DATE DUE.

DATE INVESTED - Enter the date the units in respect of your unit dividend were allocated at. This should be found on the statement from the investment fund that shows how many units were bought by the dividend run. This is generally going to be the same as DATE DUE.

TRANSACTION TYPE - This is set to ‘DIVIDEND’ and is display only.

DIVIDEND % - Against each applicable fund, enter the percentage you wish to apply as a dividend for investments in that fund. For example, if you enter 1% against the fund called ‘MANAGED’, the unit dividend for a member will be 1% of the total number of units up to and including DATE DUE in the MANAGED fund.

Pension Master then proceeds to calculate and post the unit dividends for each member selected.

If successful, a report detailing the proposed unit dividends is displayed and can be printed at this time. The type of report produced depends on the Report Type field.

If the unit dividend run was un-successful, an exception report will be produced which details the records that were rejected and the reason why. You must address each exception before being able to post the unit dividend successfully.

The options for printing are described in the document Printing From Pension Master.

The system will then return to the main menu.

Charge

Click ‘Charge’.

You will be asked if you wish to do a Provisional or an Actual run. A Provisional run does all that an actual run does, but does not write any results to the database. Then you will be asked if it should be completed on a value or unit basis.

The following screen is displayed:

SELECTION CRITERIA - Select the group of members you require to calculate unit dividends for using the Scheme, Company, Category, Status, Special Code and Member Number fields (see Selection Criteria). When the selection criteria have been entered click on ‘OK’ to proceed.

You can click ‘Cancel’ to return to the main menu and abort the unit charge run.

The following screen is displayed:

Charge Details Screen

END DATE FOR UNIT/VALUE CALCUATIONS - Enter the date the units/value in respect of your charge are to be calculated at.

DATE TYPE FOR UNIT/VALUE CALCUATIONS - Enter the date type from the investment history screen that should be used in conjunction with the end date.

TRANSACTION TYPE - Insert or select a type from the drop-down list.

DATE DUE - Enter the date for your charge.

DATE RECEIVED - Enter the date for your charge. This is generally going to be the same as DATE DUE.

All the funds and descriptions are now listed for you to insert a charge %.

Pension Master then proceeds to calculate the charge on each fund for each member selected.

Charge Post

Click ‘Charge Post’.

The option is for the Charge runs that are completed on a value basis. When the unit price is known you complete the charge in the same way as the run purchase process.

If successful, a report detailing the proposed charge is displayed and can be printed at this time. The type of report produced depends on the Report Type field.

If the charge run was un-successful, an exception report will be produced which details the records that were rejected and the reason why. You must address each exception before being able to proceed.

The options for printing are described in the document Printing From Pension Master.

The system will then return to the main menu.

Exit Program

Click ‘Exit’. This leaves the program and closes the current browser tab.
