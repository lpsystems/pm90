Introduction

This program calculates expected contributions for a specified group of members. If successful, these are loaded into a Contribution Batch for onward processing.

It is intended that you send the schedule produced by this program to your client to request the contributions that are then due. The client should then ensure they deduct these contributions from the member and send you full details. These will then be reconciled against what was expected in Batch Data Input.

The contribution amounts are calculated by reference to the Contribution Rate Percentages screen in the member file for each selected member. If, for a selected member, there are no details held in this screen, the values are defaulted to those set up in the Contribution Rate Defaults screen in Categories for the members category.

Note that types EEPR and ERPR are only calculated for members who are on NI Table F i.e. an entry of ‘F’ is in the NI Table field on the Member Indicators screen in the members record.

If a contribution type is setup to use a Formulae, the calculation date used will be the date you enter into DATE DUE on screen 2 of the Contribution Request Run. Thus, for example, if the formulae used involved factor tables, these will be looked up using the date due you enter.

The following options and screen appear when this program is selected.

Contributions Request Run Selection Screen

Run

Click ‘Run’.

You will first be asked if you wish this to be a Provisional or an Actual run. A provisional run is the same as an actual run except no details are saved to the database.

SELECTION CRITERIA - Select the group of members you require to run the Contributions Request Run for using the Scheme, Company, Category, Status, Special Code and Member fields. When the selection criteria have been entered click ‘OK’ to proceed or click ‘Cancel’ to return to the main menu.

The following screen is displayed:

Contributions Request Run Options Screen

Enter the details as follows:

DATE DUE - Enter the date due for your contributions. This is the date the contributions are or should have been deducted from the members salaries.

TRANSACTION TYPE - Enter a transaction type for your contributions. Click the lookup icon to see a list of the available transaction types.

REPORT TYPE - Specify whether you want to produce a summary or detailed report. These reports are the same except that the detailed report gives a member by member breakdown of the contribution request details.

GROUP BY COMPANY - Check this option if you wish to sort your report by company code. Otherwise all records will be grouped as if they are from the same company. This option is useful if you are sending out contribution request schedules to multiple locations.

When you have entered these details, click ‘OK’ to proceed or ‘Cancel’ to return to the main menu.

The contribution request amounts will now be calculated.

A report may be generated after the calculation:

EXCEPTION REPORT - If any exceptions have been found then you will be asked if you wish to view or print the report.

This report details any problems that have been found during the calculation of the contribution request amounts. Details are printed describing the problem that has been found. Possible exception messages are:

  • Contribution frequency is not setup for this category of members.
  • For percentage of salary calculations, no salary is available for a member in the scheme year of the contribution due date.
  • A contribution type has been found for which there are no details set up on the members Contribution Rate Defaults screen.
  • If a contribution is being calculated by a formula, the formula is returning a non-numeric value.

On closing the exceptions report, you will be returned to the main menu and any calculated contribution request amounts will not be saved.

RUN REPORT - If exceptions have not been found then a run report will be displayed to the screen.

This report details the contribution request amounts which will be saved to a batch. The number of the generated batch will be displayed to the screen. You should take a note of this.

The options for printing are described in the document Printing From Pension Master.

The program will then return to the main menu.

Exit Program

Click ‘Exit’. This leaves the program and closes the current browser tab.
