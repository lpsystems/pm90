Introduction

This program allows you to allocate contributions to investment funds as per the members current investment choice.

The following options and screen appears when this program is selected:

  • Run
  • Undo
  • Exit

Contribution Allocation Screen

Run

Click ‘Run’.

You will be asked if you wish to do a Provisional or an Actual run. A Provisional run does all that an actual run does, but does not write any results to the database.

SELECTION CRITERIA - Select the group of members you require to allocate contributions for using the Scheme, Company, Category, Status, Special Code and Member Number fields (see Selection Criteria). When the selection criteria have been entered click ‘OK’ to proceed.

You can click ‘Cancel’ to return to the main menu and abort the contribution allocation run.

The following screen is displayed:

Date Selection Screen

DATE DUE - Enter the date due for the contributions you are allocating.

DATE RECEIVED - Enter the date you received the contributions you are allocating.

TRANSACTION TYPE - Enter the transaction type for the contributions you are allocating. If you are unsure of what code to use, click the lookup icon to see a list of transaction codes.

REPORT ORDER - Select the order in which you wish records to be printed on the report. Valid options are:

  • Sort by manager/fund.
  • Sort by member surname, forename.

Pension Master then proceeds to allocate the contributions to investment funds for each member selected using that members current investment choice as set up on the member record.

If there are no records on the ‘Investment Setup’ screen in Members for the member being run, then the ‘Investment Setup’ details in Schemes are used.

If successful, a report detailing the proposed allocations is displayed and can be printed at this time.

If the allocation was unsuccessful, an exception report will be produced which details the records that were rejected and the reason why. You must address each exception before being able to allocate the contributions successfully.

The options for printing are described in the document Printing From Pension Master.

The system will then return to the main menu.

Undo Allocation Run

Click ‘Undo’.

An undo allocation run effectively reverses an actual allocation run as set out above and brings the selected member records back to the situation immediately before the actual allocation run was performed.

Details of how this works can be found in the ‘Run’ section detailed above.

Exit Program

Click ‘Exit’. This leaves the program and closes the current browser tab.
