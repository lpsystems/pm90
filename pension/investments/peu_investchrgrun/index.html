Introduction

This program allows you to to facilitate the distribution of a charge, from an external data file, on a prorate basis across the member’s total invested value. The members in the file can be from any scheme.

DATA FILE The external data file must have the following mandatory fields in the order stated in a CSV file type. Any member data held in the ensuing columns will be ignored.
1. Pension Master Member Number 2. Transaction Type Code 3. Amount

Member Number - the Member Number will be the unique identifier Transaction Types - are held at system level in Codes under ‘Contribution Transactions’ in the System Data menu tree on pension master. The transaction type code in the data file must exist on Pension Master. Amount - the charge ‘Amount’ value should be a positive value. It will be converted to negative through the load process. If the data file contains negative values, warnings will be produced, and the load will be aborted.

Run

Click ‘Run’.

You will be asked if you wish to do a Provisional or an Actual run. A Provisional run does all that an actual run does, but does not write any results to the database.

You can click ‘Cancel’ to return to the main menu and abort the unit post run.

The following screen is displayed:

Investment Charge Run Screen

FILENAME - Enter the filename path of the data file you are going to use, if you are unsure use the lookup button and select the file from the file directory.

DELIMITER - The data delimiters available are: Colon, Comma, Pipe, Semi-Colon, Tab

MONETARY FORMAT - The monetary format of amounts available are: 99999.99 or 99.999,99. .

HEADER SIZE - Enter the number of lines the footer takes up in the data file. If this box is left blank, it is assumed the data file has no footer, and a default of 0 is entered..

FOOTER SIZE - Enter the number of lines the header takes up in the data file. If this box is left blank, it is assumed the data file has no header, and a default of 0 is entered..

DATE DUE - This date will populate the date due on the investment charge record for the members investment history screen.

DATE RECEIVED - This date will populate the date received on the investment charge record for the members investment history screen .

ASSET VALUATION DATE - The Valuation Date entered will be used to calculate each member’s fund value which is used to determine how the charge will be split across fund and contribution types. Units with a date invested up to and including the Valuation Date will be used to calculate the value of the members assets using the unit price at or effective at the Valuation Date.

If successful, a report detailing the proposed unit posting is displayed and can be printed at this time.

If the unit posting was un-successful, an exception report will be produced which details the records that were rejected and the reason why. You must address each exception before being able run the charge run again.

The options for printing are described in the document Printing From Pension Master.

The system will then return to the main menu.

Undo

Click ‘Undo’.

An undo investment charge run effectively reverses an actual charge run as set out above and brings the selected member records back to the situation immediately before the actual investment charge run was performed.

A check will be made to see if the charge has been posted to units. If it has been posted to units, the user will be advised that they must use the Undo option in the Post Units program where, during the unpost process, the user will have a choice of unposting the units or deleting the records from the investment history screen. If it has not been posted to units, the following report is produced:

UNDO INVESTMENT CHARGE RUN REPORT This report details the split of the members charge across the members contribution types for each of the members investment funds for the selection criteria and charge criteria entered in the undo process. The members will be listed in scheme order.

If all data is present and correct, then the user can continue to undo the charge and the data will be removed from the investment history screen.

You are only allowed to unpost units for a member if there are no posted units for the member after the current date due. If posted units exist for the member after the current date due, you should unpost all unpsoted units for the member in date descending order, starting with the latest date due.

Exit Program

Click ‘Exit’. This leaves the program and closes the current browser tab.
