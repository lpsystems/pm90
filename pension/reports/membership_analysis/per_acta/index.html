Introduction

This program produces an analysis report of a specified field sorted by age and sex. The specified field can be any valid numeric field on any of the following database tables (see the Data Dictionary for further details):

Table Name Description
pe_defhst_b Contribution History File
pe_defhst_h Contribution Totals File
pe_salary1 Salary History File
pe_salary2 Salary History File
pe_salary3 Salary History File
pe_salary4 Salary History File
pe_dpens Basic Deferred Pension Details File
pe_dpenh Deferred Pension Increase History File
pe_mem Member File
pe_mquote Quotation Results File
pe_muds User Defined Screens File
pe_nih NI History File
pe_pens Basic Pension Details File
pe_penh Pension Increase History File
pe_tvin Transfer In Details File
pe_tvout Transfer Out Details File

The following options and screen appear when this program is selected:

  • Proceed
  • Exit

Actuarial Analysis Main Screen

Run A Report

Click ‘Proceed’.

Enter the selection criteria required in order to select the member records you require outputting to the report (see the Data Dictionary for further details on the selection of records in Pension Master).When the selection criteria has been entered click on ‘OK’ to proceed to report date.

You can click on ‘Cancel’ to return to the proceed menu and abort report selection.

REPORT DATE - The report date defaults to today’s date, but it may be overwritten with any date in the format ddmmyy or dd/mm/yyyy.

ANALYSIS FIELD - Analysis field must be entered and can be any field on the tables listed at the start of this section. If a field is entered that does not exist on any of these files an error message will be displayed. If you are unsure of the fieldname you require, click on the lookup icon to see a lookup list of available tables and fields.

Certain fields require you to add an argument, for example, telling them which investment fund to look the field up in. These are as follows:

Table Name Argument 1 Argument 2
pe_defhst_b (Contribution History) pdb_inv_fund (Investment Fund) -
pe_defhst_h (Contribution Totals) pdh_inv_fund (Investment Fund) -
pe_mquote (Quotation Results) pmq_type (Quotation Type) pmq_index (Quotation Index)
pe_muds (User Defined Fields) pmu_type (User Defined Screen Type) -
pe_tvin (Transfer In Details) ptvi_code (Transfer Code) -
pe_tvout (Transfer Out Details) ptvo_code (Transfer Code) -

e.g. if you wished to output total employee ordinary contributions for investment fund 2, you would enter pdh_tee_ordc(2)

The arguments have to be enclosed in brackets () and where there is more than one argument, the arguments are to be separated by a comma e.g. pmq_current(NORM,1) - this selects the first quotation result for Quotation Type NORM.

DATE ENTERED - Date Entered must be entered. There are three valid entries for this field:

  1. Specifies date comparison on Date Joined Company
  2. Specifies date comparison on Date Joined For LA
  3. Specifies date comparison on Date Joined For Pension

If any other value is entered in this field an error message is displayed and the user is invited to re-enter the field or click on ‘Cancel’ to abort input.

HEADINGS - Headings may be entered. If the first heading is left blank the default heading is as follows:

ACTUARIAL ANALYSIS AS AT dd/mm/yyyy

If the second heading is left blank the default heading is as follows:

Age Analysis : xx - xx

When this field has been entered a second screen is drawn as follows:

Age Bands Selection Screen

AGE ANALYSIS - The default values may be overwritten with different values if they do not apply. Click on the ‘Delete Row’ button to delete any extra rows on the screen that are not required, as the blanking of fields is not allowed.

Click ‘Add Row’ to insert a blank row on the screen.

You can click on ‘Cancel’ to exit input of age analysis and return to the report menu.

When the age ranges have been entered a message is displayed at the bottom of the screen

Processing report … Please Wait

The age next birthday for members selected are then calculated and totals for age groups calculated. There are four situations where a selected member will not be included in the totals:

  1. If the analysis field does not contain a value e.g. If Pensionable Salary is the analysis field and the member has no history records.
  2. If the member’s age next birthday does not fall between the upper and lower age limits e.g. If members age next birthday is 67 and age analysis is 0 - 65.
  3. If the date comparison field is blank e.g. If Date Joined Company was specified for the date comparison and date entered company for the member is blank.
  4. If the member joined after the report date e.g. If Date Joined Company was specified as the comparison date and Date Joined Company is 01/03/92 and Report Date is 01/02/92.

When the value entered in Date Entered is 1 the date field checked is pm_dec - Date Joined Company.

When the value entered in Entered Field is 2 the date field checked is pm_djla - Date Joined For LA.

When the value entered in Entered Field is 3 the date field checked is pm_djp - Date Joined For Pension.

When the report has been processed and no records are found to match the selection criteria an error message is displayed:

There are no records to match the selection criteria

When the report has been processed and records are found to match the selection criteria the report continues.

The options for printing are described in the document Printing From Pension Master.

Exit

Click ‘Exit’. This leaves the program and closes the current browser tab.
