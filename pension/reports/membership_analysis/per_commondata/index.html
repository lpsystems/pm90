Introduction

This program produces a series of reports to check on the common data, as defined by The Pensions Regulator resulting in the calculation of the Common Data Score for a Scheme.

It provides the option of running it on a particular scheme and then on Actives, Deferreds and Pensioners, splitting out the results within each status covering 10 areas of data checks.

The following options and screen appear when this program is selected:

  • Run
  • Exit

Selection Screen

Run A Common Data Check Report

Click ‘Run’.

Enter the selection criteria required.

SCHEME CODE - This represents the code corresponding to the scheme the member belongs to. The scheme code must be entered and the code entered must exist on the scheme file. Click on the lookup icon to see a list of the scheme codes. When a valid scheme code has been entered the scheme name will be displayed.

STATUS TYPE - The default is that all are ticked but this can be run for 1, 2 or all 3 status types by clicking the box to display a tick or not.

Click on ‘OK’ to proceed. You can click on ‘Cancel’ to return to the run menu and abort report selection.

When the report has been processed and no records are found to match the selection criteria an error message is displayed:

There are no records to match the selection criteria

If records are found to match the selection criteria the report continues and 2 Microsoft Excel reports are produced:

  • per_commondata_checked_members - This provides a spreadsheet of the basic common data and indicates whether a members data has failed or passed in the last column. It doesn’t indicate why at this point as a member could have failed more than one check. A detailed report can be selected at the next step on Pension Master which provides this detail.
  • per_commondata_score - This summarises the totals for the checks and calculates the common data score - the overall number of members failed as a percentage of total number of members checked.

On returning to the Pension Master screen, you will be prompted as to whether you want to produce the third report, the detail of the fails:

Failed report

On selecting No, the program ends.

On selecting Yes, the detailed report is produced before the program ends:

  • per_commondata_failed - For each data area checked, this report will produce a worksheet listing the member within their current status and the description of their fail for that data area. There are potentially 10 worksheets produced: nino, surname, forename, sex, dob djpdec, npdnpa, laststat, address, postcode. A worksheet will only be produced if it has found a fail in that data area. If a member has failed more than one test in that area of data, it will show all the tests.

Exit Program

Click ‘Exit’. This leaves the program and closes the current browser tab.

Tests that result in Fails

National Insurance Number

  • If the NI Number field is blank.
  • If the NI Number value is invalid for the scheme’s region.

Surname

  • If the Surname field is blank.

Forename/Initials

  • If both the Forename and Initials fields are blank.

Sex

  • If the Sex field is blank.
  • If the Sex field has an invalid value, i.e. not “M” or “F”.

Date of Birth

  • If the DOB is blank.
  • If the DOB exists, but is after the Effective Date of the report.
  • If the DOB exists, but is before the Effective Date of the report, less 110 years.

Date Pensionable Service started - Using Date Joined Company (DJC), Date Joined for Pension (DJP) and Normal Retirement Date (NRD):

  • If either, or both the dates, are blank.
  • If (DJP exists) and (DOB exists and hasn’t failed already), if DJP is earlier than the DOB + 16 years.
  • If (DJP exists) and (NPD exists), if DJP is later than NPD.
  • If (DJP exists) and (DJC exists), then if DJP is earlier than DJC.
  • If (DJC exists) and (DOB exists and hasn’t failed already), if DJC is earlier than the DOB + 16 years.
  • If (DJC exists) and (NPD exists), then if DJC is later than NPD.

Expected retirement/maturity date - Using Normal Pension Date (NPD) and Normal Pension Age (NPA):

  • If the NPD is blank.
  • If the NPA is blank.
  • If (NPD exists) and (DOB exists and hasn’t failed already), then if NPD is earlier than the DOB plus 55 years or later than DOB plus 70 years. (based on the most normal ranges of NRD within 55 to 70)
  • If (NPD exists) and (NPD is within the range of DOB plus 55 years, and DOB plus 70 years),then if the calculated category level NPD is different to the NPD held.
  • If (NPA exists), then if the calculated category level NPA is different to the NPA held.

Membership Status - There are several levels to these checks to ensure the member records are held consistently. Status is held on a history table as well as the current status held on the member’s basic record.

  • If the Status History is empty.
  • If the Status History exists, and the last entry held in the history doesn’t match with the current status.
  • For Actives - If the Date Left (DOL) is present.
  • For Actives - If the Status History Effective Date is earlier than the DJC.
  • For Deferreds - If the DOL is blank.
  • For Deferreds - If the Status History Effective Date is earlier than the DOL, or more than 1 day later than the DOL.
  • For Deferreds - If the Date of Cessation is present on the Deferred Pensioner Record.
  • For Deferreds - If the Total Current Pension from the Deferred Pensioner Totals table is blank or zero.
  • For Pensioners - If the DOL is blank.
  • For Pensioners - If the Status History Effective Date is earlier than the DOL.
  • For Pensioners - If the Date of Cessation is present on the Pensioner Record.
  • For Pensioners - If the Total Current Pension from the Pensioner Totals table is blank or zero.

Address

  • If address line 1 is blank.
  • If address line 1 is present, but address is flagged as Suspect.

Postcode

  • If postcode is blank.
  • If postcode is present, but address is flagged as Suspect.

Notes

The NI Number is renamed to the corresponding equivalent depending on the scheme region ie. Irish schemes will have PPS Number printed instead.

Only UK and Jamaican schemes will show the Normal Pension Age Months as this follows what PM8 displays on the Basic Member Details screen.
