Introduction

This program will first of all calculate the non-selection (or free cover) limit. This will be done by looking at all members of the scheme whose status type is ‘A’ (i.e. all active members of the scheme), multiplying the result by £10,000 and subjecting this to an overall maximum of £700,000. The resulting value will automatically update the new ‘Non-Selection Limit’ field on the new Actuarial Details screen in Schemes. It is envisaged that these values will be made more user-definable in a future version of Pension Master.

A report will then be produced by analysing the life assurance benefit and the capitalised spouses in service benefit values that have been held on an appropriate user defined screen. This will allow the user to select by scheme code and for all the members within that scheme, output a report as follows:

This report would also show the amount for joiners in the last year, pro-rated by reference to the ‘Date Joined For LA’ (date the member joined the scheme for death benefits) field and up to and including the day before the renewal date.

All active members (those members with a status type of ‘A’ corresponding to their respective status code) will be used in both the calculation of the NSL and the corresponding report.

Selection Screen

Run A Report

Click ‘Proceed’.

Enter the selection criteria required (see Selection Criteria).

SCHEME CODE - Enter the scheme code for the scheme you wish to run the report for. If you are unsure of the code for your scheme, click on the lookup icon for a list of schemes.

RENEWAL DATE - Enter the renewal date applicable to your report in the format dd/mm/yyyy or ddmmyyyy. The day and month fields will be validated against the renewal day and month fields on the Basic Scheme Details screen in Schemes.

When the details have been entered click on ‘OK’ to proceed. You can click on ‘Cancel’ to return to the main menu and abort report selection.

The next screen is now displayed as follows:

Life Assurance Field Location Screen

This screen is used to indicate to the report where the various costs associated with life assurance type benefits are to be found.

It is intended that these are to be calculated by Formulae and the results of these formulae written to a pre-defined user-defined screen via the Finalising a Quote method either using Members or Batch Quotation Run.

Pension Master will default the fields to the following values:

Fieldname User Defined Screen Code Position on the User Defined Screen
LSDIS Benefit DIS 1
Capitalised WDIS Benefit DIS 7

If your values are held elsewhere, amend the details on the screen as appropriate. If you are unsure of the available user defined screen codes, click on the lookup icon for a list of the values.

When the selection criteria has been entered click on ‘OK’ to proceed. You can click on ‘Cancel’ to return to the main menu and abort report selection.

When the selection is complete a message is displayed at the bottom of the screen:

Processing report … Please Wait

When the report has been processed and no records are found to match the selection criteria an error message is displayed:

There are no records to match the selection criteria

If records are found to match the selection criteria the report continues and options are then displayed for printing.

The options for printing are described in the document Printing From Pension Master.

Exit Program

Click ‘Exit’. This leaves the program and closes the current browser tab.
