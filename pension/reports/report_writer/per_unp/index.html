Introduction

Use this program to set up the data sets needed to produce the various different kinds of standard letters and benefit statements for members of your pension scheme.

It produces a data file containing member details as selected by you that is suitable for reading into the mail merge facility of Microsoft Word. The full mail merge process is automated.

Creation of a mail merged letter is a three step process:

  1. Create a Microsoft Word mail merge template document that contains your letter, but with special mail merge tags indicating where the data coming from Pension Master is to be inserted.
  2. In Pension Master, create a mail merge report specifying the name and path of the Word template document created in the above step and which items of data from Pension Master you wish to output to this template document.
  3. Merge the data from Pension Master into the MS Word template document to create the finished letter.

Anyone familiar with the mail merge facility of MS Word will have no problem creating mail merged letters from Pension Master.

The following options and screen appear when this program is selected. Certain menu options may only be visible when you have selected a mail merge report using the ‘Query’ option:

Add A New Mail Merge Report

When adding a new mail merge report, you should first of all create an empty Microsoft Word document that you wish to eventually be your mail merge template document. As part of the process of adding a mail merge report, Pension Master will automatically convert this blank document into a mail merge template document and invite you to add text and/or graphics to it to create the finished document. This should be in a location that is accessible to all Pension Master users at your site e.g. a shared network drive.

Click ‘Add’.

You will be asked if you wish to copy an existing Mail Merge Report. If you select ‘Yes’ you will be prompted to select the Mail Merge Report to copy or select one from the lookup list.

You are then requested to enter the selection criteria required in order to select the members you wish to output to the report (see the Data Dictionary of this manual for further details on selection of records in Pension Master). When the selection criteria has been entered click on ‘OK’ to proceed to the report date field. When initially setting up a mail merge it is best to select just an individual member or at least a very small group of members.

You can click on ‘Cancel’ to return to the menu and abort report selection.

REPORT DATE - The report date defaults to today’s date, but it may be overwritten with any date in the format ddmmyy or dd/mm/yyyy. This date is used in conjunction with the DATE ENTERED COMPARISON field to include only those members who have joined the scheme before a certain date.

DATE ENTERED COMPARISON - The valid entries for this field are:

  • 1 - Include members whose Date Joined For Pension is less than or equal to the report date
  • 2 - Include members whose Date Entered Company is less than or equal to the report date
  • 3 - Include members whose Date Joined For Life Assurance is less than or equal to the report date
  • ’ ‘ - Include All members

DOCUMENT PATH - This is the name and full path of the mail merge template document you have set up for merging the data in this Mail Merge Report into. This field is used to indicate which template document to open for which letter when doing a mail merge. See Members for further details of how this works when merging letters/certificates directly from the member record. An example would be g:\docs\benstat.doc.

When adding a new mail merge report, you should create an empty Microsoft Word document that you wish to eventually be your mail merge template document.

If you are unsure of the name or location of your template document, press the Search button to display a pop-up browse for file window. Select the directory you require using the mouse, then enter a name for your new template document in the ‘File name’ field, then click on ‘OK’. The filename and path will automatically be displayed in the ‘Document Path’ field.

Your template file should be in a location that is accessible to all Pension Master users at your site e.g. a shared network drive. Otherwise you will find that only users that can access the template file can run mail merges for this letter/certificate.

MACRO AFTER MERGE - Tick this box if you want to run your own Microsoft Word macro in the form letters that are produced by the mail merge process. You may want to do this for example if you want to make sure the user saves the resulting form letters. If you are unsure what this option means, it is likely you will not need it and so should set this field to unticked.

MACRO NAME - Enter the name of your Microsoft Word macro that you wish to run after the mail merge process has completed.

USE FOR MEMBER LABELS? - Check this box if you want this mail merge report to be used to generate member labels.

Any selection criteria that is entered will be used if this report is run from the Mail Merge Report screen. But, if this report is run as a Member Label report then the Member Label report selection criteria will be used.

GROUP BY - These fields can be optionally used to specify how you wish to group the member records that are output to the report e.g. you may wish to group the records by date of birth. In this case you would enter pm_dob in the first GROUP BY field. The field entered must exist on the member file. If the field entered does not exist on the member file an error message will be displayed and the group by field should be re-entered. If you are unsure of the name of the field you wish to group by, simply click on the lookup icon for a list of fields.

When all groups have been entered click on ‘OK’, or press Return down through the remaining fields to proceed to the following screen.

If flag 2 is used as the 2nd argument for the Unit Prices table, then during the report run a popup window will appear that prompts the user to enter a date. Only one popup window appears per run, no matter how many flag 2 references are used throughout the report set up. A date must be entered in this field.

If the index for Member Address Details (pe_maddr) is 0 then the current address will be returned.

The speed of data extraction during the Mail-Merge Report can be improved in two ways:

Fields that belong to the same database table only have the user security check performed once. As soon as the table changes the security check is performed again. For example:

  1. pm_member
  2. ps_sal1
  3. pmq_current(CERT,1)
  4. pmu_fld1(INFO)
  5. pm_dob

will have the security check performed 5 times, for each time the source table changes.

  1. pm_member
  2. pm_forename
  3. pm_surname
  4. pm_sex
  5. pm_dob

will only have the security checked once for the Basic Member Details (pe_mem) table.

So to improve performance it is advisable to group fields that belong to the same database table.

It is now possible to use a range of Quotation Indexes when extracting information from the Member Quotation Results (pe_mquote) table. Instead of listing 5 quotation lines individually:

  1. pmq_current(CERT,1)
  2. pmq_current(CERT,2)
  3. pmq_current(CERT,3)
  4. pmq_current(CERT,4)
  5. pmq_current(CERT,5)

it is faster to use a range:

  1. pmq_current(CERT,1-5)

To Output The Results Of A Particular Formula

To reference calculations in your report, enter the appropriate formula code in the format:

#calc(formula_code)

Where formula_code is a valid formula code for the scheme implied by the current selection criteria. If an invalid formula code is entered, you will be warned of this and invited to re-enter a valid formula code.

If you are unsure of a particular formula code, type at least the # of the #calc and then click on the lookup icon to see a list of formula codes for the current implied scheme. Upon selecting a formula code, the report field will automatically be filled in with the correct #calc(formula_code) syntax e.g. #calc(092).

The field must be set up on the Report Units file (see Member Report Units) or can be a direct link to a valid formula for the current scheme. If the field entered does not exist on the report units file an error message will be displayed and the field-name must be re-entered.

If you attempt to click on ‘OK’ to save the report whilst still in the last field and the last field of the report is blank, a message will be displayed asking you if you wish to save the report and run it immediately, or return to input another field.

When all fields have been entered click on ‘OK’, to proceed to the following screen.

The third screen is displayed as follows:

Mail Merge Output Settings

EMAIL ADDRESS - You should have your email address in the email address field; if it is not present you will need to add this in the “Users” program. (For larger mail merges this is used to notify the user that the mail merge has completed).

PDF/WORD DOCUMENT - Select from the drop-down list whether you want the final document in PDF or DOC format.

SPILT THE DOCUMENT - If you wish the final mail merge document to be split into individual documents you should check the “Split Document” check box. The remaining text boxes and drop-down lists on this screen should now become available and they determine the naming convention of the final merged documents. If you don’t click ‘OK’ to proceed.

When all fields have been entered click on ‘OK’ to process the report. A message will be displayed:

Processing report … Please Wait

You will be asked if you wish to save your report. If you wish to save the report for later use press Y, or click on the ‘Yes’ menu option. A window will be displayed:

Save Report Screen

REPORT CODE - Enter a short alphanumeric code of up to 8 characters to save your report under. If the report code already exists on the report file for the current scheme then you will be asked if you want to overwrite it, or enter another report code.

SCHEME CODE - Enter the scheme code for this report. This is the scheme code that corresponds to the scheme you wish to use this report for. You should note, however, that this will not preclude you using this report for other schemes if you so wish. The scheme code must be a valid code on the scheme file and if it is not, an error message will be displayed.

REPORT DESCRIPTION -The report description must be entered. This is a more lengthy description for your report which you can use to identify the report for future use e.g. “Standard Leaver Letter”.

When the report description has been entered click on ‘OK’ to save the report to file, or click on ‘Cancel’ to cancel saving the report to file.

If you do not wish to save the report click ‘No’.

Along the top of the window click the button “Download Template” button Microsoft Word will now be automatically started and the mail merge data file created.

You should now design your letter/certificate using all the features and abilities of Microsoft Word you need. When you wish to insert a data field from Pension Master at the current cursor location point, click on the ‘Insert Merge Field’ button in Microsoft Word and select the appropriate field. A lookup of the field names is available in the Data Dictionary.

When you have finished designing your template file, click on File > Save to save your template file and then exit Microsoft Word.

The options for printing are described in the document Printing From Pension Master.

Search For An Existing Mail Merge Report

Click ‘Query’.

The following window will be displayed:

Report Selection Screen

Enter the selection criteria for the report and click on ‘OK’, or Return when in the report description field. Pension Master defaults to selecting all available reports for the scheme implied by the current selection criteria, which is usually the scheme the current member is in.

All reports that match the selection criteria will be displayed. If more than 10 records are selected click on the ‘Next Record’ button to view the next screen of reports, and click on the ‘Prev Record’ button to view the previous screen.

Using the arrow keys, go to the report that is required and click on ‘OK’. The window will close and the report details saved for that report will be displayed automatically. Enter the selection criteria and click on ‘OK’. If no records are found to match the selection criteria an error message will be displayed.

You will then be returned to the main menu and can select the appropriate menu option as required.

Re-Run An Existing Mail Merge Report

Click ‘Run’.

You must have selected a report to run using the ‘Query’ menu option prior to selecting the ‘Run’ menu option. If you have not done this then an error message will be displayed and you will be returned to the main menu.

To process the report click on ‘OK’. A message will then be displayed:

Processing report … Please Wait

Once finished you will click the “Download Merged Document” button which can be found along the top of the screen.

If the number of members processed falls beneath a pre-determined number (this will usually be no more than 10 members) then the completed merged document will be opened and presented to the user. If the numbers exceeds this limit then you will be advised that the mail merge is being processed and you will be notified by email once it is completed.

Microsoft Word will now be automatically started and the mail merge data file created. The mail merge process will then be completed, with the mail merge data being inserted into the template document for this letter/certificate. The resulting merged document will then be displayed on the screen.

If, for some reason, the mail merge template file (as indicated by the ‘Document Path’ field) has been changed e.g. so that it now contains different data field tags than the current Mail Merge Report, Microsoft Word will complain when trying to do the mail merge as it may find data fields in the template document that are not in the mail merge data file. If this is the case, immediately exit Microsoft Word and use the edIt option in the Mail Merge Report to adjust your template document as required. This is only likely to happen if you decide to edit your template files directly in Microsoft Word rather than using the template document creation and amendment facilities within the Mail Merge Report.

Once you have finished with your merged document, exit Microsoft Word.

Edit An Existing Template Document

Click ‘edIt’.

You must have selected a report to run using the ‘Query’ menu option prior to selecting the ‘Run’ menu option. If you have not done this then an error message will be displayed and you will be returned to the main menu.

This option allows you to amend your template documents in Microsoft Word in a more controlled way that just amending the document directly in Word itself. We would recommend you use the edit option to do all changes to your document templates as this avoids potential confusion if amending documents directly in Word.

You must select at least one member record to output data for. This is then used to create a dummy mail merge data file that will be attached to your template document so you have the correct mail merge fields available to you when editing your template document.

The report details can be overwritten if required, or click on ‘OK’ to go on to the next screen.

The fields saved for this report will be displayed to the second screen. These fields may be overwritten if required.

To add a field in the middle of the report click on the ‘Add Row’ button, and enter the fieldname.

To delete a field in the middle of the report click on the ‘Delete Row’ button.

To append extra fields to the end of the report go to the next blank field and enter extra fields.

To process the report click on ‘OK’. A message will then be displayed:

Processing report … Please Wait

You will be asked if you wish to save your report. If you wish to save the report for later use press Y, or click on the ‘Yes’ menu option. The report will be saved with the code and description that it was originally saved with.

If you do not wish to save the report click ‘No’.

Once finished you will click the “Download Merged Document” button which can be found along the top of the screen.

If the number of members processed falls beneath a pre-determined number (this will usually be no more than 10 members) then the completed merged document will be opened and presented to the user. If the numbers exceeds this limit then you will be advised that the mail merge is being processed and you will be notified by email once it is completed.

Microsoft Word will now be automatically started in the background and the mail merge data file created. The mail merge template file (as specified in the ‘Document Path’ field) is then loaded into Microsoft Word and displayed to your screen. You should then edit the template document as required, adding additional fields if necessary using the ‘Insert Merge Field’ button within Microsoft Word.

If you wish numeric items that are merged into your document to appear in certain formats other than the defaults, you can use the MS Word field formatting as follows: With the mail merge template document open in Word, press Alt-F9. This expands the details of your mail merge fields e.g. {pds_grsal} becomes {MERGEFIELD pds_grsal}. If you wanted salaries to appear, for example, with commas in (e.g. 3,000.00), amend the {MERGEFIELD pds_grsal} to {MERGEFIELD pds_grsal# #,##0.00}. Further details of the options available can be found in the MS Word on-line help text.

Once you have finished with your template document, exit Microsoft Word.

The program will return to the main menu.

Delete A Mail Merge Report

Click ‘Delete’.

You must have selected a report to delete using the ‘Query’ menu option prior to selecting the ‘Delete’ menu option. If you have not done this then an error message will be displayed and you will be returned to the main menu.

This option allows you to delete an existing Mail Merge Report from Pension Master.

When the delete option is selected a window is displayed with the options yes and no.

If you do not wish to delete the report click ‘No’. You will return to the main menu.

If you wish to delete the report click ‘Yes’. Another window is displayed asking if you are sure.

If you do not wish to delete the report click ‘No’. The program returns to the main menu.

If you wish to delete the report, click ‘Yes’. The Member Report is deleted, and the program returns to the main menu.

The mail merge template document as specified in the ‘Document Path’ field is not deleted when you delete a Mail Merge Report as it is concievable that you may wish to keep this document for other uses. If this is not the case, you should now manually delete this template document using standard Windows tools e.g. Windows Explorer.
