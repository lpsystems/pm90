Introduction

This program produces a report on any database table columns selected by the user. Whilst the user enters details into the screens, an SQL statement is being produced which will then be used in running the report. You can:

  • Select the columns you wish to appear on the report.
  • Define a report title which will appear at the top of each page of your report.
  • Select how to group the data on the report e.g. by scheme.
  • Select which numeric data fields you wish to total up and how you wish the total to be applied to the data.
  • Select whether you wish the report to be output in report format or comma-delimited data format (.CSV format) and exported to Microsoft Excel. The .CSV format is suitable for importing into other applications like spreadsheets.
  • Define the headings you wish to appear for each column of data output to your report.
  • Define the layout of the report.

The following options and screen appear when this program is selected. Certain menu options may only be visible when you have selected a report using the ‘Query’ menu option.

  • Query

Field Selection Screen

Add A New Report

Click ‘Add’.

The following window will be displayed:

Add Report Options

Copy an existing report? - Checking this box will allow you to copy an existing Report rather than input the complete report from scratch. You will then be able to edit the report as required and save it using a different name.

Enter the report code of the report you wish to copy. You can also click on the lookup icon to see a list reports. Once this has been done, the window closes and you will be returned to the report Code field for your new report.

Export to Microsoft Excel? - Checking this box will create a Microsoft Excel spreadsheet on your PC that will contain the selected data.

The report will produce a data export file in .CSV (comma separated value) file format and also export the data directly into a Microsoft Excel spreadsheet on your PC.

Details of where the data file can be found is set out in the Where To Find The Resulting Exported Data File section later in this document.

Print column headings on your exported data? - If you have chosen to export your report data, you will then be able to check this box if you wish to print column headers on the exported data. If you are doing a data extract for electronic transmission onto another software system, you would normally not export column headings and just send a block of data which can then be easily imported into other software systems. If, however you wish to export the data to an Excel spreadsheet that can be easily understood by another user we would recommend that you export column headings. This box is defaulted to a tick.

If you choose not to output the report in export format, the report will be produced in normal report format and be available to view/print in the same way as all other reports in Pension Master.

You are then requested to enter the report header details you are going to save the report under.

SCHEME CODE - Enter the scheme code you wish to save the report under. You can click on the lookup icon for a list of valid scheme codes. This field is mandatory.

REPORT CODE - Enter the report code you wish to save the report under. If another report already exists for the same combination of scheme code and report code, the user is informed of this and will be prompted to enter a different report code. This field is mandatory.

REPORT DESCRIPTION - Enter the report description you wish to save the report under. This field is mandatory.

You are then requested to enter at least one line of data in the history at the bottom of the screen.

FIELD DESCRIPTION - This denotes either a field description you wish to appear on the report or the result of a particular formula output to the report. Which one is dependant on how you enter the field:

To Output A Particular Field

If you wish to enter a field description, for example Member Number off the Basic Member Details table, then you can do so via a number of ways.

  • Enter the field as pm_member, which will then automatically be altered to the wordy description of Member Number.
  • Enter the wordy description directly into the field. In the case of Member Number and some other popular columns, a pop-up window may be displayed to the screen with a list of database tables, all of which contain the same wordy description for one of their columns. You are then prompted to choose which database table the Member Number is referring to.
  • You can click on the lookup icon to see a list of all the available fields. This will return the wordy description of the field chosen, back to the Field Description field.

To Output The Results Of A Particular Formula

To reference calculations in your report, enter the appropriate formula code in the format:

#calc(formula_code,width)
  • where formula_code is a valid formula code for the scheme implied by the current selection criteria. If an invalid formula code is entered, you will be warned of this and invited to re-enter a valid formula code.
  • where width is a number that is between 10 and 100. The minimum width of the result of a formula is 10, this is the width of the standard date format (dd/mm/yyyy). The maximum width, 100, is the longest character string that can be produced by a formula.

If you are unsure of a particular formula code, type at least the # of the #calc and then click on the lookup icon to see a list of the available formula codes for the scheme code entered in the report header. Upon selecting a formula code, the report field will automatically be filled in with the correct wordy description ‘#formula_code description’ syntax e.g. if #calc(092,10) is selected then the description ‘#Total RGMP’ is displayed to the field. The # will determine that a formula has been selected. The width of the formula result is defaulted to 10.

The ‘#calc’ facility can only be used when the Member Number has also been selected from the Basic Member Details table (pe_mem.pm_member). If the Member Number has not been selected and you try to enter a ‘#calc’ formula line, the user will be informed of this and will not be allowed to continue to enter the #calc line, until Member Number has been selected.

The field must be set up on the Report Units file (see Member Report Units) or can be a direct link to a valid formula for the chosen scheme code. In the case of a field, if it does not exist on the report units file an error message will be displayed and the field-name must be re-entered. Click on the lookup icon to see a list of the available fields for the report. In the case of a formula, you can click on the ‘Formula’ button whilst in the field and the Formula program will be brought up to the screen showing the details of the formula selected.

When a field is entered a message will be displayed to the bottom of the screen:

Report Width : xx

INCLUDE ON REPORT - Tick the box if you wish to include the field description entered on the report. You cannot entered a tick in this field is the corresponding field description is blank.

PRINT TOTAL - Tick the box if you wish to print a total of the field description at the end of the report. You can only tick this field if the field description is entered, the include on report field is ticked and the field description is of a numeric/decimal type.

SORT ORDER - Enter a numeric value between 1 and 6 in this field. You can only enter a value in this field if the field description is entered. A maximum of 6 columns can be used to sort data by, per report.

ASC/DESC - Valid values for this field are ‘A’ - Ascending, and ‘D’ - Descending. This field can only be entered if the field description is entered, and the Sort Order field is entered. This field defaults to ‘A’ - Ascending, when a value has been entered in the Sort Order field.

The details entered into this screen will generate the first part of the SQL statement ie. the ‘SELECT’ section.

When all details have been entered on this first screen click on ‘OK’ to proceed to the following screen.

If you are producing the report in Report and not Export format, the width of the report should not go over 180 characters. If the report width goes over 180 characters, an error message will be displayed and you should then remove any fields that you do not require from your report so that the total report width is less than 180 characters.

If you are producing the report in Export format, then the width of a field should not go over 255 characters. This is because Microsoft Excel cannot cope with fields wider than this amount. If the field width goes over 255 characters, then the field’s data will be cut down to 255 characters before it gets sent to Microsoft Excel.

Between Screens Selection

Before the second screen is displayed to the screen, the program will work out the start of the SQL statement to be used in the report. The database tables to be used in the statement will be determined and any auto-generated lines to link tables with one another will be created to then be displayed to screen 2.

If at least one formula line has been entered in screen 1, then a line defaulting the calculation date to TODAY, will also be auto-generated for screen 2.

Criteria Selection

Press 2, or select the ‘Criteria Selection’ from the ‘Menu’ menu option with the arrow keys and press Return. If you are in the process of generating a new report, then this screen automatically gets displayed after you have entered details in screen 1.

The second screen is displayed as follows:

Criteria Selection Screen

AND/OR CLAUSE - Enter either ‘AND’ or ‘OR’ into this field. The first line of the history will not allow the user to enter into the field due to the first line going into the SELECT statement from this screen will always use the ‘WHERE’ clause. The default value for this field is ‘AND’.

BRACKET 1 - This field can take the value of either blank or ‘(‘. If a ‘(‘ is entered then ensure that a corresponding ‘)’ bracket is entered in Bracket 3 or 4 field on one of the lines to follow.

BRACKET 2 - This field can take the value of either blank or ‘(‘. If a ‘(‘ is entered then ensure that a corresponding ‘)’ bracket is entered in Bracket 3 or 4 field on one of the lines to follow.

FIELD 1 - This denotes either a field description or an actual value which you wish to compare the Field 2 value against.

If a field description is to be entered then it must be from a database table already used in the screen 1 selection. If you are unsure of which tables have been selected, then you can click on the lookup icon for a list of the database tables already used. If you know which tables are in use, then you can enter the field description using the same method used in the Field Description field on screen 1.

If an actual value is to be entered then dependant on the value’s type ie. character, date, numeric or decimal, it may need to be in single, double or no quotations:

Value Type Quotations Example
Character Single quotes ‘’ ‘hello’
Date Double quotes “” “20000101”
Numeric No quotes 12345
Decimal No quotes 1200.00

OPERATOR - Enter one of the following valid operators:

Operator Name
EF Effective at
EQ Equal to
F First entry
GT Greater than
GE Greater than or equal to
L Last entry
LT Less than
LE Less than or equal to
MA Matches
NH No history
NE Not equal to
NM Not matches

There is no limit on the number of operators used within a report, except for the No History operator. The No History operator can only be used once in a report due to it’s complex method of retrieving data from the database tables.

For more explanation on how each operator works, please click on the link.

FIELD 2 - This denotes either a field description or an actual value which you wish to compare the Field 1 value against. The same method used in Field 1 for entering a value, also applies to Field 2.

BRACKET 3 - This field can take the value of either blank or ‘)’. If a ‘)’ is entered then ensure that a corresponding ‘(‘ bracket is entered in Bracket 1 or 2 field on one of the lines previous.

BRACKET 4 - This field can take the value of either blank or ‘)’. If a ‘)’ is entered then ensure that a corresponding ‘(‘ bracket is entered in Bracket 1 or 2 field on one of the lines previous.

The program checks that the value type entered in Field 1 is also entered in Field 2. For example, if Member Number is selected in Field 1 then a numeric value should be entered in Field 2. If these two value types are not the same then the user is informed of this and is prompted to alter one of the values entered.

When a No History operator is used, the AND/OR clause is defaulted to ‘AND’, and brackets 1 and 2 are defaulted to blank.

Setting up Report Column Headings

Click ‘Screen 3’.

The following window will be displayed:

Report Column Headings

The field descriptions which are to be included in the report from screen 1, are listed in this screen. The user cannot amend the value entered in Field, as these are defaulted from the field descriptions in screen 1. If you do not want any of these fields to be reported on, then you must untick them from screen 1.

HEADING 1 - Enter the first line of column heading for the field that is to be output to the report.

HEADING 2 - Enter the second line of column heading for the field that is to be output to the report.

When a new report is being generated, the column headings default to the appropriate values that have been set up in the Member Report Units option and can be overwritten by the user. These are then saved with the report for future use.

Report Layout Settings

Click ‘Screen 4’.

The following window will be displayed:

Report Layout Settings

When a new report is being generated, the fields in this screen will be set to their defaults as described below:

REPORT TITLES - Enter the report heading. The default for the top line is ‘PENSION MASTER - ADVANCED REPORT GENERATOR’

NUMBER OF BLANK LINES AFTER EVERY ROW - Enter the number of blank lines you wish to print after every row. The default value for this field is 0.

BLANK LINE PRINTED AFTER NUMBER OF ROWS - Enter the number of rows you wish to print before a blank line is printed. The default value for this field is 0.

Report Group Settings

Click ‘Screen 5’.

The following window will be displayed:

Report Group Settings

The field descriptions which have been selected to order the data by on screen 1, will be listed in this screen. The user cannot amend the value entered in Field, as these are defaulted from the field descriptions in screen 1. If you do not want any of these fields to be used in ordering the data, then you must de-select them from screen1. Enter the remaining fields as follows:

TOTAL - Tick the box if you wish to total up the numeric values ticked on screen 1 as being ‘Included on report’, after each group of data.

HEAD - Tick the box if you wish to print a heading of the group, before each group of data.

COUNT - Tick the box if you wish to print a count of the number of records in each group, after each group of data.

PAGE BREAK - Tick the box if you wish to have a page break, after each group of data. N.B. This option does not work when reporting out to Excel.

Update An Existing Report

Click ‘Update’.

Select the screen you wish to update by selecting it from the ‘Menu’ menu option with the arrow keys and pressing Return, or by using the hot-keys 1 thru to 5 respectively.

If you update screen 1, then on clicking ‘OK’ the auto-generated links for screen 2 are checked and updated. This is in case any database tables have been added or removed from the original selection.

Click on ‘OK’ to save the alterations you have made.

Search For An Existing Report

Click ‘Query’.

The following window will now be displayed:

Report Selection Screen

Enter the selection criteria for the report and click on ‘OK’, or Return when in the report description field. All report which match the selection criteria will be displayed.

Using the arrow keys, go to the report that is required and click on ‘OK’ to select it.

You will then be returned to the main menu and can select the appropriate menu options as required.

Re-Run An Existing Report

Click ‘Run’.

You must have selected a report to run using the ‘Query’ menu option prior to selecting the ‘Run’ menu option. If you have not done this an error message will be displayed and you will be returned to the main menus.

You are now prompted as to whether you wish the output of the report to be in export file format.

When the report is being run, it will produce a data export file in .CSV (comma separated value) file format and also export the data directly into a Microsoft Excel spreadsheet on your PC.

Details of where these data files can be found is set out in the Where To Find The Resulting Exported Data File section later in this document.

If you have chosen to export your reported data, then you can the tick the Print Column Headers box if you wish to print column headers on the exported data. If you are doing a data extract for electronic transmission onto another software system, you would normally not export column headings and just send a block of data which can then be easily imported into other software systems. If, however, you wish to export the data to an Excel spreadsheet that can be easily understood by another user, then we would recommend that you export column headings.

If you choose not to output the report in export format, the report will be produced in the normal report format and be available to view/print in the same way as all other reports in Pension Master.

To process the report, click on ‘OK’. A message will be displayed:

Processing report … Please Wait

If you have selected the eXport option from the menu, Microsoft Excel will be started automatically and you data placed into a fresh spreadsheet within Excel. The filename for this spreadsheet is c:\penmast.xls. Once you have finished with this spreadsheet, exit Microsoft Excel to return control to the Advanced Report Generator program.

Pension Master will automatically overwrite the file c:\penmast.xls with the new data export if you run the Advanced Report Generator with the eXport menu option. Any data that is currently in that file will thus be overwritten. If you wish to keep this data, we suggest you manually copy the file c:\penmast.xls to another directory prior to running your data export.

Delete An Advanced Report

Click ‘Delete’.

You must have selected a report to delete using the ‘Query’ menu option prior to selecting the ‘Delete’ menu option. If you have not done this then an error message will be displayed and you will be returned to the main menu.

The option allows you to delete an existing Advanced Report from Pension Master.

When the delete option is selected, you will be asked to confirm that you want to delete the report.

If you do not wish to delete the report click on ‘No’.

If you do wish to delete the report click on ‘Yes’. You will be asked again to make absolutely certain that you do wish to delete the report.

If you still do wish to delete the report then click on ‘Yes’ and the report will be permanently removed from Pension Master. If you have any doubt about deleting the report, click on ‘No’.

Browsing Report Details

Click on the ‘Browse’ button or the Browse icon on the toolbar to view details in any screen where the data does not fit on the screen.

Once in browse mode, use the arrow keys or the scrollbar on the right of the data to navigate around the data.

Click on ‘OK’ or ‘Cancel’ to leave browse mode.

Where To Find The Resulting Exported Data File

The .CSV data file will be created as c:\penmast.csv. The Excel spreadsheet will be created as c:\penmast.xls.

Full Query

Click ‘Full Query’.

The following window will now be displayed:

Full Query Pop-up Screen

This screen displays the SQL statement as it is being generated.

When the window is first opened the SQL statement will be shown to the screen in it’s wordy description format. If you use the ‘Toggle’ button, the wordy description will change to the coded version of the statement. For example, if SELECT {Member Number} FROM {Basic Member Details} is displayed to the screen, then on clicking on ‘Toggle’ the example would be displayed as SELECT {pm_member} FROM {pe_mem}.

Click on the ‘Toggle’ button again and it will change the SQL statement back to the wordy description format.

Exit Program

Click ‘Exit’. This leaves the program and closes the current browser tab.
