Introduction

This program produces a report of member details as selected by the user. You can:

  • Select the group of members you wish to appear on the report.
  • Define a report title which will appear at the top of each page of your report.
  • Select how to group the member data on the report e.g. by date of birth.
  • Select which data fields you wish to appear on the report.
  • Select which numeric data fields you wish to total up and how you wish the total to be applied to the data.
  • Select whether you wish the report to be output in report format or comma-delimited data format (.CSV format) and exported to Microsoft Excel. The .CSV format is suitable for importing into other applications like spreadsheets.
  • Define the headings you wish to appear for each column of data output to your report.
  • Produce a list of all Member Reports on Pension Master for either a specific scheme or all schemes currently on Pension Master.

The following options and screen appear when this program is selected. Certain menu options may only be visible when you have selected a member report using the ‘Query’ menu option:

  • Add
  • Query
  • List
  • Exit

Member Report Main Screen

Add A New Report

Click ‘Add’

The following window will be displayed:

Add Report Options

Copy an existing report? - Checking this box will allow you to copy an existing Member Report rather than input the complete report from scratch. You will then be able to edit the report as required and save it using a different name.

Enter the report code of the report you wish to copy. You can also click on the lookup icon to see a list of reports. Once this has been done, the window closes and you will be returned to the report Code field for your new report.

Export to Microsoft Excel? - Checking this box will create a Microsoft Excel spreadsheet on your PC that will contain the selected data.

The report will produce a data export file in .CSV (comma separated value) file format and also export the data directly into a Microsoft Excel spreadsheet on your PC.

Details of where these data files can be found is set out in the Where To Find The Resulting Exported Data File section later in this document.

Print column headings on your exported data? - If you have chosen to export your report data, you will then be able to check this box if you wish to print column headers on the exported data. If you are doing a data extract for electronic transmission onto another software system, you would normally not export column headings and just send a block of data which can then be easily imported into other software systems. If, however you wish to export the data to an Excel spreadsheet that can be easily understood by another user we would recommend that you export column headings.

If you choose not to output the report in export format, the report will be produced in normal report format and be available to view/print in the same way as all other reports in Pension Master.

You are then requested to enter the selection criteria required in order to select the members you wish to output to the report (see Selection Criteria). When the selection criteria has been entered click on ‘OK’ to proceed to the report date field.

If you wish to enter more specific selection criteria, click on the ‘Advanced Search’ button. The following screen is displayed:

Advanced Search Screen

Enter your advanced selection criteria using any combination of the fields in the first column of this screen and click on ‘OK’ to proceed. Click on ‘Cancel’ to exit from the Advanced Search window.

You will now be moved to the right hand column of this window. This is where you can enter details that will specify how salary, contribution, unit or NI history colums are output to your report. Each of the four sections is the same so let’s take the case of salary history:

SALARY SELECT - This determines how salaries are extracted from the members salary history record as follows:

  • LA - last salary on the history record i.e. most recent salary.
  • FI - first salary on the history record i.e. oldest salary.
  • AT - salary at a date.
  • NH - no salary at a date.
  • TD - total to date i.e. total the salary up to and including the date entered.
  • ED - effective at date i.e. the salary on or before the date entered.

DATE - The date you wish to select salary at. This is only relevant for SALARY SELECT options AT, NH, TD and ED.

The following two fields will allow members to be selected that had a specific Status Code at a certain date.

HISTORIC STATUS CODE - Enter the Historic Status Code required. Individual or a range of Status Codes can be entered as described in Selection Criteria.

HISTORIC STATUS DATE - The Status Date at which the Status Code was effective.

Click on ‘OK’ to proceed or click on ‘Cancel’ to exit from the Advanced Search window and abort report selection.

REPORT DATE - This defaults to today’s date, but it may be overwritten with any date in the format ddmmyy or dd/mm/yyyy. This is used in conjunction with the DATE ENTERED COMPARISON field to only include those members who have joined the scheme before a certain date.

DATE ENTERED COMPARISON - The valid entries for this field are:

  • 1 - Include members whose Date Joined For Pension is less than or equal to the report date.
  • 2 - Include members whose Date Entered Company is less than or equal to the report date.
  • 3 - Include members whose Date Joined For Life Assurance is less than or equal to the report date.
  • ’ ‘ - Include All members.

HEADINGS - Enter up to two lines that you wish to use as headings for your report. If the first heading is left blank the default heading is the name of the scheme implied by the report selection criteria. If the second heading is left blank the default heading is “Membership Report”.

GROUP BY - These fields can be optionally used to specify how you wish to group the member records that are output to the report e.g. you may wish to group the records by age. In this case you would enter pm_age in the first GROUP BY field. The field entered must exist on the member file. If the field entered does not exist on the member file an error message will be displayed and the group by field should be re-entered. If you are unsure of the name of the field you wish to group by, click on the lookup icon for a list of the available fields.

TOTAL - This indicates that a total of a field/fields specified is required after the group.

e.g. If total is ticked for group by pm_scheme, after the members in the Scheme have been printed a total line will be printed as follows:

--------------------------------------------------------------

                                           999999999 999999999

--------------------------------------------------------------

HEADING - This indicates that a heading will precede the group on the report.

e.g. If heading is ticked for group by pm_scheme then a heading will be printed as follows before the members in the Scheme are printed.

Scheme Code : Scheme name

COUNT - This indicates that a count of members in a group is required after the members in the group have been printed.

e.g. If there were 23 members in a scheme and count is ticked for group by pm_scheme the following line will be printed after all members in the scheme have been printed:

Total number of members in Scheme : 23

SEQUENCE - The valid entries for this field are ‘A’ for Ascending and ‘D’ for Descending. The default is ‘A’. If any other value is entered in this field an error message will be displayed. This specifies that a group be sorted into ascending or descending order as required.

e.g. If a report is created, using both the scheme and members age as groups. The age could be sorted into descending sequence to produce a report listing the oldest members first for each scheme.

When all groups have been entered click on ‘OK’, or press Return down through the remaining fields to proceed to the following screen.

To Output The Results Of A Particular Formula

To reference calculations in your report, enter the appropriate formula code in the format:

#calc(formula_code,width)

  • where formula_code is a valid formula code for the scheme implied by the current selection criteria. If an invalid formula code is entered, you will be warned of this and invited to re-enter a valid formula code.
  • where width is a number that is between 10 and 100. The minimum width of the result of a formula is 10, this is the width of the standard date format (dd/mm/yyyy). The maximum width, 100, is the longest character string that can be produced by a formula.

If you are unsure of a particular formula code, type at least the # of the #calc and then click on the lookup icon to see a list of formula codes for the current implied scheme. Upon selecting a formula code, the report field will automatically be filled in with the correct #calc(formula_code,width) syntax e.g.

#calc(092,10). The width of the formula result is defaulted to 10 but may be changed as required.

The field must be set up in Member Report Units or can be a direct link to a valid formula for the current scheme. In the case of a field, if it does not exist in Member Report Units an error message will be displayed and the fieldname must be re-entered. Click on the lookup icon to see a list of fields for the report.

When a field is entered a message will be displayed:

Report Width : xx

TOTAL - This denotes whether you wish a total to appear for this field and is only relevant for numeric fields. If the field is numeric this field may be ticked to specify that this is a field to be included in the totals line which was specified in the group by section. If the field is ticked for a character or date field the field will be unticked.

e.g. if you have entered pm_scheme in the GROUP BY field on the first screen with a tick against the corresponding total line, then if you have a field pds_grsal on your report with a tick against the TOTAL field, when your report is produced, after each scheme the report will total up the gross salary column and print the total on the report.

The third screen is displayed as follows:

Heading Selection Screen

HEADING 1/ HEADING 2 - Enter up to two lines of column headings for each field that is to be output to the report. The column headings default to the appropriate values that have been set up in Member Report Units and can be overwritten by the user. These are then saved with the report for future use.

If you are producing the report in Report and not Export format, the width of the report should not go over 180 characters. If the report width goes over 180 characters, an error message will be displayed and you should then remove any fields that you do not require from your report so that the total report width is less than 180 characters.

When all fields have been entered click on ‘OK’ to process the report or click on ‘Cancel’ to abort without processing your report. A message will be displayed:

Processing report … Please Wait

If you have opted to export your data, Microsoft Excel will be started automatically and your data placed into a fresh spreadsheet within Excel. The filename for this spreadsheet is c:\penmast.xls. Once you have finished with this spreadsheet, exit Microsoft Excel to return control to the Member Report program.

Pension Master will automatically overwrite the file c:\penmast.xls with the new data export if you run the Member Report with the export option. Any data that is currently in that file will thus be overwritten.

If you wish to keep this data, we suggest you manually copy the file c:\penmast.xls to another directory prior to running your data export.

When the report has been processed you will be asked if you wish to save this report for later use.

REPORT CODE - Enter a short alphanumeric code of up to 8 characters to save your report under. If the report code already exists on the report file for the current scheme then you will be asked if you want to overwrite it, or enter another report code.

SCHEME CODE - Enter the scheme code for this report. This is the scheme code that corresponds to the scheme you wish to use this report for. You should note, however, that this will not preclude you using this report for other schemes if you so wish. The scheme code must be a valid code on the scheme file and if it is not, an error message will be displayed. If you are unsure of the scheme code, click on the lookup icon for a list of available schemes.

REPORT DESCRIPTION -The report description must be entered. This is a more lengthy description for your report which you can use to identify the report for future use e.g. “Basic Membership Listing”.

When the report description has been entered click on ‘OK’ to save the report to file, or click on ‘Cancel’ to cancel saving the report to file.

If you do not wish to save the report, click on the ‘No’ menu option .

The options for printing are described in the document Printing From Pension Master.

Search For An Existing Report

Click ‘Query’.

The following window will now be displayed:

Report Selection Screen

Enter the selection criteria for the report and click on ‘OK’, or Return when in the report description field. Pension Master defaults to selecting all available reports for the scheme implied by the current selection criteria, which is usually the scheme the current member is in.

All reports that match the selection criteria will be displayed. If more than 10 records are selected click on the ‘Next Record’ button to view the next screen of reports, and click on the ‘Prev Record’ button to view the previous screen.

Using the arrow keys, go to the report that is required and click on ‘OK’ to select it.

If the existing report used Advanced Selection Criteria then this window will automatically displayed. Otherwise the main Member Report screen will be shown. Enter the selection criteria for members as required and click on ‘OK’. If no member records are found then an error message will be displayed.

You will then be returned to the main menu and can select the appropriate menu option as required.

Re-Run An Existing Report

Click ‘Run’.

You must have selected a report to run using the ‘Query’ menu option prior to selecting the ‘Run’ menu option. If you have not done this then an error message will be displayed and you will be returned to the main menu.

You are now prompted as to whether you wish the output of the report to be in export file format.

The report will produce a data export file in .CSV (comma separated value) file format and also export the data directly into a Microsoft Excel spreadsheet on your PC.

Details of where these data files can be found is set out in the Where To Find The Resulting Exported Data File section later in this document.

If you have chosen to export your report data, you will then be asked if you wish to print column headers on the exported data. Answer ‘Yes’ or ‘No’ as appropriate. If you are doing a data extract for electronic transmission onto another software system, you would normally not export column headings and just send a block of data which can then be easily imported into other software systems. If, however you wish to export the data to an Excel spreadsheet that can be easily understood by another user we would recommend that you export column headings.

If you choose not to output the report in export format, the report will be produced in normal report format and be available to view/print in the same way as all other reports in Pension Master.

Processing report … Please Wait

If you have selected the eXport option from the menu, Microsoft Excel will be started automatically and your data placed into a fresh spreadsheet within Excel. The filename for this spreadsheet is c:\penmast.xls. Once you have finished with this spreadsheet, exit Microsoft Excel to return control to the Member Report program.

Pension Master will automatically overwrite the file c:\penmast.xls with the new data export if you run the Member Report with the eXport menu option. Any data that is currently in that file will thus be overwritten.

If you wish to keep this data, we suggest you manually copy the file c:\penmast.xls to another directory prior to running your data export.

Update An Existing Report

Click ‘Update’.

You must have selected a report to run using the ‘Query’ menu option prior to selecting the ‘Update’ menu option. If you have not done this then an error message will be displayed and you will be returned to the main menu.

The report advance selection ciretia can be overwritten if required, or click on ‘OK’ to go on to the next screen. If you need to make changes to the fields saved in this report tick screen 2. If the change is to the column headings tick screen 3.

The fields saved for this report will be displayed to the second screen. These fields may be overwritten if required by pressing update in that screen. Each screen is dealt with individually.

To add a field in the middle of the report click on the ‘Add Row’ button, and enter the fieldname.

To delete a field in the middle of the report click on the ‘Delete Row’ button.

To append extra fields to the end of the report go to the next blank field and enter extra fields. To process the report click on ‘OK’. A message will be displayed:

When you have made your changes to the screen press ok. To rerun the report with the changes simply press ‘Run’.

Delete A Member Report

Click ‘Delete’.

You must have selected a report to delete using the ‘Query’ menu option prior to selecting the ‘Delete’ menu option. If you have not done this then an error message will be displayed and you will be returned to the main menu.

This option allows you to delete an existing Member Report from Pension Master.

When the delete option is selected you will be asked to confirm and re-confirm that you wish to delete the current Member Report.

List Reports Currently Set Up

Click on the ‘List’ menu option.

This allows you to produce a list of all the Member Reports that are currently on Pension Master, sorted by scheme code and report code.

The following screen is displayed:

Scheme Selection Screen

Enter the selection criteria to select the scheme or schemes you require for the report and click on ‘OK’ to proceed.

The following message is now displayed:

Processing report … Please Wait

Once the report has been produced, options are displayed for printing.

The options for printing are described in the document Printing From Pension Master.

Where To Find The Resulting Exported Data File

The data is written into a file, c:\penmast.csv, as a comma seperated values (CSV) formatted file.

Exit Program

Click ‘Exit’. This leaves the program and closes the current browser tab.
