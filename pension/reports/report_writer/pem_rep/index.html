Introduction

This program allows maintenance of the report units fields. These are fields specified for use in the Member Report, Mail Merge Report and in the calculation engine in Pension Master.

You should not normally need to amend these records and in any case should only do so under the supervision of a member of L&P Systems Limited support staff.

Supervisor level access is required.

The following options and screen appear when this program is selected:

  • Add
  • Query
  • Update
  • Delete
  • Report
  • Exit

Member Report Units Screen

Add a Record

Click ‘Add’.

TABLE NAME - Enter the Table Name. A table name that exists on Pension Master must be entered here (See the Data Dictionary). Click on the lookup icon to see a list of database tables. If a valid table name is not entered an error message will be displayed.

FIELD NAME - Enter the Field Name. A field name that is a part of the table must be entered here (See the Data Dictionary). Click on the lookup icon to see a list of fields on the selected database table. If a valid field name is not entered an error message will be displayed.

FIELD DESCRIPTION - Enter a field description. A field description must be entered. The field description entered will be displayed in the total and count lines in the members report and full members listing.

e.g. If the description entered for pm_scheme is ‘SCHEME’ the count line will be:

Total Members in SCHEME : xx

HEADING FIELDS - Enter the heading fields. Headings for the field may be entered or can be left blank. These are used as column headings when printing off a Member Report or a Full Members Listing using this field.

e.g. If the headings entered for pm_member are ‘Pen.’ (heading 1), ‘Ref.’ (heading 2) the heading over the column will be:

Pen.Ref.

COLUMN TYPE - The column type will default to ‘Character’, ‘Date’ or ‘Number’ depending on the field. Default column widths, decimal places and print masks will also be generated at this stage.

COLUMN WIDTH - Enter the column width. the value entered in this field is the number of columns that the field requires on the report.

e.g. For printing a date in the format dd/mm/yyyy 10 columns would be required.

If the headings entered require more columns than specified the column width will default to the number of columns required to print the headings.

DECIMAL PLACES - Enter decimal places required. This field is a reference field only and may be left blank.

PRINT MASK - Enter the print mask for the field. This field only applies to numeric and date fields.

Date field print masks should be entered as ‘dd/mm/yyyy’ or ‘dd/mm/yy’.

Numeric field print masks can be entered in a variety of ways.

e.g. For a value of 123.45 the following formats would produce:

’#####&.##’ 123.45
‘&&&&&&.##’ 000123.45
’######’ 123
‘&&&&&&&&&’ 000000123
’—–&.&&’ -123.45 ( Negative sign printed on output )
’##.##’ ***** ( print mask is too small for value )
123.45 ( Left-justified - no print mask )

SYS COL TYPE - Internal value used when Pension Master is running on databases other than Informix. L&P Systems staff use only.

SYS COL LENGTH - Internal value used when Pension Master is running on databases other than Informix. L&P Systems staff use only.

WRITEBACK ORDER - This field is used in the ‘Quotation Type Setup - Writebacks’ screen in Categories. The numeric value held in this field is used to order the list of fields displayed in the writeback screen. The writebacks concerned with this new field are Refund, Death, Transfer In and Transfer Out. The order of these fields are defaulted to the order they appear in the corresponding screens in Members.

NOT MEMBER REPORT - This field is used in the Member Report and Mail Merge Report programs. If this field is ticked then the database table/field cannot be used in either of these programs. All tables/fields listed are used in the Advanced Report Generator and General Data Load programs.

MANDATORY - This field is used in the General Data Load program. If this field is ticked, then it will indicate that a data item must be present in the datafile, being loaded into the database table, for this field. Obvious mandatory fields are ones such as member columns, dates, contribution types etc. Please make sure that these fields are ticked.

MEMBER SCREEN - Internal value used to ensure that the Member Report Unit has the same security characteristics as the appropriate Member screen. L&P Systems staff use only.

OUTER JOINS - Outer join details only need be entered if details from another table other than the Member File (pe_mem) are required to link with the details being selected. E.g. If pm_scheme was being selected and the scheme name (psc_desc) was required from the Scheme Table (pe_sch).

The outer join table name must exist in the pensions database, and must represent the table name of the file being selected to link in with the field name already specified (see the Data Dictionary).

The outer join field name must exist on the file specified for the outer join and represents the field required.

SELECTION CRITERIA - The selection criteria must represent the link between the field name required and the outer field and file names. The entry of selection criteria cannot be validated so care must be taken when entering the selection criteria.

Query a Record

Click ‘Query’.

Selection criteria may be entered in the table name and field name fields. If any match to the selection criteria is found then the first record will be displayed to the screen and a message displayed to the bottom of the screen.

Record 1 of 2

Viewing Records

To view the other records selected press N to get the next record, and P to view the previous record.

If N is chosen and there are no records after the current record an error message is displayed. If P is chosen and there are no previous records an error message is displayed.

Update a Record

Before Updating, a record must be selected through the Query option, and an error message is displayed if a record for update has not been selected.

Click ‘Update’.

The table and field names cannot be changed as the combination of table and field name is a key to the report units file.

When the changes have been made click on ‘OK’ to save the changes to file, or click on ‘Cancel’ to abort the update option without saving changes.

Delete a Record

Before Deleting, a record must be selected through the query option, and an error message is displayed if a record for deletion has not been selected.

Click ‘Delete’.

When the delete option is selected a new window will appear with th eoptions yes and no.

If you do not wish to delete the record, click ‘No’. You will return to the program menu.

If you wish to delete the record, click ‘Yes’.

The following menu is displayed:

If you do not wish to delete the record, click ‘No’. The program returns to the main menu.

If you wish to delete the record, click ‘Yes’.

If the field being deleted exists on any of the prepared report formats, it will be deleted from these report formats.

Printing Report Units Details

Click the ‘Report’ icon.

If records have been selected in the program before the report option was selected the following window is displayed.

Print Options Screen 1

If no records have been selected in the program before the report option was selected the following window is displayed.

Record Prints the current screen Record.
Query Prints the all the selected records.
Select Allows selection of records for the report using the Table name and Field name fields.
Exit Closes the window and leaves the report option.

If the Select option was chosen and no records are found to match the selection criteria entered, an error message is displayed and new selection criteria should be entered.

If records are found to match the selection criteria the report continues and options are then displayed for printing.

The options for printing are described in the document Printing From Pension Master.

Exit Program

Click ‘Exit’. This leaves the program and closes the current browser tab.
