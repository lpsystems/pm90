Introduction

Only users who belong to a User Group that has the correct permission can access this option.

If you do not have permission, you will not be permitted access to this option.

This program allows maintenance of System Parameters. This contains details of any system-wide variables that can be set for Pension Master.

The following options and screen appear when this program is selected:

  • Update
  • Report
  • Menu
  • Exit

Basic System Parameters Screen

Add a Record

Basic System Parameters

Click ‘Add’. You cannot use the Add option if details already exist on the System Parameter file. In this case you must use the query option first (see below).

System Company Name - Enter the full Company name. This is used by Pension Master at the top of all the main menus. An entry must be made in this field and the system will display an error message if this field is left blank.

Short Name - Enter a short name for the Company. This is used at the top of all screens within Pension Master. An entry must be made in this field and the system will display an error message if this field is left blank. The user will then be required to enter the Company short name in this field.

Address & Postcode - Enter the address and postcode of the Company. At least the first two lines of the Company address must be entered and the system will display an error message if these fields are left blank.

Telephone Number - Enter the telephone number of the Company (optional).

Fax Number - Enter the fax number (if any) of the Company (optional).

Country - Enter your country or click on lookup icon and select it from the list. This is used to determine whether the first field on the ‘Fund Restrictions’ screen of ‘Investment Managers’ is displayed as ‘Post 1997 Investment?” (UK) or “Post 1991 Investment?” (Ireland).

Default Base Currency - Select a Default Base Currency from the drop-down list e.g. if your schemes are predominately UK based schemes, select ‘Pound Sterling’.

At the end of the transitional period in the introduction of the Euro i.e. on 31/12/2001, you should update this field to be ‘Euro’ i.e. make the system default base currency the Euro.

You will then be prompted that all annual statutory amounts are to be converted from the NCU to Euro and that this process in not reversible. You will be asked whether you have an up to date backup of the current system data and if not, you will be given an opportunity to exit the change and take a backup. Once this has been done, all annual statutory amounts will be converted to Euro’s.

You should also make sure that each of your schemes has the scheme base currency field in the Basic Scheme Details screen of Schemes set to ‘EUR’ at this stage to ensure that all scheme and membership monetary data is converted into Euro’s.

Calculate - Select one of the following options from the drop-down list:

  • Age Next Birthday
  • Age Last Birthday

This relates to the field pm_age in the member record which stores the age for each member and can be used in formulae, reports and mail merged output from Pension Master. The calculated age is displayed on the Basic Member Details screen in Members. It is dynamically calculated each time you go into the Basic Member Details screen, add a member, run a Member Report or Mail Merge Report or a formula that references this new field.

The way this is calculated differs on which program you are using as follows:

  • Members - always calculate age at today’s date i.e. the system date.
  • Member Report/Mail Merge Report - always calculate age at the report date.
  • Formula File/Quotations - always calculate age at quotation date.

Web Server User Manual Address - Enter the URL of the web-server on your network that contains the Pension Master User Manual. This can either be in the format http://webserver/pendoc/ or the hostname of the server, provided this is resolvable by all connected clients PC’s onto the correct IP address of the Pension Master server. If your network doesn’t have a web-server then the Pension Master User Manual folder should be held on the Pension Master server. The folder should be accessible to all users via a drive mapping. This field should then contain the drive mapping and the path to the folder e.g. M:\pendoc.

This is used by the Pension Master software to find the appropriate User Manual page when the user clicks or selects the User Manual icon on their screen.

Import Source - The Import Source can either be set to Network Drive or Local Drive.

If the Import Source is set to Network Drive then a Network Drive Path can be entered for either General Data Load and Mail Merge. This can either be a drive mapping (e.g. L: or P:) or a network location (e.g. \PMSERV\dataloads). The folders would need to be set as shareable with the correct permissions for the Pension Master user account on the server. Contact L&P Systems Limited for assistance.

The Network Drive Path could also be left blank if data is loaded from more than one location on the server. If the Import Source is set to Network Drive then the file that is entered as the Import File will be used for the data load. The Import File can be manually changed or selected from the data load folder using the file picker.

If the Import Source is set to Local Drive then General Data Load, allows the file to be selected manually from a folder using the browser each time.

When the System Parameter details have been entered click on ‘OK’ to save the details to the system parameter file. You can click on ‘Cancel’ at any stage to exit the add option without saving the details.

Basic System Parameters 2

Click ‘Update’.

Select the Basic System Parameters 2 screen as set out in the section Navigating Between Screens later in this section.

This screen allows you to set the order by which the member query displays the results of a search. In addition it holds certain indicator fields which allows the user to enable the following processes:

  • Workflow
  • Delete Quotation History
  • Member Session Report
  • Bulk Delete in Member File
  • Include NI/RSI/PPS/TRN Numbers in Standard Reports

The following screen is displayed:

Basic Parameter Details 2 Screen

Member Query Sort - Select the fields in the order that you wish Pension Master to sort the member query results, e.g. Surname then Forename will sort the members in alphabetical order of their surnames with sub-sorts of their forenames:

  • Category Code
  • Company Code
  • Date of Birth
  • Forename
  • Member Number
  • NI/PPS Number
  • Scheme Code
  • Staff Number
  • Status Code
  • Surname
  • Title

Capitalise Names/Text Fields - This allows you to switch on the automatic casing of certain fields. The field will default to casing these fields if ticked. If you wish your Pension Master system not to do this, for example if you are an Irish site with Gaelic surnames, you will have to untick the field. At present the following fields are supported:

  • Title
  • Forename
  • Surname
  • Job Title
  • Member’s salutation
  • Full forenames
  • Address Lines

If you tick this field, a prompt will be displayed asking if you really wish to change it as this will result in Pension Master automatically re-casing the above fields for each member record on the system. You will then be prompted again to ask if you really are sure that you wish to do this. This acts as a security procedure as it will not be possible to un-case the above fields for each member once you have indicated that you wish the casing to apply. To change back to un-cased fields, you will manually have to amend each members record, one by one, which may take a long time, or alternatively, restore a backup of your system data taken prior to you changing the casing flag. You should thus change this field only if you are sure this is what you want to do and are aware that the process is not reversible other than by a lengthy manual procedure or by restoring a system backup.

Use Workflow - This field denotes whether you wish to disable the workflow features in Pension Master. To enable workflow, make sure this field is checked (i.e. shows a tick in it). To disable workflow, make sure this field is not checked.

Delete Quotation History - This field denotes whether you wish to delete member’s quotation history details. To enable deletion, make sure this field is ticked. To disable deletion, make sure this field is not ticked.

After n Months Old - This field denotes the number of months old the quotation history details must be before they will be deleted. A minimum of 1 month old is required. This field works in conjunction with the ‘Delete Quotation History’ field. If the ‘Delete Quotation History’ field is ticked then this field must be entered. If the ‘Delete Quotation History’ field is un-ticked then this field must be left blank.

Run Member Session Report - This field denotes whether you wish to disable the member session report features in Pension Master. To enable the member session report, make sure this field is checked (i.e. shows a tick in it). To disable the member session report, make sure this field is not checked.

Allow Bulk Delete In Members File - This field denotes whether you wish to disable the bulk delete option in the Members file. To enable bulk delete, make sure this field is checked (i.e. shows a tick in it). To disable bulk delete, make sure this field is not checked.

Include NI/RSI/PPS/TRN Numbers in Standard Reports - This field denotes whether you wish to disable the displaying of the PPS Number in any of the standard reports delete option in the Members file. To enable, make sure this field is checked (i.e. shows a tick in it). To disable bulk delete, make sure this field is checked.

Internet Security Details Encryption - This option encrypts the following fields on the Internet Security Details screen in Members using either no encryption, MD5 or SHA-2:

  • Mother’s Maiden Name
  • Place of Birth
  • First School
  • Memorable Word

This process cannot be reversed and should only be selected after consultation with L&P Systems Limited.

Number of Days to Keep Trash - This option allows you to set the number of days that the application should keep the generated reports, spreadsheets and files that have been created by Pension Master. The default is 30 days.

Audit Trail Settings

Click ‘Update’.

Select the Audit Trail Settings screen as set out in the section Navigating Between Screens later in this section.

This screen allows you to set the criteria by which the Audit Trail logs any changes and activity on Pension Master by your users.

The following screen is displayed:

Audit Trail Settings Screen

Logins - Successful Logins - This field denotes whether you wish to switch on the logging of successful logins to Pension Master for your system. If this field is ticked all password changes made by users are also logged to the audit trail.

Logins - Unsuccessful Logins - This field denotes whether you wish to switch on the logging of unsuccessful logins to Pension Master for your system.

Maintenance Modules - Member File Changes 1 - This field denotes whether you wish to switch on the logging of changes made to member details in Members for the following areas in your system:

  • Basic Member Details
  • Part Time Service
  • Special Codes
  • Salary History
  • Additional Benefits
  • Transfer In Details
  • Transfer Out Details
  • Refund of Contributions
  • Deferred Pensioner
  • Pensioner
  • Valuations
  • Disability Benefit
  • Career Average Revalued Earnings
  • Death Details
  • Annual Allowance
  • User Defined Screens
  • User Defined History

Maintenance Modules - Member File Changes 2 - This field denotes whether you wish to switch on the logging of changes made to member details in Members for the areas not covered by Member File Changes 1 for your system.

Maintenance Modules - All Other Maintenance - This field denotes whether you wish to switch on the logging of changes made to all maintenance modules other than Members for your system.

Update Modules - Log Changes - This field denotes whether you wish to switch on the logging of changes made to items on the ‘Updates’ menu in Pension Master for your system.

Reports - Run Reports - This field denotes whether you wish to switch on the logging of any reports that are run on your system.

Reports - Print Reports - This field denotes whether you wish to switch on the logging of any reports that are printed on your system.

System Administration - Log Changes - This field denotes whether you wish to switch on the logging of changes made to any items on the ‘System Administration’ menu in Pension Master for your system.

When the details have been entered click on ‘OK’ to save the details to the system parameter file. You can click on ‘Cancel’ at any stage to exit the add option without saving the details.

Password Option Details

Click ‘Update’.

Select the Password Option Details screen as set out in the section Navigating Between Screens later in this section.

This screen allows you to set various criteria that are used by the logon screen of Pension Master.

The following screen is displayed:

Password Option Details Screen

Minimum Password Length - This field is used to set a minimum length for passwords that a user may select using the logon screen of Pension Master. If you do not wish to set a minimum password length, set this field to 0. Otherwise, enter an the minimum length to be used e.g. 6.

Must Contain Numbers As Well As Characters - This field is used to ensure that any password a user chooses contains a mixture of numbers as well as characters. This gives Pension Master an extra level of security as it is likely that passwords containing both letters and numbers will be harder to guess than those just containing letters. If this field is ticked then any password a user chooses must contain at least one of the characters a-z (case insensitive) and at least one of the numbers 0-9.

Days Allowed Between Password Changes - This sets how frequently the user is forced to change their password e.g. if you set this field to 30, the user will be forced to change their password every 30 days. If you do not wish to force password changes in this way, set this field to 0.

Password History - Set this field to a valid number X, where X denotes to number of passwords to keep in the password history file. This controls the number of new passwords that must be used by a user before an old password can be re-used e.g. if this is set to 3, every time a user changes their password, they must choose a new password that is not the same as any of the last three passwords used. If you do not wish to check password uniqueness in this way, set this field to 0.

Limit User To One Concurrent Session - This field allows you to specify that each user can only log onto Pension Master once at any one point in time.

Password Required - This field allows you to specify if a user has to choose a password to log in to Pension Master. If this field is ticked then a user may log in to Pension Master without supplying a password, otherwise the user will be prompted for a password at login and will not be permitted access to Pension Master without first supplying a non-blank password.

Password Encryption - This field allows the following fields to be encrypted using the more secure SHA-2 instead of MD5: User Passwords, the Internet Password on the Internet Security Details page in Members and Password Settings in Payroll Master.

The Password Encryption field should only be changed by L&P Systems Limited.

When the details have been entered click on ‘OK’ to save the details to the system parameter file. You can click on ‘Cancel’ at any stage to exit the add option without saving the details.

Administration Report Configuration

Click ‘Update’.

This screen allows you to set various criteria that are used by the Scheme Administration Report

The following screen is displayed:

Administration Report Configuration

Membership Movements Configuration - This field is used to set which date is use for comparisons purposes for risk only members.

Monetary Movements Configuration - This section is where you define which status codes should be included for the monetary amounts.

Contribution Summary Configuration - This sets out which contribution codes you can use in and what order to display them in.

Query a Record

Click ‘Query’.

If any match to the selection criteria is found then the record will be displayed to the screen. There is only one record allowed on the System Parameter file at any one time as this is the current corporate user of Pension Master.

Update a Record

Before Updating, a record must be selected through the query option, and an error message is displayed if a record for update has not been selected.

Click ‘Update’.

When the changes have been made to the record click on ‘OK’ to save the changes to file, or click on ‘Cancel’ to exit the update option without saving changes.

Printing System Parameter Details

The options for printing are described in the document Printing From Pension Master.

Navigating Between Screens

To change to a one of the other screens available in System Parameters, type ‘M’, or click on the ‘Menu’ menu option. The screens available are:

  1. Basic System Parameters
  2. Audit Trail Settings
  3. Password Option Details

Exit Program

Click ‘Exit’. This leaves the program and closes the current browser tab.
