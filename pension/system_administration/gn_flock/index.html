Introduction

Only users who belong to a User Group that has the correct permission can access this option.

If you do not have permission, you will not be permitted access to this option.

This program allows you to administer the lock file within Pension Master.

Locks are devices that are created every time a user updates a record within any part of Pension Master and are used to indicate that that particular record is already open for editing by a user and hence to deny all other users access to update this record until the original user has finished making their changes to the record and has exited input mode.

For each user that is in update mode, a lock is created which consists of a Lock ID, Username and Date/Time the lock was created. The current system locks in force are displayed on the Locks screen.

If a user exits a program within Pension Master abnormally i.e. by a means other than using the ‘Exit’ option from the program menu, a lock file may remain in place for that user. In this case, the System Administrator should use Locks to locate the appropriate lock file and remove it. Failure to do so will mean that no users will be permitted access to this record until such time as the lock is removed.

The following options and screen appear when this program is selected:

  • Delete
  • 1-Day
  • Refresh
  • Browse
  • Exit

Locks Screen

Delete A Lock File

Click ‘Delete’.

You will be moved into the lock id field you may wish to delete the locks that match a criteria of the lock id field or for a certain user. When you press okay you will be presented with the locks of that criteria you can select the one you wish to delete or all. Move the cursor using the arrow keys until it is on the lock file you wish to delete and press Return or click on ‘OK’. Or press ‘ALL’ and then press ‘OK’

A window will be displayed with the options ‘Yes’ and ‘No’.

If you do not wish to delete the record click ‘No’. You will return to the program menu.

If you wish to delete the record click ‘Yes’. A new window is displayed asking if you are sure.

If you do not wish to delete the record click ‘No’. The program returns to the main menu.

If you wish to delete the record click ‘Yes’. The record is deleted, and the program returns to the main menu.

Delete All Lock Files Over A Day Old

Click ‘1-Day’.

This option will delete all lock files over a day old, by reference to the current system date and the Date/Time column on the screen. It is intended to be a quick way of deleting stray lock files that have been left over from any users who have exited programs abnormally.

A window is displayed with the options ‘Yes’ and ‘No’.

If you do not wish to delete the record click on ‘No’. You will return to the program menu.

If you wish to delete the record click on menu ‘Yes’. A window is displayed asking if you are sure.

If you do not wish to delete the record click ‘No’. The program returns to the main menu.

If you wish to delete the record click ‘Yes’. The record is deleted, and the program returns to the main menu.

Refresh The Display

Click ‘Refresh’.

The list of lock files currently in force will be automatically updated and you will be returned to the main menu.

Browsing Lock Details

Click on the ‘Browse’ button or the Browse icon on the toolbar to view details in any screen where the data does not fit on the screen.

Once in browse mode, use the arrow keys or the scrollbar on the right of the data to navigate around the data.

Click on ‘OK’ or ‘Cancel’ to leave browse mode.
