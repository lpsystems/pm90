Introduction

Only users who belong to a User Group that has the correct permission can access this option.

If you do not have permission, you will not be permitted access to this option.

This program allows maintenance of the User file. This allows the System Administrator to control who is allowed access to Pension Master and what options they can access e.g. they can set a user up so that they do not have access to the more sensitive areas of the system like Formulae.

The following options and screen appear when this program is selected:

  • Add
  • Query
  • Update
  • Delete
  • Report
  • Password
  • Menu
  • Exit

User File - Screen 1

Add a User

Basic User Details

Click ‘Add’.

You will be asked if you want to copy the user from another user on the database. Doing that will allow you to update users with the same access to menu’s and schemes.

USER ID - Enter the User ID. This must consist of alphanumeric characters only.

A User ID of ‘intranet’ cannot be used. This is reserved for members of a Company Pension Scheme that have accessed their own information via the Company’s intranet.

USER NAME - Enter the full name of the user. This can be used when printing off standard letters. An entry must be made in this field.

JOB TITLE - Enter the user’s job title. This can be used when printing off standard letters.

WORKGROUP - Enter the user’s workgroup or leave blank if you do not wish to assign this user to a workgroup. See the Workgroups option. If you are unsure of the workgroups currently set up on your Pension Master system, click on the lookup icon to see a list of Workgroups.

DATE PASSWORD LAST CHANGED - This is a display only field and shows the date the user last changed their password. This can give the System Administrator an indication of when the user will next be required to change their password if password expiration controls in System Parameters are set.

FORCE PASSWORD CHANGE NEXT LOGIN - You can use this field to force the user to change their password the next time they logon to Pension Master via the logon screen.

SECURITY LEVEL - Enter a security level for your user. This is used in conjunction with the security levels on each scheme record and each member record to determine if the user has access to that record. E.g. if a user has a security level of 99 and a scheme, screen or member record has a security level of 1 or 50, then the user will not be allowed access to that scheme/member record.

The valid security ratings are any number in the range 1 to 99, with 1 being the highest security level and 99 being the lowest. If you are unsure of the security levels that are available on your system, click on the lookup icon to see a list. See the Security Levels option.

USER GROUP - This determines what parts of the system the user has access too. Exact details of how this works can be found in the User Group option.

UPDATE MEMBER RECORDS - This determines whether you wish to allow the current user to add/update/delete member records in Members.

AUTOMATICALLY EXPAND FAVOURITES - This determines if the user is automatically presented with their favourites menu rather then the normal Pension Master menus when they log in.

DELETE MEMBER RECORDS -If “Delete Member Records” is set to “No” then Bulk Delete will not be a menu option in the Basic Member Details screen.

INTERNET ADMINISTRATOR - Used by Pension Master Online in order to control access to the administration section of Pension Master Online Portal.

When the first screen of user details have been entered click on ‘OK’ to save the details to the user file. You can click on ‘Cancel’ at any stage to exit the add option without saving the details.

Menu Option Details

When the first screen of details has been saved the second screen of user details is automatically displayed. This allows you to set up the menu options that will be available for this user.

Pension Master will default to allow all menu options with a ‘PE’ in the first column of the screen i.e. all pension related menu options. It will also default to allow access to the ‘Main Menu’ and ‘Diary Entry’ menu options.

The second screen of user details is as follows:

User File - Screen 2

This screen is where you setup which of the standard Pension Master menu items will be available to the current user. If you untick the Access field the corresponding menu item will not be displayed when the user uses Pension Master. In this way, you can effectively and quickly restrict acces to only those menu items a user need to do their job.

All available menu titles and menu options are displayed to the screen. Tick or untick the Access field depending on whether you wish this user to have access to this menu/menu option or not.

When the second screen of user details have been entered click on ‘OK’ to save the details to the user file. You can click on ‘Cancel’ at any stage to exit the add option without saving the details.

User Access Details

When the second screen of details has been saved the third screen of user details is automatically displayed. This allows you to set up the scheme access options that will be available for this user.

The third screen of user details is as follows:

User File - Screen 3

This screen is where you can define which schemes this user has access to. This can be done at three different levels:

  • All users have access to the scheme. This option is set via the User Access Details screen in Schemes.
  • All members of a particular workgroup have access to the scheme. See Workgroups. This option is set via the User Access Details screen in Schemes.
  • User level access. Here you define particular users via their user ID so as to restrict access to a scheme. You can set this option either using this screen or via the User Access Details screen in Schemes.

To see who is allowed access to a particular scheme, see the User Access Details screen in Schemes.

Amend the details for user access as appropriate.

The last column on the screen indicates what type of access level that line refers to. It takes the values ‘A’ (for all), ‘W’ (for workgroup) or ‘U’ (for user) corresponding to each of the levels outlined above respectively.

Click on ‘Add Row’ to insert a line and click on ‘Delete Row’ to delete the current line of details. Click on ‘Next Record’ to scroll down the screen or click on ‘Prev Record’ to scroll up the screen.

If you delete a line of data by clicking on ‘Delete Row’, that line will only be removed from the database if the last column is equal to ‘U’ i.e. user level security access. Otherwise, the line will be removed from the screen, but will re-appear once you click on ‘OK’ to save the details.

When the third screen of user details have been entered click on ‘OK’ to save the details to the user file. You can click on ‘Cancel’ at any stage to exit the add option without saving the details.

You will be returned to the main menu.

Click ‘Query’.

Browsing User Details

Click on the ‘Browse’ button or the Browse icon on the toolbar to view details in any screen where the data does not fit on the screen.

Once in browse mode, use the arrow keys or the scrollbar on the right of the data to navigate around the data.

Click on ‘OK’ or ‘Cancel’ to leave browse mode.

Viewing User Records

To view the other records selected press N to view the next record, and P to view the previous record. If N is chosen and there are no records after the current record an error message is displayed. If P is chosen and there are no previous records an error message is displayed.

Update A Users Record

Before Updating, a record must be selected through the query option, and an error message is displayed if a record for update has not been selected.

Click ‘Update’.

The User ID field cannot be changed as this is the key to the user file. When the changes have been made to the record click on ‘OK’ to save the changes to file, or click on ‘Cancel’ to exit the update option without saving changes.

Delete A User Record

Before Deleting, a record must be selected through the query option, and an error message is displayed if a record for deletion has not been selected.

Click ‘Delete’.

When the delete option is selected a window appears with the options ‘Yes’ and ‘No’.

If you do not wish to delete the record, click ‘ No ‘. You will return to the program menu.

If you wish to delete the record, click ‘Yes’. A new window is displayed asking if you are sure.

If you do not wish to delete the record, click ‘No’. The program returns to the main menu.

If you wish to delete the record, ‘Yes’. The record is deleted and the program returns to the main menu.

Printing User Details

Click the ‘Report’ icon.

If records have been selected in the program before the report option was selected the following window is displayed.

Print Options Screen 1

If no records have been selected in the program before the report option was selected the following window is displayed.

Print Options Screen 2

Record Prints the current screen Record.
Query Prints the all the selected records.
Select Allows selection of records for the report using User ID or User Name.
Exit Closes the window and leaves the report option.

If the Select option was chosen and no records are found to match the selection criteria entered, an error message is displayed and new selection criteria should be entered.

If records are found to match the selection criteria the report continues and options are then displayed for printing.

The options for printing are described in the document Printing From Pension Master.

Changing A User’s Password

Click ‘Password’.

A popup window now appears as follows:

Enter the fields as follows:

NEW PASSWORD - This field is used to set a password for the user when they log into Pension Master. This is typically done by the System Administrator when adding a new user, at which time the System Administrator also sets the ‘Force Password Change Next Login?’ field to ‘Yes’ so that the user can choose their own password the first time they use Pension Master.

CONFIRM PASSWORD - Enter the pasword again to confirm the spelling. If any differences are found with the entry you made in the New Password field you will be prompted to re-confirm the password.

Click on ‘OK’ to save the new password, click on ‘Cancel’ to abort the new password entry without saving any changes and click on ‘Help’ to display this help text.

Re-Set Default Menu Options

Click ‘Default’.

This option is only available when in the second screen of Users.

This option re-sets the user’s menu options to the defaults. This will allow the user access to:

  • all menu options with a ‘PE’ in the first column of the screen i.e. all pension related menu options
  • the ‘Main Menu’ and ‘Diary Entry’ menu options.

Navigating Between Screens

To move between screens, click ‘Menu’. The menu options are as follows:

  • Basic Member Details
  • Menu Option Details
  • User Access Details

Exit Program

Click ‘Exit’. This leaves the program and closes the current browser tab.
