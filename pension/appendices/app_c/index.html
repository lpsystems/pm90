Introduction

Throughout Pension Master pairs of Codes and Descriptions are saved to database records to maximize disc space and access speed. The codes themselves can be somewhat cryptic so for users of Pension Master we show a wordy description of the code. In some cases it is necessary for the user to know which code has been assigned to the description. For example if you needed to access the Marital Status of a Member in Formula you would use #pm_marital_stat, that could then return ‘MA’ as a value. The value in this case has a description of ‘Married’.
