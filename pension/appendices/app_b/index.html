Selecting All members

All members may be selected by leaving all fields on the screen blank and clicking on ‘OK’.

Simple Selection Criteria

A search for members within certain groups can also be executed. E.g. to select all members in scheme 0001 enter the following:

Member        [         ] Marital Status [  ]
Scheme Code   [0001   ]   Sex            [ ]
Company Code  [       ]
Category Code [       ]
Status Code   [       ]

When you click on ‘Ok’ the following members are selected:

Member Name Initials Scheme Company Category Status Marital Status Sex
1 Anderson F. 0001 0001 001 0 MA F
2 Barnewell J. 0001 0002 001 2 SI F
3 Canavan W. 0001 0002 009 11 MA M
4 Daly B. 0001 0003 009 40 MA M

E.g. to select all Female, Married members in scheme 0003 enter the following:

Member        [         ] Marital Status [MA]
Scheme Code   [0003   ]   Sex            [F]
Company Code  [       ]
Category Code [       ]
Status Code   [       ]

When you click on ‘OK’ the following members are selected:

Member Name Initials Scheme Company Category Status Marital Status Sex
8 McCaffrey O. 0003 0002 001 10 MA F
12 Quilligan P. 0003 0003 003 14 MA F

When entering selection criteria the exact format of the field must be entered. Status code is a good example of this.

E.g. If the following criteria were entered:

Member        [         ] Marital Status [  ]
Scheme Code   [       ]   Sex            [ ]
Company Code  [       ]
Category Code [       ]
Status Code   [10     ]

When you click on ‘OK’ NO members are selected.

The reason for this is that Status code is a three character field and the format of status code 10 is ‘ 10’ ( i.e. space 10 ). Therefore to select members with a status code of 10 the following criteria would be entered:

Member        [         ] Marital Status [  ]
Scheme Code   [       ]   Sex            [ ]
Company Code  [       ]
Category Code [       ]
Status Code   [ 10    ]

When you click on ‘OK’ the following members are selected:

Member Name Initials Scheme Company Category Status Marital Status Sex
7 Marshall G. 0002 0002 009 10 MA M
8 McCaffrey O. 0003 0002 001 10 MA F

Relational Operators

Relational operators may also be used when entering selection criteria, the operators that may be entered are as follows:

> Greater Than >xxx >= Greater Than or Equal To >=xxx < Less Than <xxx <= Less Than or Equal To <=xxx <> Not Equal To <>xxx : Range xxx:xxx | Either Or xxx|xxx
Operator Name Pattern

## Greater Than

E.g. To select all members with a Scheme code greater than 0001 and a status code greater than ‘ 10’ enter the following:

Member        [         ] Marital Status [  ]
Scheme Code   [>0001  ]   Sex            [ ]
Company Code  [       ]
Category Code [       ]
Status Code   [ 10    ]

When you click on ‘OK’ the following members are selected:

Member Name Initials Scheme Company Category Status Marital Status Sex
9 O’Connell M.B. 0003 0003 001 11 SI F
10 O’Donnell K. 0003 0003 002 12 MA M
12 Quilligan P. 0003 0003 003 14 MA F

Greater Than or Equal To

E.g. To select all members with a Scheme code greater than 0001 and a status code greater than or equal to ‘ 10’ enter the following:

Member        [         ] Marital Status [  ]
Scheme Code   [>0001  ]   Sex            [ ]
Company Code  [       ]
Category Code [       ]
Status Code   [>= 10  ]

When you click on ‘OK’ the following members are selected:

Member Name Initials Scheme Company Category Status Marital Status Sex

Range

The Range operator allows a search for all values that lie between one value and another, the search is inclusive.

E.g. To select the members who have a Scheme code of 0001 and who have a status code of ‘ 0’ to ‘ 11’ inclusive enter the following:

Member        [         ] Marital Status [  ]
Scheme Code   [0001   ]   Sex            [ ]
Company Code  [       ]
Category Code [       ]
Status Code   [ 0: 11 ]

When you click on ‘OK’ the following members are selected:

Member Name Initials Scheme Company Category Status Marital Status Sex
1 Anderson F. 0001 0001 001 0 MA F
2 Barnewell J. 0001 0002 001 2 SI F
3 Canavan W. 0001 0002 009 11 MA M

Either Or

The either Or operator allows a search where the value in a field is equal to EITHER one value OR another.

E.g. To select members where the member number is equal to EITHER 3 OR 12 enter the following:

Member        [3|12     ] Marital Status [  ]
Scheme Code   [       ]   Sex            [ ]
Company Code  [       ]
Category Code [       ]
Status Code   [       ]

When you click on ‘OK’ the following members are selected:

Member Name Initials Scheme Company Category Status Marital Status Sex
3 Canavan W. 0001 0002 009 11 MA M
12 Quilligan P. 0003 0003 003 14 MA F

Wildcard Operators

In addition to the operators already described Wildcard operators may be used on character fields.

Wildcard Name Pattern

* Asterisk x* / x / *x

? Question Mark x? / ?x / ?x?

Asterisk

The Asterisk may be used to match any group of zero or more characters.

E.g. To select members whose surname ends in ‘ll’ enter the following:

Member   [         ]
Forename [                    ]
Surname  [*ll                 ]

When you click on ‘OK’ the following members are selected:

Member Name Initials Scheme Company Category Status Marital Status Sex
2 Barnewell J. 0001 0002 001 2 SI F
7 Marshall G. 0002 0002 009 10 MA M
9 O’Connell M.B. 0003 0003 001 11 SI F
10 O’Donnell K. 0003 0003 002 12 MA M

E.g. To select members with any occurrence of ‘ll’ in surname enter the following:

Member   [         ]
Forename [                    ]
Surname  [*ll*                ]

When you click on ‘OK’ the following members are selected:

Member Name Initials Scheme Company Category Status Marital Status Sex
2 Barnewell J. 0001 0002 001 2 SI F
7 Marshall G. 0002 0002 009 10 MA M
9 O’Connell M.B. 0003 0003 001 11 SI F
10 O’Donnell K. 0003 0003 002 12 MA M
12 Quilligan P. 0003 0003 003 14 MA F

Question Mark

The question mark may be used to match a single character.

E.g. To select members whose surname starts with “O’”, and ends with “onnell” with one unknown character between the start and end of the surname enter the following:

Member   [         ]
Forename [                    ]
Surname  [O' ?onnell          ]

When you click on ‘OK’ the following members are selected:

Member Name Initials Scheme Company Category Status Marital Status Sex
9 O’Connell M.B. 0003 0003 001 11 SI F
10 O’Donnell K. 0003 0003 002 12 MA M
