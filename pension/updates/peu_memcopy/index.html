Introduction

The Copy Members allows the user to copy a selection of members from one scheme to another. The user is able to select which parts of the member’s record they wish to copy over. All the program expects the user to do is select at least the Basic Member Details.

The following options and screen is displayed:

  • Copy
  • Exit

Copy Members Screen

Copy

Click ‘Copy’.

SELECTION CRITERIA - Select the group of members you wish to copy using the Scheme, Company, Category, Status, Special Code and Member fields (see Selection Criteria). When the selection criteria have been entered click on ‘OK’ to proceed or click on ‘Cancel’ to return to the main menu.

Once the selection of members has been made, a popup window will appear so the user can enter where they wish to copy the members to.

Copy Members To Window

SCHEME - Enter the Scheme Code or select a code from the lookup list. The code that is entered will be assigned as the scheme code for the selected members to copy.

COMPANY - Enter a valid Company Code for the Scheme entered or select a code from the lookup list. The code that is entered will be assigned as the company code for the selected members to copy.

CATEGORY - Enter a valid Category Code for the Scheme entered or select a code from the lookup list. The code that is entered will be assigned as the category code for the selected members to copy.

One the user has entered these codes a popup window will appear which will allow the user to select which areas of the member’s record they wish to copy.

Select Data To Copy Window

The areas of data that are available to the user to copy are dependant on a few things:

  • If the scheme, the user is copying members from, is the same as the scheme, the user is copying members to, then all the User Defined Screens (UDS) and User Defined Histories (UDH) will be available for them to copy. However, if the schemes are different then they will not be. This is because the UDS and UDH in the scheme being copied into may not be set up in the correct format as the way they are in the scheme copying from.
  • Members Quotations will not be available to copy over, due to the necessity of needing the quotations and formulas to be set up in exactly the same way from one scheme/category to another.
  • Members Switches will not be available to copy over, due to the complexity of the switches, especially of the switches have been created via the Bulk Switch program.

The user must select at least one area of a member’s record to copy over, and the Basic Member Details must also be part of the data being copied over too. If either of these criterias are not met then the user will be warned of this and will not be allowed to proceed until they have been met.

Once the areas of the member’s record have been selected then click ‘OK’ to continue, or ‘Cancel’ to return to the main menu.

If there are no members selected from the criteria entered by the user, then the user will be informed of this and the program will return to the main menu. Also, if the scheme the user is trying to copy members into is flagged as a test scheme, then the program will check that the number of members already held in that scheme, plus the number of members the user is trying to copy doesn’t exceed 10. If it does then the user will be informed of this and the program will return to the main menu.

A report will now be produced out to Excel listing the members that have been selected to be copied. The user will now be asked if they wish to continue with the copying of these members. Click ‘Yes’ to continue, or ‘No’ to return to the main menu.

The following screen will now be displayed that informs the user of the copying process.

Copying Data Progress Screen

Once all the data has been copied over, 1 or 2 more reports are produced out to Excel. The first one lists out the members that have been copied and the corresponding new member numbers for each of these members. The second one will be a Code Setup Report.

The Code Setup report is only created if the scheme the user is copying members into is different to the scheme they are copying the members from, and areas such as Special Codes, Deferred Pensioner, Pensioner, Contribution History, Investment History etc have been selected to copy. In order for these areas of data to be displayed correctly to the screens in Members, the associated codes such as Special codes, Deferred Pensioner codes, Pensioner codes, Contribution codes, Investment Funds etc are needed to be set up in the scheme being copied into. If they are not already existing on the scheme then this report is produced to inform the user to set them up now.

Exit

Click ‘Exit’. This leaves the program and closes the current browser tab.
