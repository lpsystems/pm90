Introduction

This program calculates results from formulae and then writes the results back to one of the Salary History screens in Members, depending on which screen the user has selected.

The following options and screen appears when this program is selected:

  • Run
  • Reports
  • Undo
  • Exit

Writeback Salaries Screen

Run

Click ‘Run’.

You will be asked if you wish to do a Provisional or an Actual run. A Provisional run does all that an actual run does, but does not write any salaries to the database.

The following screen is displayed:

Select Writeback Screen

You must select which Salary Screen you wish to write the salary results back to. Once selected you will then need to enter the selection criteria for the members you wish to writeback salaries for.

SELECTION CRITERIA - Select the group of members you require to writeback salaries for using the Company, Category, Status, Special Code and Member fields (see Selection Criteria). When the selection criteria has been entered click ‘OK’ to proceed or click ‘Cancel’ to return to the main menu and abort the writeback salaries program.

The following screen is displayed:

Writeback Salary Screen Results

The fields are described as follows:

SALARY DATE - Enter a date. This date will be used to insert or overwrite a record in the Salary History screen.

IGNORE/FORMULA CODE - If the Ignore option is selected, then you will be sent to the next field. If the Formula Code option is selected, then you will be required to enter a valid formula code in the field next to it. You can do this by using the lookup list.

If all Ignores are chosen then a warning message will appear informing you that a least one formula code must be entered.

The formula code must produce a numeric result. If it doesn’t then it will be reported out to the exceptions report, and no writebacks will occur.

If the Ignore option is selected, then if a new record has to be inserted into the Salary History screen for the date entered, then 0.00 will be sent through for the field in question. However, if a record already exists for the date, then the existing value for the field will not be touched.

When the writeback information has been entered, click ‘OK’ to proceed or click ‘Cancel’ to return to the main menu. Only fields that have a Formula are processed.

Reports

Three reports are generated after the calculation of the writeback salary results:

EXCEPTION REPORT - If any exceptions have been found then the number of exceptions will be displayed and you will be asked if you wish to view or print the report. If there are any exceptions found then writebacks cannot be processed until the exceptions have been dealt with.

This report details any problems that have been found during the calculation of writebacks. Where applicable the Member Number and Name are printed along with a comment describing the problem that has been found. Possible exception messages are:

  • Record locked.
  • Members Formula Code (TEST) used for Salary Field 1 not found in Scheme.
  • The formula TEST used for Salary Field 1 has not returned a numeric value.

The options for printing are described in the document Printing In Pension Master.

WARNING REPORT - If any warnings have been found then the number of warnings will be displayed and you will be asked if you wish to view or print the report. The purpose of warnings is to inform you what’s going to happen, they won’t prevent the writebacks from continuing.

This report details information that has been found during the calculation of writebacks. Where applicable the Member Number and Name are printed along with a comment. A possible warning message is:

  • A record already exists on Salary Screen 1 for Salary Date 01/01/2000. N.B. Data will be overwritten.

The options for printing are described in the document Printing In Pension Master.

RESULTS REPORT - If any writebacks have been successfully calculated then the number of writebacks will be displayed and you will be asked if you wish to view or print the report.

The report shows:

  • Member Number and Name.
  • The 5 Salary values that have been created.
  • Whether a new record will be inserted or and existing one will be overwritten.

The options for printing are described on the Printing page.

When an Actual Writeback is performed the program may encounter records that are locked by other users on the system. These locked records will appear on the Exception Report with a comment of ‘Record locked’. It is therefore important that this report is checked to ensure that ALL the selected members have had the calculated amount written back to them.

Undo

Click ‘Undo’.

This allows data to be removed from the Salary History screen.

A small Select Writeback Screen window is displayed allowing the user to select which screen they wish to delete data from. This can be either one of the 4 Salary History screens.

Select Writeback Screen

On selecting the screen type, you will be prompted back to the Undo Writeback Salaries screen (similar to the Writebacks Salaries screen as shown above).

When the selection criteria has been entered, click ‘OK’ to proceed or click ‘Cancel’ to return to the main menu.

A small window will appear titled either Salary History Date.

Salary History Date

SALARY HISTORY DATE - Enter the date that you want to delete data for in the format ddmmyy or dd/mm/yyyy. A value must be entered in this field and the date must be a valid date.

This date is used to delete data from the Salary History screen.

When the Date has been entered, click ‘OK’ to proceed or click ‘Cancel’ to return to the main menu.

After the processing has been completed you will be asked if you wish to view the undos that will delete data. The Undo Report contains the following information:

  • Member details.
  • The 5 Salary values that will be deleted.

The options for printing are described in the document Printing In Pension Master.

When you have checked that the procedure will remove the correct salary records, you will be asked to confirm that you wish to remove the number of undos found. If you have selected ‘Yes’ you will then be asked to re-confirm that you wish to remove these undos. If you select ‘No’ at any time you will be returned to the menu with no further action taking place.

Exit Program

Click ‘Exit’. This leaves the program and closes the current browser tab.
