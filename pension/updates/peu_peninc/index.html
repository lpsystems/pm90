Introduction

The Pension Increase Run calculates new pension values based upon either a percentage rate or the result of a formula.

The program is split into 4 main sections described as follows:

Provisional This allows an increase run to be performed but does not update the Pension/Deferred Pension History.
Actual This allows an increase run to be performed and does update the Pension/Deferred Pension History.
Undo This allows an increase run to be removed from the Pension/Deferred Pension Increase History screen and reverts the values on the History screen back to before the Actual Increase run was performed.
Delete This allows a previously saved Increase Parameter to be removed.

The members that are increased can be selected using standard Simple Selection Criteria (i.e. Scheme, Company, Category, Status, Special Code and Member). A list of Pension Codes currently allocated to the scheme is displayed allowing either all the codes to be updated with a global Formula/Fixed Rate or to be increased individually.

The information used to select the members and the specific increase information used against each Pension Code can be saved as an individual set of parameters (known as a Pension Increase Parameter Set). These parameters are specific to each scheme and each pension type, for either Deferred Pension or Pension increases. This allows them to be saved, using a descriptive name, for future use.

Select the Pension Increase Run option from the Updates menu.

The following options and screen is displayed:

  • Run
  • Undo
  • Delete
  • Exit

SCHEME - Enter the Scheme Code or select a code from the lookup list. The code that is entered will be used to select only members for this scheme. It will also allow any increase parameters that are saved to be re-used for increases to this scheme.

PENSIONER/DEFERRED PENSIONER - Click on a button to select either Pensioners or Deferred Pensioners to be increased. If no option is selected then Pensioner will be selected by default.

This date is used to update the Date of Last Increase on the History screen (see Pension History and Deferred Pension History) and also to generate a new line on the Increase History screen (see Pension Increase History and Deferred Pension Increase History).

USE A SAVED INCREASE PARAMETER? - Check the tick box if you wish to select a previously saved Increase Parameter.

If you are using a saved Increase Parameter then a lookup list will be displayed showing all the saved parameters for that particular scheme and pension type (either Pensioner/Deferred Pensioner).

Provisional Pension Increase Run Screen

If an Existing Increase has been selected from the Run Parameter window then the name of the parameter file selected will appear at the top of the screen. The screen will also be pre-filled with any information that was used as selection criteria when the original increase as saved.

SELECTION CRITERIA - Select the group of members you require to run pension increases for using the Company, Category, Status, Special Code and Member fields (see Selection Criteria). When the selection criteria have been entered click on ‘OK’ to proceed or click on ‘Cancel’ to return to the main menu.

A confirmation message will be displayed if no selection criteria has been entered. This simply states that all members have been selected and that the processing could possibly take some time.

The following screen is displayed if Pensioners have been selected using the Run Parameter window:

Provisional Pension Increase Run Screen

The following screen is displayed if Deferred Pensioners have been selected using the Run Parameter window:

Provisional Pension Increase Run Screen

When the increase information has been entered click on ‘OK’ to proceed or click on ‘Cancel’ to return to the main menu.

NOTE: If an Existing Increase has been selected: A check is made at this point to ensure that there is no descrepancy between the Pension Codes saved and Pension Codes allocated to the scheme. A message will be displayed showing any Pension Codes that have been allocated to, or removed from, the scheme since the increase was saved. Any new Pension Codes will be displayed with no formula or fixed rate against it.

This screen displays all the Pension Codes, and descriptions, of the members that have been selected using the selection criteria. The screen allows global or individual Pension Codes to be increased by either the result of a Formula Code or by a Fixed % Rate. For example to increase Pensioners:

  • If you want to increase all the Pension Codes, for the members, by the same percentage then enter a Fixed % Rate in the global section on the member line. The Global Pro Rata field can be ticked as required and a Global Reason for Increase Code must be entered. When the global details have been entered the information will be copied down the screen against each of the Pension Codes.
  • If you want to increase a single Pension Code, for spouses, using the result of a formula then click in the field required, and enter a valid Formula Code against the Pension Code on the spouses line. Also a valid Reason for Increase Code must be entered.

The fields on this screen vary depending on whether you have selected Pensioner or Deferred Pensioner in the Run Parameter window:

For Pensioners Members and Spouses pension values can be increased.
For Deferred Pensioners Members and Spouses (Before and After the Normal Pension Date) pension values can be increased.

The fields are described as follows:

FORMULA CODE - Enter a valid Formula Code or select a code from the lookup list. If a Formula Code is entered with a Fixed % Rate on the same line then an error message is displayed indicating that one of them must be removed. Any Formula Code that is entered on the global line will be automatically entered into the Formula Code fields against each of the Pension Codes.

The Formula must have previously been entered into the formula file using Formulae. It should be checked to ensure that the formula correctly calculates an INCREASE VALUE ONLY. If the formula generates an inaccurate value, then the Exception Report will ONLY detect any new pension value that is more or less than 100% different from the original value.

The calculation used is as follows (All the calculations should be rounded to 8 decimal places unless otherwise stated):

New Pension Value = (Current Pension + Formula Result) / Pay Frequency (Rounded to 2 decimal places)

New Pension Value = Pay Frequency * New Pension Value

FIXED % RATE - Enter a Fixed Percentage Rate. If a Fixed Percentage Rate is entered with a Formula Code on the same line then an error message is displayed indicating that one of them must be removed. Any rate that is entered on the global line will be automatically entered into the Fixed Percentage Rate fields against each of the Pension Codes.

The calculation used is as follows (All the calculations should be rounded to 8 decimal places unless otherwise stated):

New Pension Value = (Current Pension / Pay Frequency) * (Fixed % Rate / 100 + 1) (Rounded to 2 decimal places)

New Pension Value = Pay Frequency * New Pension Value

PRO RATA - Tick this box if a Pro Rata increase should be calculated. This will determine the fraction of the Formula/Fixed % Rate increase based on the following:

  • If the member is a New Pensioner (a pensioner that has had no previous increases) then the partial year is determined from the Date to Commence to the Increase Date.
  • If the member is a New Widower (a widower that has had no previous increases) then the partial year is determined from the Original Members Start Date to the Increase Date.
  • If the member is a New Deferred Pensioner (a deferred pensioner that has had no previous increases) then the partial year is determined from the Date Left to the Increase Date.
  • If the member is a Pensioner/Widower/Deferred Pensioner who has had a previous increase for a pension element then the partial year is determined from the Date of Last Increase to the Increase Date.

Note: When using a Pro Rata increase, the increase will be calculated for both new and existing Pensioners/Deferred Pensioners using the rules above.

If the partial year includes the 29th February, obviously of a leap year, then the number of days in the calculation below will be 366. Otherwise the number of days will be 365.

The calculation used for a Fixed % Rate is as follows (All the calculations should be rounded to 8 decimal places unless otherwise stated):

New Pension Value = (Current Pension / Pay Frequency) * ((Fixed % Rate * (Period in Days / 365)) / 100 + 1) (Rounded to 2 decimal places)

New Pension Value = Pay Frequency * New Pension Value

The calculation used for a Formula is as follows:

New Pension Value = (Current Pension + ((Formula Result / 365) * Period in Days)) / Pay Frequency (Rounded to 2 decimal places)

New Pension Value = Pay Frequency * New Pension Value

ROUNDING - Enter a figure here if you want Rounding to be applied to the final figure. An entry of 0.05 will round the final figure to the nearest 5c.

REASON FOR INCREASE CODE - Enter a valid Reason for Increase Code or select a code from the lookup list. A Reason Code must be entered if a Formula Code or Fixed % Rate is entered on the line.

This code is used to describe the type of increase that has been performed. It is shown on the Increase History screen (see Pension Increase History and Deferred Pension Increase History) on the new line generated by the increase run.

When the increase information has been entered, click ‘OK’ to proceed or click ‘Cancel’ to return to the main menu. Only codes that have a Formula or a Fixed % Rate are processed.

If you selected a New Increase in the Run Parameters window or if you selected an Existing Increase (and have changed some of the parameters) then you will be asked if you wish to save this set of parameters. A small window will appear prompting you to enter a suitable name. If the name you have entered already exists then you will be asked to confirm that you want to overwrite these details.

Two reports are generated after the calculation of the pension increases:

EXCEPTION REPORT - If any exceptions have been found then the number of exceptions will be displayed and you will be asked if you wish to view or print the report.

This report details any problems that have been found during the calculation of pension increases. Usually a problem may occur because of existing pension details. Where applicable the Pension Code and it’s Description are printed along with a comment describing the problem that has been found. Possible exception messages are:

  • No Pensioner Records Found.
  • Reason for Increase Code (RPENINC) is not allocated to this Scheme.
  • Date to Commence (07/07/1999) is after the Increase Date.
  • Date of Cessation (01/01/1999) is before the Increase Date.
  • Date of Last Inc. (02/02/1999) is on or after the Increase Date.
  • Formula Code (A550) not found in Scheme.
  • Pension increase (from 10123.45 to 20123.45) is greater than 100%.
  • Pension increase (from 20123.45 to 10123.45) is less than -100%.
  • The formula has not returned a numeric value.

The options for printing are described in the document Printing From Pension Master.

INCREASE REPORT - If any pension increases have been successfully calculated then the number of increases will be displayed and you will be asked if you wish to view or print the report.

The report shows:

  • Member details.
  • The Pension Code that has been increased and it’s description.
  • Details of the calculation used for the Pension Code.
  • The pension values both before and after the increase.

The options for printing are described in the document Printing From Pension Master.

Actual Increase

Click ‘Run’, and then select the ‘Actual’ option from the pop-up window.

This option allows an increase run to be performed and does update the Pension/Deferred Pension History.

The Actual Increase run is performed in the same way as the Provisional Increase run except that the members details are updated.

When you have checked that the selected members will be increased by the amount required, you will be asked to confirm that you wish to update the number of increases generated. If you have selected ‘Yes’ you will then be asked to re-confirm that you wish to update these increases. If you select ‘No’ at any time you will be returned to the menu with no further action taking place.

When the Actual Increase is performed the program may encounter Pensioner or Deferred Pensioner records that are locked by other users on the system. These locked records will appear on the Exception Report with a comment of ‘Record locked’. It is therefore important that this report is checked to ensure that ALL the selected members values have been increased by the required amount.

Undo Increase

Click ‘Undo’.

This allows an increase run to be removed from the Pension/Deferred Pension Increase History screen and reverts the values on the History screen back to before the Actual Increase run was performed.

A small Run Parameters window is displayed allowing basic details of the Pension Increase Undo to be entered:

Run Parameters Window

This window allows additional undo parameters to be entered as follows:

SCHEME - Enter the Scheme Code or select a code from the lookup list. The code that is entered will be used to select only members for this scheme.

PENSIONER/DEFERRED PENSIONER - Click on a button to select either Pensioners or Deferred Pensioners to be undone. If no option is selected then Pensioner will be selected by default.

UNDO INCREASE DATE - Enter the date that you want to undo in the format ddmmyy or dd/mm/yyyy. A value must be entered in this field and the date must be a valid date.

This date is used to change the Date of Last Increase on the History screen (see Pension History and Deferred Pension History) to the previous increase date, if any. Also it will delete the corresponding lines on the Increase History screen (see Pension Increase History and Deferred Pension Increase History).

UNDO A SAVED INCREASE? - Check the tick box if you wish to select a previously saved Increase Parameter to undo.

When the Run Parameters have been entered, click ‘OK’ to proceed or click ‘Cancel’ to return to the main menu.

If New Undo has been selected then the following selection criteria screen will be displayed:

If Undo Existing Increase has been selected then a lookup list will be displayed showing all the saved parameters for that particular scheme and pension type (either Pensioner/Deferred Pensioner). Upon selecting a parameter from the list the following selection criteria screen will be displayed showing the saved information:

Pension Increase Undo Screen

When the selection criteria has been entered, click ‘OK’ to proceed or click ‘Cancel’ to return to the main menu.

The following screen shows all the codes that have been found using the above criteria. This criteria also includes the Scheme, Pension type and the Date of Increase that was entered using the Run Parameter Window.

Pension Increase Undo Screen

UNDO - Tick the box to select the Pension Code to undo. If the undo box on the global line is ticked then all the Pension Codes will be undone.

NOTE: Only the last increase can be removed from Pension/Deferred Pension Increase History. If a date is entered that is not the last increase then a ‘No Members have been selected with this criteria.’ error message will be displayed. This is because checks will be made against the Pension/Deferred Pension History screen to find information that matches the criteria entered.

After the processing has been completed you will be asked if you wish to view the increases that will be undone. The Undo Report contains the following information:

  • Member details.
  • The Pension Code that has been changed and it’s Description.
  • Details from the Increase History screen (see Pension Increase History and Deferred Pension Increase History). This shows all the pension details that will be removed and also the information which will then be current. If only the existing data is printed and no historical data, then the existing data will be deleted from the Pension Increase History or Deferred Pension Increase History screen.

The options for printing are described in the document Printing From Pension Master.

When you have checked that the procedure will remove the correct increase details, you will be asked to confirm that you wish to remove the number of increases found. If you have selected ‘Yes’ you will then be asked to re-confirm that you wish to remove these increases. If you select ‘No’ at any time you will be returned to the menu with no further action taking place.

Delete Increase Parameter

Click ‘Delete’.

This option allows Increase Parameters to be deleted.

Delete Pension Increase Parameters Screen

The fields are described as follows:

PENSIONER/DEFERRED PENSIONER - Click on a button to select either Pensioner or Deferred Pensioner Increase Parameters to delete. If no option is selected then Pensioners will be selected by default.

SCHEME - Enter the Scheme Code or select a code from the lookup list. The code that is entered will be used to select the increase parameters for this scheme.

DESCRIPTION - Double click on the description required or move the cursor using the arrow keys and press return.

You will be asked to confirm that these are the Increase Parameters that you wish to remove. If you have selected ‘Yes’ you will be asked to re-confirm that you wish to remove these Increase Parameters. If you select ‘No’ at any time you will be returned to the menu with no further action taking place.

Exit

Click ‘Exit’. This leaves the program and closes the current browser tab.
