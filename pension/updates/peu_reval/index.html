Introduction

The Care Revaluation Run calculates new pension values based upon the result of a formula.

The program is split into 4 main sections described as follows:

Provisional This allows an increase run to be performed but does not update the Care Amount History.
Actual This allows an increase run to be performed and does update the Care Amount History.
Undo This allows an increase run to be removed from the Care Amount History screen and reverts the values on the History screen back to before the Actual Increase run was performed.
Delete This allows a previously Care Revaluation Parameter to be removed.

The members that are increased can be selected using standard Simple Selection Criteria (i.e. Scheme, Company, Category, Status, Special Code and Member). A list of Care Codes currently allocated to the scheme is displayed allowing either all the codes to be updated with a Formula or to be increased individually.

The information used to select the members and the specific increase information used against each Care Code can be saved as an individual set of parameters (known as a Care Run Parameter Set). These parameters are specific to each scheme and each care code type. This allows them to be saved, using a descriptive name, for future use.

Select the Care Revaluation Run option from the Updates menu.

The following options and screen is displayed:

  • Run
  • Undo
  • Delete
  • Exit

Selection Screen

Run CARE Revaluation Runs

Click ‘Run’, and then select either ‘Actual’ or ‘Provisional’ option from the pop-up window.

Provisional option allows an revaluation run to be performed but does not update the Care History Screens.

Then you have the option of doing a Revaluation Run or a Recalculation of Revaluation Run . The Revaluation Run simply calculates the revaluation and updates the members care screen. The Recalculation of Revaluation Run recalculates from either:

  • The first revaluation record on the members revaluation screen
  • Date Joined for Pension
  • A date from which the recalculation of the revaluation is to commence

A Run Parameters window is displayed allowing you enter the scheme code and the date of the Care Run to be entered:

Run Parameters Window

This window allows additional run parameters to be entered as follows:

SCHEME - Enter the Scheme Code or select a code from the lookup list. The code that is entered will be used to select only members for this scheme. It will also allow any care parameters that are saved to be re-used for increases to this scheme.

REVALUATION DATE - Enter the date from which your care increases are applicable in the format ddmmyy or dd/mm/yyyy. A value must be entered in this field and the date must be a valid date.

USE A SAVED REVALUATION PARAMETER? - Check the tick box if you wish to select a previously saved Increase Parameter.

If you are using a saved Revaluation Parameter then a lookup list will be displayed showing all the saved parameters for that particular scheme and care pension types.

When the Run Parameters have been entered, click ‘OK’ to proceed or click ‘Cancel’ to return to the main menu.

The following selection criteria screen will be displayed:

CARE Increase Run Screen

If an existing Increase has been selected from the Run Parameter window then the name of the parameter file selected will appear at the top of the screen. The screen will also be pre-filled with any information that was used as selection criteria when the original increase as saved.

SELECTION CRITERIA - Select the group of members you require to run pension increases for using the Company, Category, Status, Special Code and Member fields (see Selection Criteria). When the selection criteria have been entered click on ‘OK’ to proceed or click on ‘Cancel’ to return to the main menu.

A confirmation message will be displayed if no selection criteria has been entered. This simply states that all members have been selected and that the processing could possibly take some time.

The following screen is displayed if a saved parameter wasn’t selected using the Run Parameter window:

CARE Pension Increase Run Screen

You are now given the option to run a quote before the actual care run. You would use this option if you needed the results of a current quote in the formulae for your care run. Therefore if you select Y then in the next field you are able to select a quotation type that you have set up for that scheme. On selecting the quote to run, the batch quotation run will be triggered in the background

  • Any warnings/exceptions will be produced as usual and you will be given the option to continue or cancel the care run program. Please note that the use of pop ups in any formulae within this quotation will return a value of zero for that formulae. So it’s advised that you remove any pop-ups from formulae so that no warnings are produced

  • Each of the care code assigned to the scheme appear here, you can insert a formula from the scheme formulae file or use the lookup.

  • You don’t have to have a formula against every Care Code set up for the program to run as there may be times when you only want to run a revaluation for one of the Care Codes. If there are no formulas at all against a Care Code nothing runs for that Care Code.

When the formula information has been entered click on ‘OK’ to proceed or click on ‘Cancel’ to return to the main menu.

The Formula must have previously been entered into the formula file using Formulae. They should be checked to ensure that the formula correctly calculates the values.

When the run has completed you will receive a message telling you how many revaluations have been completed and how many warnings have been found. You have the option to view them in a report. An example of a warning would be that; ‘Revaluation Date 01/02/2011 is on or after the care revaluation run date’. It is essential that you view these reports, which are output to Microsoft Excel, to check the warnings and ensure the calculations are correct.

If a member has a date joined plan but it is after the revaluation date, then they will be ignored as mentioned above. If the member has no date joined plan then they will be output to the warnings report informing the user of their blank date, but they will not be processed. Members whose date joined plan is on or before the revaluation date will be processed in the normal manner. Members that have been listed as having a warning will not be updated

If you had selected to run a provisional run the process has now completed. However if you opted to run an actual run you are given a message asking if you want to update the revaluations. If you select to update the member records then the revaluation is written back to screen 1 and screen 2 of the members care record. The user is now offered the option to save the care revaluation parameters to that scheme for further use.

Run CARE Revaluation Runs - Recalculation of Revaluation Run

Click ‘Run’, and then select either ‘Actual’ or ‘Provisional’ option from the pop-up window.

Provisional option allows an revaluation run to be performed but does not update the Care History Screens.

The Recalculation of Revaluation Run recalculates from either: * The first revaluation record on the members revaluation screen * Date Joined for Pension * A date from which the recalculation of the revaluation is to commence

Please be aware that by using this Revaluation Type, you will not be able to undo the records if mistakes are made. The undo option in this program simply removes the last CARE Revaluation History record for the member. It doesn’t return the record to its original state.

A Run Parameters window is displayed allowing you enter the scheme code and which option you wish to Run:

Run Parameters Window - Recalculation of Revaluation

OPTION 1 Recalculation from 1st revaluation record on revaluation screen. - The recalculation process takes the earliest date on the care screen and uses that as the revaluation date for the first recalculation and then reruns and overwrites. Then it goes to the second earliest revaluation date and does the same process and so on and so on till the latest date. This is the default option.

OPTION 2 Revaluation from date joined for pension.- All existing records for that member for that care code will be removed and all newly calculated records will be added on. As a result, when selecting this option you are asked twice if you are sure you want to use the option.

This option has 3 additional parameters that are filled in under the ‘Option 2 Only’ section so that the dates of each revaluation can be determined. This information can also be entered at scheme level under the Care Titles screen. If there are values in the Care Title screen at scheme level, the ‘Option 2 Only’ section will be auto populated with these values but they can be overwritten if required.

  • Revaluation Frequency
    • Annual
    • Biannual
    • Quarterly
    • Monthly
  • Effective date type
    • Renewal date
    • Calendar year
  • Day of Revaluation
    • Month Start
    • Month End
    • Specific Day in Month

Then assumptions for frequency:

  • If the frequency is annual and the effective date is renewal year with a value of 1/7 and the day of revaluation day was ‘Month End’, the revaluation date would be 30/6 after the date joined for pension for the first revaluation and the next revaluation would be 30/6 of the next year etc. If it is calendar year, it would be 31/12 after the date joined for pension for the first revaluation and the next revaluation would be 31/12 of the next year etc.
  • If the frequency is biannual and the effective date is renewal year with a value of 1/3 and the day of revaluation day was ‘Month End’, the revaluation date would be 31/8 if the date joined for pension was prior to 31/8 or 28/2 if after 31/8 for the first revaluation and the next revaluation would be six months later. If it was calendar year, it would be 30/6 if the date joined for pension was prior to 30/6 but 31/12 if it was after 30/6 for the first revaluation. The next revaluation would be 6 months later.
  • If the frequency is quarterly and the effective date is renewal year with a value of 1/3, and the day of revaluation day was ‘Month End’, the revaluation date would be 31/5 after the date joined for pension if date joined for pension was prior to 31/5 or 31/8, 30/11 or 28/2 depending on what month the member joined within the renewal year for the first revaluation and the next revaluation would be 3 months later. If it was calendar year, it would be 31/3 after the date joined for pension if the date joined for pension was at or prior to 31/3 and if not the quarter end after they joined for pension, 30/6, 30/9 or 31/12 for the first revaluation and 3 months later for the next revaluation.
  • If the frequency is monthly and the day of revaluation day is ‘Month End’, the first revaluation date is the month end relative to date joined for pension.

OPTION 3 Date from which revaluation is to commence. - It is assumed that there are records already on the care screen if you choose this option, and that only records where the revaluation date is at or after the date input would be recalculated in the same way as option one. A message to this effect is presented when you select this option

The process is then the same as the revaluation run. When the run is completed you will be given the option to save this parameter and if you opt to do so the parameter is saved under the revaluation of recalculations types. If you wish to edit afterwards you tick the saved parameters field and its the same process as the revaluation run.

Undo CARE Revaluation Runs

Click ‘Undo’.

The latest revaluation can be undo via the care revaluation menu. This can be done for all members or a selection based on your selection criteria. Therefore this is the way you can unwind the record back using multiple undo runs. You must start at the latest date and you select per care code. If you have two care codes at the same date they can be removed together, if they are at separate dates you must remove each one separately

Undo Parameters Window -

The undo option in this program simply removes the last CARE Revaluation History record for the member. It doesn’t return the record to its original state if you have done a recalculation of revaluation.

Delete CARE Revaluation Runs

Click ‘Delete’.

You can only delete saved parameter runs and you are asked before it you are certain. This doesn’t delete anything off the member record it only deletes saved parameter runs assigned to the scheme.

Exit Program

Click ‘Exit’. This leaves the program and closes the current browser tab.
