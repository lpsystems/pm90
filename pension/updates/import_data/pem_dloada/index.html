Introduction

The following table lists all database tables held on Pension Master, which can be used in the General Data Load program.

Query the template load code and then press update to assigned it to a scheme for loading. Also, during the load some fields may not be included in the data file, or the data in the data file is found to be blank. In these cases the fields are automatically updated by the General Data Load with a default value. The fields with default values are also listed below for each database table

The data file format is:

Addresses - ‘Update’ type load only - Template TEMPSADU

Any of the fields on the scheme address screen table can be updated, apart from the Scheme Code and Address Code, as these are the table’s mandatory fields.

The data file format is:

schemecode,addresscode,addressname,addressline1,addressline2,addressline3,addressline4,postcode,telephone,fax,email

If any of the datalines in the datafile reference a scheme code and address code that do not exist on Pension Master e.g. there is a dataline that references address code TEST but there is no TEST address for the scheme on Pension Master, then an exception will be produced for this dataline and the line will need to be removed from the data file before you try to load the data again.

Basic Member Details - ‘Update’ type load only - Templates TEMPCOMP/TEMPMAR

Any field on the Basic Member Details table can be updated, apart from the Member Number, as this is the member’s unique identifier on Pension Master. The data file format is:

identifier,fieldname

If the user is updating a member’s company, category or status code then they will be prompted with a pop-up window which will ask them to either enter a ‘Start’ position or a ‘Default Date’, which will be used to set the Effective Date of the code amendment. This will be displayed on the corresponding company/category/status history screen for the member.

The ‘Start’ field allows the user to have an effective date present in the datafile for each member, so the user should enter the field position of where this date is in the datafile being used.

If the ‘Start’ field is left blank, or the data is found to be blank in the datafile, then the ‘Default Date’ will be used instead.

Contribution History Batch Details - ‘New’ type load only - Template TEMPCONT

Pension Master has been changed so that an unlimited number of Contributions can now be held against a scheme, rather than just the standard five (EEORD, EEAVC, ERORD, EEPR and ERPR). Due to this enhancement, the database tables, which hold all the Contribution details, have also been changed to allow for this.

It would be unpractical and unfair to expect the user to change their old data file format of:

identifier,amount1,amount2,amount3,amount4,amount5

to the new data file format of:

identifier,batchnumber,ordernumber,contribution,amount,expenses

where batchnumber and ordernumber are defaulted to 0, with the use of Rule 1 and Value of 0.

The General Data Load program would ideally like the data file to be in the new format, however, the user can still use the old format.

If the old format is being used then it is recommended to use the template TEMPCONT to add the load on, and then the user should leave the details in screen 2 as they appear. However, as the template uses the Staff Number as the default identifier, the user must change the first line in screen 2 to either Member Number or NI/PRSI Number if needs be.

It is left soley down to the user to instruct the program of the data file format they are using, during the running process.

Factors - ‘New’ type load only - Template TEMP

A new type load will be used to add a brand new table, a new index on an exsisting table or to add a new date to an exsisting table. If a description is not present in the data field the table code will be used as the description. If a new index is being added and other indexes with the range weren’t included those indexes are set to zero. Example range currently 1 to 5 new index of 8 to 10 added the index of 6 and 7 is set to zero.

The data file format is:

factorcode,description,type,sex,date,index,value,

Factor - ‘Update’ type load only - Template TEMP

The only field that can be updated is an exsisting Factor Value.

The data file format is:

factorcode,type,sex,date,index,value,
