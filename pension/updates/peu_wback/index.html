Introduction

This program allows you to writeback (and optionally finalise) the results for any quotation run on Pension Master. The data is written to that members exit screens or to that members user defined screens. You can also use the Undo option to reverse (and optionally un-finalise) a writeback that you have previously done.

The term ‘finalise’ means to tick the finalised field on the members quotation results and to update the members status code to the new specified status code. Additionally, if the member’s status code is changing from an active code to a non-active code, the members date left is also updated.

This has the effect of locking that quotation type for the member as Pension Master will not allow you to re-run finalised quotations. As an example, you might want to finalise a quotation type if a member retires and you have run a final retirement quote and written back the data to the members pensioner screens.

‘Un-finalise’ effectively reverses the finalise process described above.

The following options and screen appears when this program is selected:

  • Run
  • Undo
  • Exit

Writeback Quote Results Screen

Run

Click ‘Run’.

You will be asked if you wish to do a Provisional or an Actual run. A Provisional run does all that an actual run does, but does not write any results to the database.

SELECTION CRITERIA - Select the group of members you require to writeback quote results for using the Scheme, Company, Category, Status, Special Code and Member Number fields (see Selection Criteria). When the selection criteria have been entered click ‘OK’ to proceed.

You can click ‘Cancel’ to return to the main menu and abort the writeback run.

The following screen is displayed:

Quote Type Selection Screen

QUOTATION TYPE - Enter the quotation type to write quote results back for.

QUOTATION DATE - Enter the date for the quotes you wish to write back.

FINALISE QUOTES? - Check this if you want to also finalise the specified quotation type for the selected group of members

FINALISED STATUS - This option is only applicable if you have checked the Finalise Quotes? field and is used to specify the new status code for all selected members for which the quotation is to be finalised.

If you are writing back results to the Lifetime/Annual Allowance screen for the selected members, then a popup window will appear which will prompt you to enter a year for the 5th April you wish to write the results back for. The popup window is as follows:

Annual/Lifetime Allowance Date Screen

Pension Master then proceeds to write the appropriate quotation results to the specified exit or user defined screens (as specified in Category File) for each member selected. If successful and the Finalise Quotes? option was checked, it also finalises any quotations for this quotation type and selected group of members.

There are several situations that can cause exceptions:

All types of writeback Quotation results must exist for the given quotation type and date.
All types of writeback The field to be written back to must exist in Report Units.
All types of writeback The result to be written back must be of the same data type as the field it is being written back to.
Transfer In The transfer type field must be one of Additional Pension, Additional Service or DC Transfer.
Transfer In/Out A transfer in/out cannot already exist for the given transfer in/out code.
Deferred Pensioner/Pensioner If any deferred pensioner/pensioner basic details are included in the writeback, then a deferred pensioner/pensioner basic details record cannot already exist.

Otherwise, A deferred pensioner/pensioner basic details record must already exist before you can writeback deferred pensioner/pensioner history.
Deferred Pensioner/Pensioner Pay frequency (if written back) must be one of 1, 2, 4, 12, 13, 52.
Reason for cessation (if written back) must be one of B, C, D, E, O.
Account Type (if written back) must be either BANK or BSOC.
Sort Code (if written back) must be in the format xx-yy-zz.
Deferred Pensioner/Pensioner An entry cannot already exist on the deferred pensioner/pensioner history for the given deferred pension/pension code.

If successful, a report detailing the proposed writebacks is displayed and can be printed at this time.

If the writeback was un-successful, an exception report will be produced which details the records that were rejected and the reason why. You must address each exception before being able to writeback the quotation results successfully.

The options for printing are described in the document Printing From Pension Master.

The system will then return to the main menu.

Undo

Click ‘Undo’.

You must be of Supervisor level to use this option. See Users for more details.

An undo quote writeback run deletes any records created by a quote writeback run as set out above and brings the selected member records back to the situation immediately before the actual quote writeback run was performed. It also un-finalises any quotations for this quotation type and selected group of members.

Use the Undo Writeback option with extreme caution as it will delete data in the appropriate area for a member. This data may have been updated manually since the writeback was originally performed and these changes will be lost.

The exact details of records removed by the Undo Writeback option are:

Death All death details for the member.
Deferred Pensioner If any deferred pensioner basic details exist in the writeback, then all deferred pension details for the member, including deferred pension history, totals and increase history.

Otherwise, all deferred pension history and increase history for the member and specified deferred pension code.
Annual/Lifetime Allowance The user will again be prompted to enter a year for the 5th April record you wish to delete for the member. Once entered, the details for the member and specified date will be removed.
Pensioner Basic Details If any pensioner basic details exist in the writeback, then all pension details for the member, including pension history, totals and increase history.

Otherwise, all pension history and increase history for the member and specified pension code.
Protected A-Day Benefits All protected A-Day benefit details for the member.
Refund All refund details for the member.
Transfer In All transfer in details for the member and specified transfer in code.
Transfer Out All transfer out details for the member and specified transfer out code.
User Defined Screens User defined field details, based on the field number entered at writeback setup, for the member and specified user defined screen code (N.B. may include more than one user defined screen).

Before the process deletes any data, a warning message is displayed informing you of the data that is about to be deleted as per the above table. This is your final chance to stop the process before any data is deleted.

If successful, a report detailing the data that has been deleted is displayed and can be printed at this time.

Once the process has been run you will be returned to the main menu.

Exit

Click ‘Exit’. This leaves the program and closes the current browser tab.
