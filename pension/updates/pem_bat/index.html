Introduction

This program allows entry and maintenance of batches of either salary, NI history or contribution history data for a group of members. This is typically used to input the annual renewal data for a defined benefit type scheme.

The data is held at the batch level until you are satisfied that it is correct, at which time you can opt to post the data off to the individual member records.

This means that the data is held separate from the main member data until such time as you are satisfied it is correct and choose to update the main member data with the details.

You can use this to:

  • Enter batches of salary details
  • Enter batches of NI contracted-out earnings details (UK Only)
  • Enter batches of contribution details.
  • Post batched data to the relevant history file on each members record.

The following options and screen appear when this program is selected:

  • Add
  • Query
  • Update
  • Delete
  • Report
  • Exit

Batch Data Input Screen

Add a Batch

Click ‘Add’.

The batch number is initially set to zero and will default to the next batch number in sequence once the first screen of batch details has been entered.

BATCH TYPE - Select the type of batch from a list of available batch types:

  • Salary Screen 1
  • Salary Screen 2
  • Salary Screen 3
  • Salary Screen 4
  • NI History
  • Contribution History

BATCH DATE/DATE DUE - Enter the batch date (Date Due for Defined Contribution batches). A value must be entered in the date field. The date must be a valid date.

DATE RECEIVED (Defined Contribution batches only) - Enter the date received for your defined contribution batch. A value must be entered in the date field. The date must be a valid date.

TRANSACTION TYPE (Defined Contribution batches only) - Enter the transaction type for your defined contribution batch. If you are unsure of which code to use, click the lookup icon to see a list of available codes.

The date entered for the batch date (due date, date received and transaction type for contribution batches) will be used to post the Salary/NI History/Contribution Details to the relevant file in the members record. Thus if you enter, say, 1/4/97 for a salary batch, when you post the batch, a salary record will be generated on each members salary history record as at 1/4/97.

POSTED - This field will default to No. Only when the user has chosen the ‘Post’ option, will this field be set to Yes. Once this field is set to Yes, the user cannot update the batch details.

When the first screen of Batch Data Input details have been entered click ‘OK’ to save the details. Click ‘Cancel’ at any stage to exit the add option without saving the details.

The following screen is displayed:

Batch Data Input Screen

This second screen is only available in the ‘Add’ option.

Enter the fields as follows:

SCHEME CODE - The scheme code must exist on the scheme file. When a valid scheme code is entered, the scheme description will be displayed to the screen. Click the lookup icon to see a list of scheme codes. This field must be entered.

COMPANY CODE - The combination of scheme and company code must exist on the company file. When a valid company code is entered, the company description will be displayed to the screen. You can click the lookup icon to see a list of company codes within the chosen scheme.

CATEGORY CODE - The combination of scheme and category code must exist on the category file. When a valid category code is entered, the category description will be displayed to the screen. Click the lookup icon to see a list of category codes for the chosen scheme and company.

STATUS CODE - The combination of scheme and status code must exist on the status file. When a valid status code is entered, the status description will be displayed to the screen. You can click the lookup icon to see a list of status codes for the chosen scheme.

SPECIAL CODE GROUP - The Special Code Group entered must be valid. Click the lookup icon to see a list of special code groups.

SPECIAL CODE - The Special Code entered must be valid. When a valid special code is entered, the special code description will be displayed to the screen. Click the lookup icon to see a list of special codes.

MEMBER NUMBER - The Member Number entered must be valid. When a valid member number is entered, the member’s name will be displayed to the screen. Click the lookup icon to see a list of member numbers.

At least the Scheme code must be entered.

When the second screen of Batch Data Input details have been entered click ‘OK’ to save the details. Click ‘Cancel’ at any stage to exit the add option without saving the details.

To Add A Salary History Batch

If you chose a Salary History batch type, the following Screen is displayed:

Salary Batch Details Screen

The description of the salaries will change depending upon those entered against each scheme for this history screen.

A list of valid members, selected from the selection criteria entered in the second screen, will be displayed.

Enter the member number. The member number entered must exist on the members file and the member must belong to the Scheme/Company/Category entered. When a valid member number is entered the member name will be displayed. Click the lookup icon to see a list of members for the chosen scheme/company/category combination.

Enter salary details for the member. (See Members for detailed descriptions of the salary fields if required).

When the third screen of batch details have been entered click ‘OK’ to save the details or click ‘Cancel’ to exit without saving.

To Add A NI History Batch

If you chose a NI History batch type, the following screen is displayed:

NI History Batch Details Screen

Enter the member number. The member number must exist on the member file and the member must belong to the Scheme/Company/Category entered. When a valid member number has been entered the member name will be displayed to the bottom of the screen. Click the lookup icon to see a list of members for the chosen scheme/company/category combination.

Enter the NI contributions/contracted-out earnings details for the member.

When the third screen of batch details have been entered click ‘OK’ to save the details or click ‘Cancel’ to exit without saving.

To Add A Contribution History Batch

If you chose a Contribution History batch type, the following screen is displayed:

Contribution Batch Details Screen

Enter the member number. The member number entered must exist on the members file and the member must belong to the Scheme/Company/Category entered. When a valid member number is entered the member name will be displayed. Click the lookup icon to see a list of members for the chosen scheme/company/category combination.

Enter contribution details for the member. (see Members for detailed descriptions of the fields if required).

When the third screen of batch details have been entered click ‘OK’ to save the details or click ‘Cancel’ to exit without saving.

At this stage, expenses will be calculated for each member as appropriate. The expense assumptions used are as set out in the Expense Assumptions screen in Categories.

At this stage the contributions have not been allocated to individual funds. To do this you first need to post the contributions to the member records using the Batch menu option, then allocate these contributions to funds using the Contribution Allocation program.

Query/Search For A Batch

Click ‘Query’.

Enter details of your selection and click ‘OK’ to retrieve the records. Click ‘Cancel’ to abandon your search.

Viewing Records

To view the other records selected press N to view the Next record, and P to view the Previous record.

Update a Batch

Click ‘Update’.

The record for update must be selected and displayed to the screen before you will be allowed to update it.

Before updating, a record for update must be selected through the Query option.

If ‘Update’ is selected, the only field on the first screen (Batch Details Screen) that may be modified is the batch date (due date, date received and transaction type for contribution batches).

Delete a Batch

Click ‘Delete’.

The record for deletion must be selected and displayed to the screen before you will be allowed to delete it.

You will be asked if you are sure you wish to delete the record. If you answer yes to this the record is deleted.

The program then returns to the main menu.

Printing Batch Details

Click the ‘Report’ icon.

The options for printing are described in the document Printing From Pension Master.

Navigating Between Screens

Click ‘Menu’. A pop-up menu will open on the screen which allows the user to pick a screen from a list of the screens available. You can also click ‘Cancel’ to abort the screen selection process.

The menu selection pop-up is as follows:

Screen Selection Screen

Posting Batch Details

Click Post. This option allows you to post a batch of history details to each members record.

If the batch is a Contribution Batch and has been generated by the Contribution Request Run the following window is displayed:

Actual Contributions Window

Enter the date you received the contributions and the total amount received. If the amount received is the same as amount expected, the process continues as below. If not, a message is displayed explaining the difference and the batch post is abandoned until such time as the total amount expected equals the total amount received. You should now use the Update menu option to alter any individual amounts in this batch as required.

The following pop-up window is displayed:

Posting Batch

You have the option to either run a Provisional or an Actual data load. Select the option you want and then click ‘OK’.

A matrix will now be displayed to the screen as follows:

Post Options

Enter the method you wish the data to use, in writing back to member records. The valid options are as follows:

Add Add the specified field value to that entry that already exists for the member for this batch date. If the data is not already on the member record for this specified field, simply insert the new data.
Insert Only save the data on the member record if the data is not already on the member record for this specified field. If the data already exists then do not update it.
Ignore Do not make any update at all for this specified field.
Replace Replace any data that exists for the specified field for this batch date on the member record. If the data is not already on the member record for this specified field, simply insert the new data.

Click ‘OK’ to save the details entered and to continue with the data load. Click ‘Cancel’ to exit out of the matrix and return to the main menu.

You will be asked if you wish to continue with the Batch Posting. If you have selected the Yes option, the batch will be posted and you will be returned to the main menu.

If an Actual load has been selected then a record will be inserted into the relevant history file for each member in the batch, and, in the case of a contribution batch, update the contribution totals for each member.

When the posting has finished, you will be returned to the batch file main menu.

Undo Batch Details

Click Undo. This option allows you to undo a posted batch of history details to each members record. The undo option results in the history details being removed, so please ensure that this is what you want to happen before using this option.

The members to have their history records removed are the same selected members used in the posted batch. This saves the user from re-entering the selection details.

On selecting this option, the following window is displayed:

All details will be removed

Click ‘Yes’ to continue with the undo process. Click ‘No’ to return to the main menu.

A report is produced listing all the history records to be removed, and any records which wil not be removed due to the record not existing. In the case of Contribution History, the contribution may have already been allocated either on or after the date due. In this case the record cannot be removed.

After viewing the report, you will be asked if you wish to continue and update the database with the undo details. Click ‘Yes’ to complete the undo process, and to un-tick the ‘Posted’ field. Click ‘No’ to return to the main menu.

Browse Batch Details

Click ‘Browse’ to view details in any screen where the data does not fit on the screen.

Once in browse mode, use the scrollbar on the right of the data to navigate around the data.

Click ‘OK’ or ‘Cancel’ to leave browse mode.

Exit Program

Click ‘Exit’. This leaves the program and closes the current browser tab.
