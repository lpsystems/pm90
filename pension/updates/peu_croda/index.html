Introduction

This program allows the user to load data into Pension Master from external data files for contributions/nis, salaries and addresses.

The program goes through a checking process, to check that the format and validity of the data is correct. If any problems are found, e.g. a date value is trying to be put into a decimal field, or if a staff number/NI number doesn’t exist on Pension Master then a warnings report will be generated informing the user of this. The warnings report will not stop the user from continuing to load the data, however, any data lines found with a warning will not be processed.

The external data files being used must be tab delimited. In order for the user to ensure this, they should use a software package, like Microsoft Excel, to save the spreadsheets as Text (Tab delimited) prior to using this program.

Datafile Layouts

This program can load from up to a maximum of 4 datafiles. Each datafile chosen MUST be in a certain layout/format and contain no headers or footers.

CONTRIBUTION AMOUNTS DATAFILE LAYOUT - This file must be in the following format and must be saved as a Tab delimited file:

Staff Number
Name
Exec 6% E&M
Exec 6% H
Exec Vol
Group 6% B
Group 6% SL
Group Vol
Pen Specials 1/80
EE/ET up to ER/ET
NI Code
Pens Scheme No

SALARY AMOUNTS DATAFILE LAYOUT - This file must be in the following format and must be saved as a Tab delimited file:

Staff Number
Name
Commission
Shift Allowance
Bonus
Xmas Bonus
Annual Salary
NI Code
Div/Unit
NI Number
D.O.L
Pens Scheme No

PENSIONER ADDRESSES DATAFILE LAYOUT - This file must be in the following format and must be saved as a Tab delimited file:

Pens Scheme No
Name
Address Line 1
Address Line 2
Address Line 3
Address Line 4
Postcode
NI Number

The Pens Scheme No for this datafile only is expected in the format of xxxxy, where xxxx are 4 numbers and y is a character.

ACTIVE ADDRESSES DATAFILE LAYOUT - This file must be in the following format and must be saved as a Tab delimited file:

Staff Number
Name
Address Line 1
Address Line 2
Address Line 3
Address Line 4
Postcode
Pens Scheme No

Data Load Screen

Run a Data Load

Click ‘Run’.

Enter the fields required to run the load you want.

CONTRIBUTION DATE - If loading a file for Contribution Amounts then you must enter a contribution date in this field. This date will then be used for all lines within the Contribution Amount datafile.

NI DATE 5TH APRIL - If loading a file for Contribution Amounts, which should also contain NI amounts, then enter the year for year ending 5th April. This year will then be used for all lines within the Contribution Amounts datafile.

CONTRIBUTION AMOUNTS - Enter the filename path of the data file you are going to use to load on contribution/NI amounts. The window, shown at the end of this section, will pop-up on entering this field.

SALARY AMOUNTS - Enter the filename path of the data file you are going to use to load on salary amounts. The window, shown at the end of this section, will pop-up on entering this field.

PENSIONER ADDRESSES - Enter the filename path of the data file you are going to use to load on pensioner addresses. The window, shown at the end of this section, will pop-up on entering this field.

ACTIVE ADDRESSES - Enter the filename path of the data file you are going to use to load on active addresses. The following window will pop-up on entering this field.

Select your data file, then click ‘Open’.

If the data file is held on the Pension Master server then click ‘Cancel’ and enter the filename straight into the filename field on the screen. Note that in this case, the data file is expected to be in the same directory as the Pension Master programs on the server.

If a filename is entered which does not exist then an error message will appear informing you of this, and you will be invited to re-enter a valid filename for your data file.

The program expects the datafile(s) entered to be complete with no blank lines.

At least one filename needs to be entered in order for the loading process to continue.

Once all necessary fields have been entered click ‘OK’ to continue and the following pop-up window appears informing the user of where the data is being loaded, and then asking them if they wish to continue:

Continue window

Click ‘Yes’ to continue loading the data, or click ‘No’ to stop the process and return to the menu. The process starts by checking that the datafiles entered do not contain blank lines nor are empty. If any of them are then an appropriate message will be displayed informing the user of this. The process is then abandoned.

The process will continue to check that the data inside the datafiles is correct and valid. If any errors are found, then a Warnings report will be produced listing these errors. The user will be asked if they wish to view the report, and then asked if they wish to continue and load the data that have no errors.

On continuing with loading the data, reports for each datafile will be produced out to Microsoft Excel, which will list all the data that will be loaded. Once these reports have been produced then the user will finally be asked if they wish to load this data. Click ‘Yes’ to load the data onto Pension Master.

For Contribution Amounts, if data already exists on Pension Master for the member and contribution data entered, then the data in the data file will overwrite the data found on Pension Master, else it will be added on as a new record.

For NI Amounts, if data already exists on Pension Master for the member and NI year entered, then the data in the data file will be added to the amount found on Pension Master, else it will be added on as a new record.

For Salary Amounts, if data already exists on Pension Master for the member and salary date, found in the datafile, then the data in the data file will overwrite data found on Pension Master, else it will be added on as a new record.

For Pensioner and Active Addresses, if an address already exists for the member and address number 1, then the address in the datafile will overwrite the address found on Pension Master, else it will be added on as a new record and the current flag for the address will be flagged as ‘Yes’.

Exit Program

Click ‘Exit’. This leaves the program and closes the current browser tab.
