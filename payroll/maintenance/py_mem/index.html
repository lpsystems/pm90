Introduction

This program allows maintenance of the Member file. This is where you will find data relating to a member’s pension details. The data is accessed via a series of screens, each grouping together similar parts of the member’s pension data. To navigate between the screens in the Member File, click ‘Menu’. The menu option is described in more detail later in this section. You are not able to Add new members in this program. This must be done through the Member File in Pension Master.

The different screens of information that you will find in the Member File are as follows:

  • Member’s Personal Details
  • Member’s Pension Details
  • Member’s Year To Date Details
  • Member’s Bank Details
  • Member’s History Details
  • Member’s Pay Element Details

The following options and screen appear when this program is selected.

  • Query
  • Update
  • Menu
  • Report
  • Exit

Member Personal Details Screen

Query/Search For A Member

Click ‘Query’. The following pop-up window is displayed:

Query Options Screen

You may select records by using the Member Number, Member Forename, Member Surname, Scheme Code, Company Code, Category Code, Status Code, NI/PRSI Number, Staff Number, Sex, Marital Status, Date of Birth fields. You may also use any combination (or even partial combination) of these fields to select the required member(s). See the Selection Criteria page.

Once you have entered your selection criteria, click ‘OK’ to proceed and retrieve the records from the database or click ‘Cancel’ to abort the selection and return to the Members main menu.

If more than one record is found to match your search criteria, the following screen is displayed:

Multiple Member Selection Screen

Use the arrow keys to locate the member record you require and click ‘OK’ to select it.

If only one record is found to match your search criteria, the member record is displayed directly to the screen.

A message is then displayed to the bottom of the screen.

Record 00001 of 0000x

If you wish to view any other page of member details other than the current screen, click ‘Menu’, then select the appropriate screen by moving the highlight bar and clicking ‘OK’.

Quick Search

Click ‘Quick Search’. This automatically re-opens the ‘Multiple Member Selection Screen’ as detailed in the Query/Search For A Member section. This enables you to select another member record based upon the selection criteria entered.

This saves you having to use the Query option to re-select the current set of member records.

Viewing Member Details

You can use the options on the toolbar to view the first, previous, next and last records in your query.

If the member has a valid e-mail address, click ‘Send E-Mail’ to launch your mail program and send an e-mail directly to the member.

Updating A Member

Member’s Personal Details

Click ‘Menu’ to move to the Member’s Personal Details screen and then click ‘Update’.

The Member’s Personal Details screen is used to hold the personal details about the member of the scheme.

Enter the fields required as follows:

STAFF NUMBER - Enter the member’s staff number. Payroll Master will check other member records in the member’s scheme for duplicate Staff numbers and if any are found, the user will be asked whether they would like to continue. Click ‘Yes’ to continue or ‘No’ to exit. This field is optional.

NI/PRSI NUMBER - National Insurance Number (UK) or PRSI Number (Ireland). Payroll Master will check all other member records on the database for duplicate NI numbers and if any are found, the user will be asked whether they would like to continue. Click ‘Yes’ to continue or ‘No’ to exit. If the UK / Irish Scheme indicator, in Pension Master, is set to ‘I’, i.e. Irish Scheme, then this field will be shown as PRSI Number and will relate to the Pay Related Social Insurance Number (IRL).

TITLE - The member’s title (e.g. Mrs, Mr, Dr ). This field is optional, but can be used in Mail Merge Reports.

FORENAME & SURNAME - A forename and surname for the member must be entered. The forename field is used to automatically generate the member initials field on the Member Address screen in Pension Master.

PAYROLL NUMBER - This field may be useful in linking the payroll number with member records for data transfer or reports.

MARITAL STATUS - Select one of the following options from the drop-down list:

  • No Data Found

ON HOLD - Tick this field if the member’s payroll is to be put on hold. If the field is ticked then the member will not be included in any payroll runs.

SEX - You must enter a value in this field. Select one of the following options from the drop-down list:

  • No Data Found

D.O.B. - Date of Birth. This is a date field and should be entered in the format ddmmyy or dd/mm/yyyy. If you enter a date for which the member’s current age is less than 16 or over 75, an error message will be displayed and you will be asked to re-enter the member’s date of birth.

ADDRESS LINES & POSTCODE - Enter the member’s address and postcode (if known). These will be used in the Payroll Run RTI submissions, and P60 forms in the Year End process.

TELEPHONE NUMBER - Enter the member’s evening and daytime telephone numbers (if known).

E-MAIL ADDRESS - Enter the member’s e-mail address (if known). This must contain the ‘@’ symbol that is present in all e-mail addresses.

SUSPECT - If the address entered is suspect then tick this field.

Click ‘OK’ to save the Member’s Personal Details information, or click ‘Cancel’ to exit the screen without saving the details entered.

Member’s Pension Details

Click ‘Menu’ to move to the Member’s Pension Details screen and then click ‘Update’.

The following screen appears when selected.

Member Pension Details Screen

The Member’s Pension Details screen is used to hold the pension details about the member of the scheme.

Enter the fields as required as follows:

TAX CODE - Enter the member’s tax code. The member’s amount of taxable pay is calculated from this. This field must be entered.

WEEK/MONTH 1 BASIS - If the member’s pay is to be calculated on a week/month 1 basis, then this field must be ticked.

PAYMENT METHOD - This represents the method used for paying the member. Simply click the method the member is to use:

  • BACS - Member is paid directly into their bank/building society account by BACS
  • Manual Cheque - Member is paid by cheque
  • Other - Member is paid by another payment method

PAYMENT FREQUENCY - Select one of the following options from the drop-down list:

  • No Data Found

This field must be entered. If it is not then it will default to Monthly.

The Payment Frequency of a member can only be changed before the first Payroll Run of a new tax year is carried out. This is to prevent irregularities in the member’s net pay.

TAX MONTH OF 1ST PAYMENT - This field holds the tax month the first payment of the tax year the member is to be paid in. This field is referred to in the Payroll Run if the member being paid has a payment frequency of Annual, Half Yearly or Quarterly, AND has either had no payroll run for them before on Payroll Master i.e. a new member, or it is the first payroll of the tax year for them. Once the first payroll of the tax year has been processed for the member, this field isn’t referred to again until the next new tax year. A tax month must be entered in this field.

The Tax Month of 1st Payment can take the following values:

  • Annual - 1 to 12
  • Half Yearly - 1 to 6
  • Quarterly - 1 to 3
  • Monthly - 1
  • 4 Weekly - 1
  • Fortnightly - 1
  • Weekly - 1
  • Irregular - 1
  • One-Off - 1

PRINT PAYSLIP? Select one of the following options from the drop-down list:

  • Always - the payslip will always be printed.
  • Tolerance - the payslip will only be printed if outside the Payslip Tolerance value at Payroll Scheme level.
  • Never - the payslip will never printed.

If the Member’s Payroll Scheme has a Payslip Tolerance then Print Payslip? will be set to Tolerance by default. Otherwise Print Payslip? will default to Always.

DATE STARTED PENSION - This is a date field and can be entered in the format ddmmyy or dd/mm/yyyy. The date the member started their pension, must be entered in this field.

DATE STARTED PAYROLL - This is a date field and can be entered in the format ddmmyy or dd/mm/yyyy. The date the member started having their payroll processed through Payroll Master, must be entered in this field.

DATE FINISHED PENSION - This is a date field and can be entered in the format ddmmyy or dd/mm/yyyy. The date the member finished their pension is entered in this field. When a date is entered in this field the Last Payment indicator will also be ticked.

LAST PAYMENT - This indicates to HMRC that the leaving date should be sent with the next Payroll Run. Also that this will be the last time that the members information will be submitted to HMRC. Setting this indicator will not trigger a Payroll Run, it will merely ensure that this member is included.

Click ‘OK’ to save the Member’s Pension Details information, or click ‘Cancel’ to exit the screen without saving the details entered.

Member’s Year To Date Details

Click ‘Menu’. Double click “Member’s Year To Date Details” then click ‘Update’.

The following screen appears when selected.

Member Year To Date Screen

The Member’s Year To Date Details screen is used to hold the year to date details about the member of the scheme.

Enter the fields as required as follows:

TOTAL GROSS PAY - PREVIOUS (P45) EMPLOYMENT - The member’s amount of gross pay from their previous employment/pension. This field is not accessible to the user.

TOTAL GROSS PAY - CURRENT EMPLOYMENT - The member’s amount of gross pay for their current employment/pension. This field is automatically updated when a payroll and/or year end are run. If the amount shown in this field needs to be amended, it can be done in this screen.

TOTAL GROSS PAY - TAX YEAR VALUES TO DATE - The member’s amount of gross pay to date. This field is automatically updated when a payroll and/or year end are run. It is not accessible to the user.

GROSS PAY FOR TAX - PREVIOUS (P45 ) EMPLOYMENT - The member’s amount of gross pay for tax from their previous employment/pension. The amount for this field can be found on the member’s P45 form.

GROSS PAY FOR TAX - CURRENT EMPLOYMENT - The member’s amount of gross pay for tax for their current employment/pension. This field is automatically updated when a payroll and/or year end are run. If the amount shown in this field needs to be amended, it can be done in this screen.

GROSS PAY FOR TAX - TAX YEAR VALUES TO DATE - The member’s amount of gross pay for tax to date. This field is automatically updated when a payroll and/or year end are run. It is not accessible to the user.

TAX PAID - PREVIOUS (P45) EMPLOYMENT - The member’s amount of tax paid from their previous employment/pension. The amount for this field cam be found on the member’s P45 form.

TAX PAID - CURRENT EMPLOYMENT - The member’s amount of tax paid for their current employment/pension. This field is automatically updated when a payroll and/or year end are run. If the amount shown in this field needs to be amended, it can be done in this screen.

TAX PAID - TAX YEAR VALUES TO DATE - The member’s amount of tax paid to date. This field is automatically updated when a payroll and/or year end are run. It is not accessible to the user.

PAYMENT NAME - PENSION - The member’s amount of pension. The amount for this field is generated from the ‘Total Current Pension’ field. This is held on Pension Master on the Pension History screen.

Click ‘OK’ to save the Member’s Year To Date information, or click ‘Cancel’ to exit the screen without saving the details entered.

Member’s Bank Details

Click ‘Menu’. Double click “Member’s Bank Details” and then click ‘Update’.

The following screen appears when selected.

Member Bank Details Screen

The Member’s Bank Details screen is used to hold the bank details of the member of the scheme.

Enter the fields as required as follows:

ACCOUNT TYPE - Select the Account Type.

BANK NAME - Enter the name of the member’s account.

BANK ADDRESS & POSTCODE - Enter the address and postcode of the member’s account.

COUNTRY - Select the country of the member’s account.

ACCOUNT NAME - Enter the name of the account of the member.

ACCOUNT NUMBER - Enter the account number of the member’s account.

SORT CODE - Enter the sort code of the member’s account. The “-“ seperators are automatically added.

BIC - Enter the BIC of the member’s account.

Current History Details

Click ‘Menu’. Double click “Member’s History Details”.

The following screen appears when selected.

Current History Details Screen

The details on this screen cannot be manually altered or amended, they are a record of the payments calculated at each payroll run throughout the tax year. The details are copied into the Historic Details screen (screen 2) when the Year End program is run. The Current History Details screen is cleared out in preparation for the new tax year.

The fields are described as follows:

TAX WEEK - The tax week the payroll was run in.

TAX MONTH - The tax month the payroll was run in.

DATE PROCESSED - The date the payroll was processed.

TAX CODE - The tax code of the member, used in the payroll.

WEEK1/MONTH1 - If the member was using the Week1/Month1 Basis for calculating tax a tick will be displayed here. If the field is blank the member was using the Cumulative Basis for calculating tax.

TOTAL GROSS PAY - The total gross pay of the member, calculated from the payroll.

TOTAL TAX PAID - The total tax paid of the member, calculated from the payroll.

TOTAL NET PAY - The total net pay of the member, calculated from the payroll.

Click ‘OK’ to save the Member’s History Details information, or click ‘Cancel’ to exit the screen without saving the details entered.

Deleting Member History Details

Click ‘Delete’, and you will be prompted to enter a password. This password will need to be set up in the Password Settings program. The following pop-up window will appear:

Enter Password

The field is described as follows:

PASSWORD - Enter the password that has been set up in the Password Settings program here.

Once a valid password has been entered the user will then be allowed to delete the latest history entry on the screen. On deleting this entry, the Year To Date screen will be updated showing the correct figures.

Historic Details

Historic Details

Member’s Pay Element Details

Click ‘Menu’. Select Member’s Pay Element Details and then click ‘Update’.

The following screen appears when selected.

Member Pay Elements Details Screen

The Member’s Pay Element Details screen is used to allocate pay elements to the member. These pay elements are then used in the member’s payroll run to either add or deduct amounts from the gross or net pay. The year to date figures are erased when the Year End program is run in preparation for the new tax year.

Enter the fields as follows:

CODE - Enter the pay element code. Click lookup icon for a list of valid pay element codes.

DESCRIPTION - The pay element code description will automatically be displayed to this field when the pay element code has been selected in the ‘Code’ field.

START - Enter the Start date the pay element code is to be effective from. If this field is left blank then it will be assumed that the pay element is active and should be processed so long as either the End date is blank or the End date is after the payroll date. If a date is entered then it must be on or before the End date, if an End date is also entered.

END - Enter the End date the pay element code stops being effective from. If this field is left blank then it will be assumed that the pay element is active and should be processed so long as either the Start date is blank or the Start date is before the payroll date. If a date is entered then it must be on or after the Start date, if a Start date is also entered.

AMOUNT - Enter the amount for the pay element code. This will default to the amount held against this code in the Pay Elements program. Only pay element codes with a non-zero value will be processed so long as the Start and End dates are valid for the payroll date.

AMOUNT YTD - The amount year to date is automatically set to 0.00 when a new pay element code is allocated to the member. This field is updated by the payroll run program with the amount used in the run.

Click ‘OK’ to save the Member’s Pay Element Details information, or click ‘Cancel’ to exit the screen without saving the details entered.

Printing Individual Screens, Full Member Details & Address Labels

Click the ‘Report’ icon.

The following window now appears:

Select Report Options Window

The options are as follows:

  • Current screen - This allows you to print off details of the current screen only.
  • Full member details - Print off a Full Member Report.
  • Address labels - Print off address labels for the current member or another group of members. See the section ‘Address Labels’ for further details.

You can also select the records to print in this dialog box. Further details of these options can be found here.

Select the appropriate option and click ‘OK’. Click ‘Cancel’ to the printing or ‘Help’ to display this help text.

Full Member Report

If you chose to print off a Full Member Report, a window will appear which asks which screens you wish to print for this member. All screens are defaulted to Yes.

If you wish to alter the defaults for this printout, simply click in the box next to the desired screen. A tick will appear for yes, or the box will go blank for no.

Click ‘None’ to de-select all screens for the printout.

Click ‘All’ to select all screens for the printout.

When you have finished, click ‘OK’ to proceed to the printing stage. The screen selection screen is as follows:

Report Selection Screen

Printing Address Labels

Printing Address Labels works using Mail Merge Report . When adding a Mail Merge Report check the ‘Use for Member Labels?’ box. This will indicate that this particular Mail Merge Report is to be used to generate Member Labels. When the report is saved, irrespective of the scheme it is saved under, the report ID and description will appear in the Member Labels selection window.

To generate member address labels select the Report option from the Members menu or press R. Select ‘Address Labels’ as the Report Type and either ‘Current Record’, ‘All Selected record(s)’ or ‘Select record(s)’ to print from the Records to Print section. Then select a Mail Merge Report from the list of those that have been previously setup to produce member labels. Your word processor will open and automatically generate the member labels that you have selected.

To return to the main menu when you have printed your labels, press E, or select the ‘Exit’ menu option with the arrow keys and press Return.

To return to the main menu when you have printed your labels, click ‘Exit’.

Instigating A Workflow Action

A member record must be selected by clicking Query before you can instigate a workflow action. Select ‘Workflow’ to instigate a workflow action for the current member.

The following pop-up window is now displayed:

Workflow Instigation Popup

Enter the workflow code for the action you wish to instigate or click the lookup icon for a list of available workflow codes. To set up a new workflow code, use the Workflow Management option.

Click ‘OK’ to continue or click ‘Cancel’ to abort this workflow action.

Screen Selection Screen

Producing Ad-Hoc Letters

If you wish to send an ad-hoc letter to a member, you can do this using the ‘Quick Letter’ facility.

Click ‘Quick Letter’. This automatically starts Microsoft Word and generates the members address, today’s date, and Yours sincerely… If the member’s address it not available, the address of the member’s company is used instead. Your name and job title will be automatically generated at the end of the letter.

When you have finished your letter, you can print it in the normal way. You can then exit from Microsoft Word as required.

To return to Payroll Master, simply click the titlebar of an active Payroll Master window.

Exit Program

Click ‘Exit’. This leaves the program and closes the current browser tab.
